# Slack connector catalog split

## Summary

The existing Socket Mode Slack connector remains the live bot driver, while the
workflow/catalog layer now distinguishes Slack app and Slack login connection
types.

## Design

- `puffer-connector-slack` still owns Socket Mode inbound events, mention
  gating, thread-scoped sessions, and bot replies.
- `slack-app` uses the same app/bot credential model for workflow/catalog
  purposes and direct Web API actions.
- `slack-login` is account-oriented and implemented through the shared
  `puffer-slack` client rather than the Socket Mode bot driver.
- Public connector constructors now carry explicit doc comments per repo
  constraints.

## Compatibility

No Socket Mode behavior changes. The split only gives workflows a clearer
connector/account model and action path.
