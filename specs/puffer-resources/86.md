# Browser Field Fill Contract

Browser and login resources now distinguish replacement input from appended
typing.

- `BrowserAction` explains that `fill` replaces editable control values and is
  the default for known final values in email, username, password, search, and
  address fields.
- `type` and `insertText` are documented as append-style actions that can
  duplicate existing field content when retried.
- Login guidance tells agents to use RequestSecret `username` metadata for
  email, phone, username, and account fields.
- `PUFFER_SECRET_...` placeholders are reserved for password, token, API key,
  and other secret-value fields.
