# puffer-resources update 54

The bundled Telegram skill is no longer user-invocable as a slash command.

- `resources/skills/telegram/SKILL.md` now marks the skill
  `user-invocable: false`, preventing `telegram` or `skill:telegram` entries
  from being synthesized into the slash-command surface.
- The skill guidance now tells agents to use `/connect telegram-login
  <connection>` for auth and the internal `telegram ...` CLI from Bash for peer
  lookup, message listing, and message search.
- The skill explicitly rejects a user-facing `/telegram` slash command so future
  prompt/resource edits do not reintroduce connector-specific slash aliases.
- `resources/internal_tools/telegram.yaml` remains the authoritative internal
  Telegram tool descriptor for Bash helper generation and runtime dispatch.
