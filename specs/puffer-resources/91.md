# Zhipu Image Batch Limit

## Scope

Zhipu BigModel synchronous image generation currently returns one image per
request and should not receive multi-image `n` requests.

## Change

- `zhipu` image execution now declares `max_images_per_call: 1`.
- A governance test asserts that the bundled Zhipu image descriptor keeps this
  limit.

## Compatibility

Multi-image Zhipu requests are fulfilled as repeated single-image requests under
one Puffer media job. OpenAI, xAI, BytePlus, Vercel AI Gateway, MiniMax, and
OpenRouter keep their existing image execution configuration.
