# Monitor Task Resource Fields

## Summary

Bundled tool resources now expose the fields needed by connector monitor
agents.

## Behavior

- `TaskCreate` documents monitor metadata and accepts `actions` plus
  `possibleIgnoreReasons` for `/tasks` choices.
- Each action requires `actionName` and `actionPrompt`.
- The internal Telegram tool schema advertises `list_messages` and `before_id`
  so monitor backfill agents can page through recent Telegram messages.

## Compatibility

The schema additions are additive. Existing `TaskCreate`, Telegram login, peer
lookup, and search inputs keep their prior shapes.
