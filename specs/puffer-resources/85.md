# Browser Readiness and Login Secrets

The model-visible browser and login resource contract now hardens the paths
that do not go through a loaded browser skill.

- `BrowserAction` no longer advertises `waitNetworkIdle` in its public action
  enum.
- Browser readiness guidance points agents to `snapshot`, `screenshot`,
  `domInspect`, or targeted actions after `open` and `navigate`.
- Legacy `idleMs` and `timeoutMs` fields are documented as internal
  wait-network-idle compatibility fields, not normal model inputs.
- The base system prompt instructs agents to search `RequestSecret` before
  entering reusable login credentials.
- If no stored credential exists, the base prompt tells agents to collect the
  reusable secret through `RequestSecret` instead of normal chat.
- One-time verification codes remain user-entered or masked transient input,
  not reusable stored credentials.
