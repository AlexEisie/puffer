# Provider Image Model Catalog Refresh

## Summary

Provider image resources now mirror the current official image-generation
models that can be executed by the existing media adapters. The catalog remains
strict: only models with documented generation endpoints and a compatible
adapter are advertised.

## Updated Static Resources

- OpenAI includes `gpt-image-2` ahead of the GPT Image 1.x fallbacks on the
  OpenAI Images API path.
- xAI includes both `grok-imagine-image-quality` and `grok-imagine-image` on
  the OpenAI-compatible image-generation endpoint.
- MiniMax CN includes `image-01-live`; global MiniMax remains limited to
  `image-01` because the global API reference only documents that model.
- Vercel AI Gateway static image fallbacks exactly mirror the official
  unauthenticated `/v1/models` entries whose `type` is `image`, and every
  static entry overrides provider-level chat image-output execution with the
  OpenAI-compatible `/images/generations` adapter.

## Unchanged Resources

- Zhipu already covers the officially documented GLM Image and CogView image
  generation models.
- BytePlus keeps only `seedream-5-0-260128`; official ModelArk documentation
  confirms that model on the OpenAI-compatible image-generation API, while
  older Seedream IDs were not added without equally direct official model-id
  confirmation.
- OpenRouter remains discovery-driven through `output_modalities=image`,
  because its official model catalog is designed to be queried dynamically.

## Compatibility Boundaries

Anthropic, Groq, Cerebras, Kimi, LM Studio, Ollama, vLLM, llama.cpp,
MiniCPM5, and WorldRouter are not given static image-generation descriptors in
this refresh. Their official docs either describe image input, local or
experimental behavior, or routing surfaces that do not map cleanly to an
implemented Puffer media execution adapter.

## Validation

Resource tests assert:

- OpenAI declares `gpt-image-2` with large GPT Image 2 size options;
- xAI and MiniMax CN include all documented static image models compatible with
  current adapters;
- Vercel static image fallback IDs are duplicate-free and exactly match the
  official Gateway image-type model list;
- every Vercel image-only fallback uses the `images_json` execution override.
