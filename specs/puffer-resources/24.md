# 24 - Lark Skill Subscription Boundary

The Lark skill now states that first-party `lark-app` and `lark-login`
connectors are action/lookup surfaces only.

- Agents must not create Lark-triggered workflows or Lark agent proxies until a
  typed subscribe runtime exists.
- The skill still documents account setup, id lookup, message search/read, and
  outbound `ConnectorAct` usage.

This keeps Lark guidance aligned with the connector catalog, where both Lark
templates have `can_subscribe=false` and `can_proxy_agent=false`.
