# Secret Question Resource Contract

The `AskUserQuestion` resource now documents first-class secret input handling.

- `secret: true` is available only for input questions.
- Secret input answers are returned as `PUFFER_SECRET_...` placeholders instead
  of raw text.
- Reusable credentials still belong in `RequestSecret action="collect"` so they
  are saved to the encrypted vault.
- The tool schema exposes the `secret` boolean so agents and UIs share the same
  contract.
- `RequestSecret action="create"` is documented as accepting exactly one
  `PUFFER_SECRET_...` placeholder, matching runtime validation.
- The browser skill names the managed CEF/Chromium browser and keeps the login
  workflow anchored on `RequestSecret` search/request/collect before manual
  credential entry.
