# GitLab Webhook Skill Resource

## Design

The bundled `gitlab-webhook` skill documents the deterministic
`/connect gitlab-webhook <connection>` setup flow, the default `/gitlab` path,
and the `puffer serve` runtime requirement.

## Contracts

- The skill explains that GitLab project or group webhooks should post to the
  configured public URL.
- It lists the recognized GitLab event families: issue, merge request, note,
  and push events.
- It documents setup-only catalog behavior so agents do not claim a workflow
  trigger runtime or outbound action support for this preset.

## Compatibility

The new skill only increases bundled skill inventory. Existing skill names and
aliases remain unchanged.
