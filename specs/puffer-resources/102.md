# Provider Media Resource Validation

Provider resource loading now validates exact media descriptors after layered
provider merging.

- `ProviderPack.media` maps directly into `ProviderDescriptor.media`.
- `load_resources` fails when the final provider descriptor has invalid media
  metadata, including empty declared image parameter arrays.
- Bundled OpenAI resources declare only concrete OpenAI Images-compatible image
  model ids for the `openai_images` adapter.
- Router-only resources such as WorldRouter keep chat routing separate and do
  not advertise image media capabilities.

This keeps invalid media YAML from reaching desktop, daemon, workflow, or tool
runtime capability resolution.
