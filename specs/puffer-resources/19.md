# Telegram multi-account resource guidance

- The `Telegram` internal tool schema now includes `connection_slug`,
  `account_slug`, and `account`, allowing agents to select a specific
  personal-account connection without relying on a singleton `telegram-user`
  process.
- Telegram skill guidance documents `telegram --connection <slug>` for
  importing multiple local Desktop accounts, searching peers/messages on the
  selected account, and using the same slug in workflows and connector
  actions.
- Login/import guidance now states that successful auth automatically
  registers the `telegram-login` connection; agents should use the returned
  connection slug rather than creating a duplicate connection manually.
- Connector action examples show account selection through top-level
  `connection_slug`; `ConnectorAct` also accepts `account_slug`/`account` as
  aliases when the connector connection represents an account. The action
  payload remains the connector-defined JSON shape.
