# Deterministic Connect Resources

Bundled resources now describe `/connect` as a deterministic local command
that calls typed connector helpers after `AskUserQuestion` answers are
collected.

- The `connect` prompt resource is removed. Connector auth no longer depends on
  a model-rendered instruction prompt or prompt-level allowed-tool filtering.
- `AskUserQuestion` keeps first-class `type: "input"` support for typed
  credential, phone number, login code, password, passcode, host, port, token,
  and URL values. Choice questions remain the method-selection surface.
- Telegram, Slack, Lark, and Email internal tool descriptions now state that
  `/connect` calls those helpers directly for auth. The skills continue to
  expose lookup-oriented internal CLI workflows.
- Connector skills route setup and auth repair to
  `/connect <connector-slug> <connection>` and keep Bash/stdin credential
  collection out of the normal auth path.

This supersedes the prompt-resource contract in `26.md`.
