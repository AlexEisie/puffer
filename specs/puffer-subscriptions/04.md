# Connector Reply Metadata

- Built-in connector message output schemas now allow `reply_to` and
  `reply_count` fields for platforms that expose reply relationships.
- Built-in `send_message` actions now document optional `reply_to` and
  `reply_to_message_id` inputs for sending a message as a reply.
- The fields are optional and keep existing `to` plus `message` action inputs
  compatible.
