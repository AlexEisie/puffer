# Connector Capability Honesty

## Scope

Corrects connector catalog capabilities used by workflow routing.

## Design

- `slack-app` remains an authenticated Slack app action connector, but it no
  longer claims subscription or agent proxy support.
- `slack-bot` remains visible as a legacy placeholder, but exposes no workflow
  actions and does not claim subscription or proxy support.
- `slack-login` continues to support direct Web API actions only.

## Compatibility

Slack inbound event workflows require a typed Slack `subscribe` runtime before
the catalog may advertise `can_subscribe=true`. This prevents workflows from
appearing active while no stream can start.
