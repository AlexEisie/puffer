# SlackBot Proxy Capability Alignment

## Scope

Aligns the subscription runtime's built-in proxy routing with the connector
catalog.

## Behavior

- `slack-bot` is not returned by `builtin_agent_proxy` until a typed Slack
  subscription stream and action path exist in this runtime.
- `handle_agent_proxy_event` ignores SlackBot events instead of binding a proxy
  that would later fail on `send_message`.
- TelegramBot proxy behavior is unchanged.

## Compatibility

The existing Socket Mode Slack connector remains separate. This change only
prevents the subscription manager from advertising or half-executing SlackBot
proxy behavior it cannot deliver end to end.
