# Monitor Triage Action Dispatch

## Summary

The subscription dispatcher supports `triage_agent` actions for connector
monitor workflows.

## Behavior

- Matching connector events render the action prompt with `{{text}}` and
  `{{payload.*}}` placeholders before dispatch.
- The workflow runner receives a structured `connection` trigger containing
  the envelope id, connection id, event text, event kind, dedup key, and raw
  payload.
- The action succeeds only when an installed workflow runner accepts the
  triage call; otherwise the router records a failed workflow action.

## Compatibility

This is additive to existing workflow actions. `run_workflow`, `tool_call`,
connector actions, file append, and SQLite actions keep their existing trigger
payloads and validation behavior.
