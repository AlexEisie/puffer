# 15 - Unknown Auth Check Is Warn-Only

Auth refresh now degrades a connection only when an auth check explicitly fails
or the checker returns an error.

- Connectors that require auth but do not expose an `auth-ok` implementation
  still log a warning when they have active consumers.
- They are no longer marked `degraded` solely because no checker exists.
- This prevents configured legacy subscribers such as `email` from being
  disabled by the routine auth monitor.

The one-time user-visible broken-auth notice remains tied to concrete auth
failure, not missing checker coverage.
