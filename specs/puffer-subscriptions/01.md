# puffer-subscriptions update 01

## Purpose

Subscription control commands can now be paired with a caller-visible terminal
event from the subscriber bus.

## Design

`SubscriptionManager::send_command_and_wait` subscribes to a topic before
writing the control command to the child process. It then waits for one of the
caller-provided event kinds or returns a timeout/bus error.

## Logic

The wait loop ignores unrelated events on the same topic and applies one shared
deadline across the whole wait. It returns the full `EventEnvelope` so workflow
tools can inspect payloads and surface precise subscriber errors.

## Contracts

Existing fire-and-forget `send_command` behavior is unchanged. Callers that
need handshake semantics should use `send_command_and_wait` with explicit
terminal event kinds and a bounded timeout.
