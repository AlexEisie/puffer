# Monitor digest owns noise handling

Monitor triage no longer applies the deterministic low-signal task gate before
dispatch.

- Router-side monitor skips remain for muted/silent notifications, installed
  ignore filters, contact filters, include filters, and classifier rejection.
- Broad monitor `triage_agent` bindings now enqueue every remaining matched
  event into the delayed digest, including acknowledgements, casual questions,
  and exploratory suggestions.
- The digest triage agent is responsible for using the full time-window context
  to distinguish actionable tasks from noise.
- `monitor_task_gate_skip` is no longer emitted for new monitor runs.

This favors conversation-level recall over early token savings. It prevents the
router from dropping useful context before the hourly digest agent can evaluate
the message batch.
