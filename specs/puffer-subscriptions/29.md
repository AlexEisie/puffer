# GitLab Webhook Connector Preset

## Design

The built-in connector catalog includes `gitlab-webhook`, a setup-only
serve-mode preset backed by the generic `puffer connector webhook` runtime.
It is intentionally parallel to the GitHub and Linear webhook presets:
it has no connector protocol command, no subscriber manifest, no outbound
actions, and no workflow trigger runtime capability.

## Contracts

- Suggested connection slug: `gitlab-webhook`.
- Skill: `gitlab-webhook`.
- Runtime hint: `serve`.
- Output schema: object-shaped `gitlab_event` metadata with event, object kind,
  action, project, sender, message, and URL fields.
- The preset must remain visible in the connector catalog without implying
  that it can subscribe or proxy agent events.

## Compatibility

Existing webhook, GitHub, and Linear presets are unchanged. The new preset only
adds catalog metadata and does not change stored connection records.
