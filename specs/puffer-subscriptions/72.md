# Telegram Bot Monitor Rule Fields

## Summary

The `telegram-bot` monitor rule schema exposes only user-facing routing fields:
message text, group chat status, and bot mention status.

## Contract

- `Message text` continues to map to the event text field.
- `Group chat` maps to the boolean `is_group` payload field.
- `Bot mentioned` maps to the boolean `bot_mentioned` payload field.
- Internal identifiers such as `conversation_id`, `user_id`, `thread_id`, and
  `from_bot` are not exposed as rule fields.

## Compatibility

Existing stored filters that target removed internal fields may still exist in
persisted workflow bindings, but new UI-created telegram-bot filters cannot
select those fields. The bundled UI matrix count for `telegram-bot` is reduced
to the remaining text and boolean rule combinations.
