# Jira Webhook Connector Preset

## Design

The built-in connector catalog includes `jira-webhook`, a setup-only
serve-mode preset backed by the generic `puffer connector webhook` runtime.
It follows the GitHub, GitLab, and Linear webhook preset pattern: catalog
metadata only, no connector protocol command, no subscriber manifest, no
outbound actions, and no workflow trigger runtime capability.

## Contracts

- Suggested connection slug: `jira-webhook`.
- Skill: `jira-webhook`.
- Runtime hint: `serve`.
- Output schema: object-shaped `jira_event` metadata with event, issue event
  type, issue key, project, actor, message, and URL fields.
- The preset must remain searchable without implying that it can subscribe or
  proxy agent events.

## Compatibility

Existing webhook presets and stored connection records are unchanged. The new
preset only adds catalog metadata and expands connector-list result counts.
