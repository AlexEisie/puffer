# Subscription review fixes

## Summary

Workflow routing now ignores subscriber control events, subscriber command
waits are runtime-safe from manager tasks, connector action failures preserve
`success=false`, and common command terminal events are matched against the
command shape before being accepted.

## Design

- `Event.control=true` envelopes are not counted or dispatched as workflow
  triggers.
- `send_command`, `send_command_and_wait`, subscriber start, and connector
  stream refresh paths use the runtime-aware blocking helper.
- `parse_action_response` honors a present `success` boolean even when the
  connector omits optional protocol fields such as `summary`.
- Command wait filtering rejects stale send, Telegram peer search, message
  search, and Telegram action terminal events whose payload clearly belongs to
  another command.

## Compatibility

Incoming message events keep `control=false` and existing serialized events
without the field deserialize as workflow-triggerable events.
