# Slack app and login connector templates

## Summary

The built-in connector catalog now models Slack app installs and Slack account
logins as separate connector templates.

## Design

- `slack-app` represents a Slack app install backed by bot/app tokens. It can
  subscribe, proxy agents, and perform Slack Web API connector actions.
- `slack-login` represents a workspace account backed by OAuth or local browser
  session tokens. It is action-capable and lookup-capable but does not expose a
  subscription stream in this update.
- Legacy `slack-bot` remains present for existing connector config and shares
  the Slack action set.
- Slack action metadata includes `send_message`, `react`, `send_reaction`, and
  `remove_reaction` with host-enforced external-side-effect permissions.
- The subscription manager accepts a host-provided auth checker for built-in
  connectors whose credentials are not owned by connector subprocesses.

## Compatibility

Existing Telegram and email templates are unchanged. Existing `slack-bot`
references continue to resolve, while new workflows can choose `slack-app` or
`slack-login` explicitly.
