# Media and poll connector action metadata

- Built-in connector templates still expose `send_message` for Telegram,
  Slack, and email-compatible connectors.
- `send_message` input metadata now documents media attachments, captions,
  file aliases, and reply target fields. The schema requires only `to`; host
  fallback validation requires either text/caption or media before enqueueing.
- The Telegram personal-account connector template exposes `vote_poll` with an
  `external_message_interaction` permission category. Slack, email, and
  Telegram bot templates keep only the generic actions they can actually run.
- `vote_poll` documents message id fields and answer selector shapes for
  single-select and multi-select poll votes.
- Message output schema includes optional `media` and `poll` fields so
  workflows can reason about downloaded files and structured poll answers.
