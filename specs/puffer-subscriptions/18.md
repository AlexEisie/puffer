# Router startup subscription race

- The subscription router now creates its event-bus receiver synchronously
  before spawning the async router task.
- This guarantees subscriber events published immediately after runtime
  startup, including Telegram catch-up events emitted before the live update
  loop, have an active broadcast receiver and are eligible for workflow
  matching.
- Router processing semantics are unchanged: control events are ignored,
  enabled bindings match on connection topic or connector topic, filters run
  before optional classification, and matched actions are dispatched through
  the configured action dispatcher.
- A regression test publishes an event immediately after router construction
  and verifies that a `file_append` action observes it.
