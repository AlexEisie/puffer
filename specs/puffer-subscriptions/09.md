# Manager runtime blocking audit

## Summary

The subscription manager no longer uses direct `Handle::block_on` for router
spawn or shutdown paths that can be reached from a Tokio runtime context.

## Design

- Manager construction enters the supplied runtime handle before spawning the
  router, avoiding a nested `block_on` call while still giving `tokio::spawn` a
  runtime context.
- Manager shutdown uses the existing runtime-aware blocking helper for router,
  subscriber, and connector-stream teardown.
- The helper remains the central policy for synchronous manager APIs that must
  wait on async runtime work.

## Compatibility

The public manager API is unchanged. The update only removes panic-prone
blocking in embedded runtime contexts.
