# Ignore filters

- Workflow bindings now carry `ignore_filters`, a list of normal Puffer-side filters evaluated after topic matching and before the action or classifier.
- When any ignore filter matches the connector event text/payload, the router suppresses the event and records no action run.
- JSON ignore filters use the existing subset matching semantics, so a filter such as `{"room_id": "security", "author_handle": "bot"}` suppresses only events with those payload fields.
- Existing bindings remain compatible because the field defaults to an empty list and is omitted when empty.
