# File append `/tmp` runtime validation

- The `file_append` runtime path resolver now treats absolute `/tmp/...`
  paths as valid while still rejecting parent traversal and non-temporary
  absolute paths.
- The previous resolver reused a helper that considered the root component of
  any absolute path unsafe, so specs could validate `/tmp/msgs` but dispatch
  failed at runtime.
- SQLite actions remain relative-only because they separately reject absolute
  paths before checking for parent traversal.
- A regression test dispatches `file_append` to a unique `/tmp` path and
  verifies the file is created.
