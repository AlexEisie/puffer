# Subscription Runtime Review Fixes

## Summary

The subscription manager now treats connector proxy support as a capability,
not an enabled consumer. A connection is active only when an enabled workflow
binding or explicit agent-proxy binding consumes it.

## Design

- `refresh_connection_consumers` computes proxy consumption only from
  `AgentProxyStore::has_enabled_consumer`.
- Connector streams are stopped when their connection no longer has a consumer,
  when the connection disappears, or when the connector template no longer has a
  runnable subscribe command.
- `send_command_and_wait` serializes command/reply waits per subscriber topic so
  concurrent tool calls cannot consume each other's terminal events.
- Built-in Telegram personal-account auth checks fall back to a lightweight
  `TelegramListPeers` probe when the subscriber is already running.
- Active `requires_auth` connections without any available auth checker degrade
  instead of being silently skipped.
- The Puffer path-expression filter keeps its limited grammar but parses string
  literals as JSON strings, so escaped quotes and backslashes are handled by a
  structured parser instead of manual slicing.

## Compatibility

Existing connector templates and workflow bindings remain valid. Telegram auth
checks still return `None` when the subscriber is not running, so inactive
connections are not degraded solely because no probe process is available.
