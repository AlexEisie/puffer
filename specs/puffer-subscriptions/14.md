# 14 - Connector Send Message Schema Parity

The built-in `send_message` connector action schema now reflects the target and
media shapes accepted by first-party direct action executors.

- Targets can be `to`, `target`, `channel`, `chat_id`, `open_id`, `user`, or
  `receive_id`.
- Reply-only payloads can satisfy the schema with `reply_to` or
  `reply_to_message_id`.
- Media kind metadata includes `audio`, `voice`, `video`, and `media` in
  addition to image/document/file aliases.

The schema remains permissive with `additionalProperties: true` because each
connector has platform-specific fields, but it no longer incorrectly requires
only `to` or omits supported media aliases.
