# Native File Append Actions

- Workflow bindings now support `action.type = "file_append"` for direct file
  writes from the subscription router without shell templating.
- `file_append.path` accepts safe relative paths rooted in subscription
  storage, plus absolute paths under `/tmp` for operator-requested scratch
  pipelines. Other absolute paths and parent traversal are rejected.
- `file_append.format` defaults to `text` and also supports `jsonl`. Text mode
  appends the event text plus a newline; JSONL mode appends the full connection
  trigger object, including envelope id, topic, kind, text, and payload.
- This action is intended for high-volume connector pipelines where `Bash`
  templating would be unsafe or flaky for arbitrary message text.

# Per-Connection Subscriber Instantiation

- Connector templates with a reusable subscriber manifest are instantiated for
  the connection before falling back to bundled exact-manifest identity.
- User or workspace manifests named exactly after a connection slug still take
  precedence and keep their declared identity.
- This prevents default connection names that match a bundled reusable
  subscriber, such as `telegram-user`, from accidentally using bundled
  resource state instead of per-account user state.
