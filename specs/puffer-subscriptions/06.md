# Telegram action catalog expansion

- The `telegram-login` built-in connector template now exposes Telegram
  personal-account actions for poll votes, message management, reactions,
  chat state, membership/admin operations, account profile updates, group
  metadata updates, avatars, and stories.
- Telegram-specific actions have host-enforced permission categories such as
  `external_message_interaction`, `external_message_admin`,
  `external_chat_admin`, `external_chat_membership`, and
  `external_account_profile`.
- `send_message` remains available on generic message-capable connectors.
  Telegram-specific actions are scoped to `telegram-login`.
- Action schemas intentionally stay permissive with documented common fields
  because Telegram action validation lives in the subscriber and varies by peer
  type and account permissions.
