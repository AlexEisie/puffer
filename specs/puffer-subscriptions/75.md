# Hourly monitor digest triage

Monitor `triage_agent` bindings now support delayed digest dispatch in the live
subscription manager.

- Production manager builds a `MonitorDigestQueue` with a one-hour interval.
  Local test runs can set `PUFFER_MONITOR_DIGEST_INTERVAL_SECONDS=300` to flush
  every five minutes without changing the production default.
- Matched monitor triage events are acknowledged quickly, recorded as
  `monitor_digest_queued`, and buffered in memory instead of starting an agent
  immediately.
- When the interval expires, queued events are grouped by monitor binding and
  sent through the existing `dispatch_batch` path as one triage turn.
- Non-monitor actions, agent-proxy routing, include filters, skip filters,
  contact filters, classifier checks, and muted-notification skips keep their
  existing behavior.
- Manager shutdown flushes any pending digest before stopping subscribers and
  connector streams.

This shifts monitor task creation from per-message immediacy toward hourly
conversation-level summarization while preserving the existing task schemas and
history storage.
