# Connector Subscriber Manifest Metadata

- `ConnectorTemplate` now has optional `subscriber` metadata describing a
  reusable subscriber manifest for connection-scoped event streams.
- `ConnectorSubscriberTemplate` carries a `manifest_slug`, optional
  per-connection `state_root`, and optional display-name prefix.
- Subscriber manifest discovery is centralized in `subscriber_manifest` with
  workspace, user, then bundled resource precedence.
- `connection_subscriber_manifest` loads a connection-specific manifest when
  present, otherwise instantiates the connector's configured shared manifest,
  otherwise preserves the legacy connector-slug manifest behavior.
- `SubscriptionManager` no longer contains connector-specific auth probes.
  Hosts provide those through `ConnectionAuthChecker`, which now receives the
  manager so it can use running subscribers without putting connector logic in
  the generic manager.
