# Lark Connectors Backed By lark-cli (login + bot)

## Change

Replaces the hand-made OpenAPI Lark connector (`puffer-lark` crate, `lark-app` /
`lark-login` templates, the `Lark` internal tool) with **two** command-backed
connectors that delegate to the official [larksuite/cli](https://github.com/larksuite/cli),
mirroring Telegram's login/bot split:

- **lark-login** — `--as user` (your personal account): streams your messages
  (monitor → tasks) and acts as you. No auto-reply.
- **lark-bot** — `--as bot` (the app bot): streams and auto-replies (agent proxy).

Both gain an incoming-message stream and (bot) auto-reply, which the old
connector lacked.

## Contract

- Two built-in templates (skill `lark`), `requires_auth = true`,
  `can_subscribe = true`:
  - `lark-login`: `command = ["puffer","__connector","lark-user"]`,
    `can_proxy_agent = false`, default connection slug `lark-user`.
  - `lark-bot`: `command = ["puffer","__connector","lark-bot"]`,
    `can_proxy_agent = true`, default connection slug `lark-bot`.
- Shared actions: `send_message` (text, media, reply), `react` /
  `send_reaction`, `remove_reaction`. Identity defaults to the connector's
  (`user`/`bot`); `as` input field or `LARK_CLI_AS` overrides.
- The `puffer __connector lark-user|lark-bot` bridge implements `auth-ok`
  (`lark-cli auth status`), `act` (`im +messages-send` / `+messages-reply` /
  `reactions create|delete`), and `subscribe` (`event consume
  im.message.receive_v1 --as <identity>`, re-emitted as puffer event frames).
  `LARK_CLI_BIN` overrides the binary.
- `/connect lark-login|lark-bot` verifies auth and records a `ConnectionCreate`;
  if `lark-cli` is missing it asks consent and runs the official installer
  (`npx @larksuite/cli@latest install`), else shows guidance. No token stored.
- Auto-reply: `builtin_agent_proxy("lark-bot")` → `LarkBotAgentProxy`
  (`lark-login` has none). Its `route_prompt` tells the agent it IS the bound
  persona and replies directly (no forwarding); `manager.rs` builds the proxy
  prompt via `AgentProxy::route_prompt`, whose default preserves Telegram's
  behavior. Command-backed connectors are skipped by the manifest-subscriber
  autostart.

## Compatibility

Auth is delegated to `lark-cli`. Lookups are no longer puffer actions; agents
call `lark-cli` directly per the `lark` skill. Old `lark-app` / `lark-login` /
`lark-cli` connection records no longer resolve to a built-in template and should
be re-created with `/connect lark-login` or `/connect lark-bot`. The old `Lark`
internal tool and its resource/registry entries are removed.
