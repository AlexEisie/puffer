# Lark Thread Container Reads

## Scope

Adds first-party client support for Lark thread message history.

## Behavior

- `LarkClient::read_messages` continues to read a `chat` container.
- `LarkClient::read_thread_messages` reads a `thread` container by thread id.
- Both paths share pagination, sort, and time-window query handling.

## Compatibility

Existing chat history callers are unchanged. Video sends now use Lark's
`media` message type while retaining `mp4` file upload classification.
