# Lark Connection Client

## Scope

`puffer-lark` owns first-party Lark/Feishu credential files and a blocking
OpenAPI client shared by workflow tools and connector actions.

## Design

- Credentials are stored under `<user-config>/lark-accounts/<connection>/credentials.json`
  with best-effort `0600` permissions.
- `LarkBrand` selects either international Lark (`open.larksuite.com`) or
  Feishu (`open.feishu.cn`) endpoints.
- `LarkAuthKind::App` stores `app_id` and `app_secret`; the client mints a
  tenant access token through `/open-apis/auth/v3/tenant_access_token/internal`.
- `LarkAuthKind::UserToken` stores a user access token and optional `app_id`;
  auth checks call `/open-apis/authen/v1/user_info`.
- The client validates standard Lark envelopes by requiring `code == 0` and
  returning the `data` object.

## Operations

- Chat lookup: list chats and search chats.
- User lookup: search users through `/open-apis/contact/v3/users/search`.
- Message lookup: read chat messages, read thread replies by thread id, search
  messages, and `mget` message details.
- Message actions: send message, reply, upload image/file, add reaction, and
  delete reaction by `reaction_id`.

## Compatibility

This crate follows the endpoint and request shapes used by the Lark CLI
reference: app credentials mint tenant access tokens; user-token operations use
user access tokens; chat ids are `oc_...`, user open ids are `ou_...`, and
message ids are `om_...`.

## Gaps

OAuth device flow and event subscriptions are intentionally not implemented in
this first crate. Catalog templates therefore do not claim subscription or
agent-proxy support.
