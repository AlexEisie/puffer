# 01 - Conservative Local Session Import

Slack local browser-session import must not silently combine credentials from
different workspaces or stale token candidates.

- `import_local_slack_session` rejects multi-workspace local profiles unless
  `workspace_url` is supplied.
- Token extraction requires exactly one `xoxc` token and exactly one `xoxd`
  cookie candidate in the selected profile scope.
- A `workspace_url` hint selects only matching workspace sources, borrowing a
  single unambiguous global cookie/token candidate only when needed.
- Slack workspace URLs are normalized to `https://<workspace>.slack.com` for
  both local import and explicit browser login.
- Browser-cookie requests normalize both decoded and URL-encoded `xoxd` input
  before writing the `d=` cookie header, so explicit `login-browser` input and
  local import share the same behavior.

This is intentionally conservative: when Slack desktop storage contains
multiple accounts or stale tokens, agents must ask for a workspace hint, a
narrower `--path`, or explicit `login-browser` tokens rather than guessing.
