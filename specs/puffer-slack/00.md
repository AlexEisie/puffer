# Slack credential and Web API support

## Summary

`puffer-slack` is a shared crate for Slack account/app credentials, local app
session import, and blocking Web API calls used by Puffer runtime tools and
connector actions.

## Design

- Credentials are stored per Puffer connection under
  `<user-config>/slack-accounts/<connection>/credentials.json`.
- Auth kinds distinguish Slack apps (`xoxb` + `xapp`), regular OAuth tokens,
  and browser sessions (`xoxd` + `xoxc` + workspace URL).
- The client verifies every Web API response through Slack's `ok` field and
  exposes auth testing, conversations/users listing, message read/search,
  posting, reactions, DM open, and external file upload.
- Local import scans standard Slack app storage paths or an explicit path,
  extracts browser tokens and workspace URLs, and supports a workspace URL hint
  for multi-workspace profiles.

## Compatibility

The crate does not alter the existing Socket Mode connector. It provides the
shared implementation needed by `slack-app` and `slack-login` while keeping
Slack secrets out of workflow JSON responses.
