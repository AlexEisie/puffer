# Desktop Workflow And Connector Test Expansion

## Summary

Adds regression coverage for the redesigned desktop workflow and connector setup
surfaces without changing runtime behavior.

## Desktop Workflow Coverage

- The Pipelines overview now has an explicit Playwright regression proving an
  ongoing run card opens the separate workflow detail page.
- The test verifies the selected run remains selected in the detail page and
  the trajectory output is visible.
- The same test verifies Back returns to the overview and a workflow row opens
  detail for a different workflow, preserving the overview/detail/create page
  separation.

## Connector Setup Coverage

- The fake daemon can now emit arbitrary connector setup `AskUserQuestion`
  fixtures for desktop tests.
- Settings -> Connectors now has a regression for dynamic password inputs,
  single-choice radio questions, and multi-select checkbox questions.
- The test asserts required answers gate submission and that
  `resolve_user_question` receives the exact dynamic answer shape, including
  multi-select arrays.

## Compatibility

- The default fake daemon connector setup payload remains unchanged for
  existing tests.
- No production daemon protocol, UI copy, or connector implementation contract
  changes are introduced.
