# 757 - Provider Model Introductions

## Summary

Provider setup cards now show a short model-family introduction where they
previously showed a raw model count.

## Design

- The provider setup view keeps using the existing provider registry snapshot.
- A small UI helper maps known provider ids to concise descriptions grounded in
  each provider's model access pattern: Claude plan access, GPT general tasks,
  Gemini structured answers, Copilot coding assistance, OpenRouter aggregation,
  Vercel AI Gateway routing, and custom OpenAI-compatible endpoints.
- Unknown providers fall back to a generic model-provider description, with a
  local-endpoint description for auth-free providers.
- The connect modal reuses the same introduction for unauthenticated providers
  so the copy stays consistent with the card.

## Compatibility

No provider ids, auth modes, model registry data, settings storage, or desktop
API contracts change. The `modelCount` field remains available for code paths
that still need it; this update only changes visible setup copy.
