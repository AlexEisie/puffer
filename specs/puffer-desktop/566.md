# MiniCPM5 One-Click Desktop Setup

## Scope

The desktop Providers pane and signed-in onboarding screen expose a single local
model card for MiniCPM5-1B behavior analysis.

## Change

- `LocalModelSetupCard` renders a current-UI `MiniCPM5 local model` card with
  install state, endpoint, provider registration path, recent progress, and one
  explicit `Install MiniCPM5` primary action.
- The primary action calls the local daemon `install_local_model` method. The
  daemon returns immediately with a job id, emits `local-model:minicpm5:event`
  progress updates, and refreshes status when the job completes.
- `Check status` runs the same daemon-backed status path and displays concrete
  diagnostics: platform support, venv, Python imports, model files, provider
  YAML, and the `/v1/models` health response. The button does not mutate
  install state.
- The Rust local-model installer embeds the shim and provider YAML from the repo,
  creates `~/.puffer/venvs/minicpm5`, downloads
  `openbmb/MiniCPM5-1B-MLX` into `~/.puffer/models/minicpm5-1b`, writes
  `~/.puffer/resources/providers/minicpm5.yaml`, and starts the OpenAI-compatible
  shim on `127.0.0.1:8088/v1`.
- The same action is idempotent: if the model is already installed, it verifies
  registration and starts the local server instead of redownloading weights.

## Compatibility

MiniCPM5 remains an auxiliary behavior-analysis model, not the default frontier
coding route. The card does not change the user's default provider/model. It only
sets up the local Puffer provider resource and server needed by the MiniCPM5
behavior pipeline. Unsupported hosts show the platform reason and disable the
install action.
