# Live timeline tool reconciliation

Desktop live tool cards now reconcile with persisted transcript tool calls by a
stable signature made from normalized tool name and normalized input before
falling back to rendered body text.

This prevents completed live tool cards from surviving a transcript reload when
the persisted tool output is formatted differently from the live output. The
live card is replaced by the persisted transcript entry instead of being
appended below later messages.

`AskUserQuestion` live tool events are also suppressed in the generic tool-card
path. User questions already arrive through the dedicated question event and
render as question prompts, so the raw tool card would duplicate the answered
question after reconciliation.
