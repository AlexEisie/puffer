# Pipeline Monitor Task Panel

## Scope

Corbina's Pipelines screen now shows connector monitor tasks produced by
`/monitor <connection>` and lets users start the same follow-up actions that the
TUI `/tasks` picker exposes.

## Behavior

- The workflow snapshot model accepts optional `monitor_tasks` and
  `monitor_task_error` fields from the daemon.
- Active, non-ignored monitor tasks appear below the connection picker and are
  filtered by the existing connector search box.
- Each task row shows the subject, task id, status, monitor connector, and
  monitor connection.
- Clicking a task starts `/tasks show <task-id>` in an agent session.
- Action buttons start the monitor action prompt in an agent session.
- The ignore button starts `/tasks ignore <task-id> <first-reason>` when the
  task has suggested ignore reasons, otherwise `/tasks ignore <task-id>`.

## Compatibility

The panel is hidden when no monitor tasks exist. Existing connector search,
setup, repair, and monitor command behavior is unchanged.
