# Exact Image Media Desktop Runtime

Desktop media settings and preview generation now delegate to exact provider
media capabilities.

- The Tauri backend loads provider resources, auth-store credentials, and the
  OpenAI base URL override before listing media capabilities.
- The previous OpenAI-only capability cache is gone; desktop capability DTOs
  come from the same resolver used by the daemon.
- Image generation executes the shared exact image runtime and returns
  succeeded job/artifact metadata instead of a fake queued job.
- The media settings modal saves only parameter values declared by the selected
  capability; generic image size fallbacks are used only when a capability does
  not declare values.

No provider-specific UI branches are added for image generation execution.
