# CEF Warmup Retry

Desktop CEF warmup no longer depends on a single fixed-delay attempt.

- Startup still preinitializes native CEF before the desktop WebView starts.
- Smoke and prewarm target creation now retry until the main Tauri window is
  available or a bounded warmup deadline expires.
- The retry loop avoids silently skipping prewarm when the window is not ready
  after the previous one-second delay.
- Warmup errors are logged once the main window is available; missing-window
  warmup exhaustion logs a dedicated diagnostic.
- The native CEF external message pump now keeps a low-rate idle pump alive
  while any non-closing browser slot exists, preventing DevTools HTTP and
  websocket discovery from starving after hidden prewarm targets settle.
