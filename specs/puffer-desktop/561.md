# Task Tab Monitor Creation

The Tasks tab now supports creating connector monitors without sending a chat
command.

- The top toolbar adds a compact `New monitor` action.
- Opening it shows one inline connection selector populated from trigger-ready
  connections in the workflow snapshot.
- Creating a monitor calls the daemon `task_monitor_create` RPC and replaces the
  local task snapshot with the returned `workflow_list`-compatible response.
- Enabled monitors whose connector connection has degraded auth render an inline
  warning with a reconnect action so missing monitor tasks are attributable.
- Empty connection state is explicit and non-blocking; users can still view and
  filter existing agent and monitor tasks.
