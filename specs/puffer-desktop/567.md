# Paper projects tab parity

- The Projects tab content now follows the latest Paper workspace screen: a compact white top bar, `Projects` title, compact search field, outline create button, and project cards without the previous session-history band.
- Each project row is a bordered 8px container with a 48px header, bold project title, library-backed action icons for pin, details, tags, and delete, plus a light session strip below.
- Session chips are compact inline controls with bold titles, lowercase status pills, elapsed time, and no per-chip edit/delete affordances, matching the Paper page hierarchy.
- Pinned projects keep the filled branding-purple pin icon. The delete icon is mapped through `lucide-svelte` as `trash`, so Projects tab controls continue to use the shared icon library.
- Existing data and callbacks are preserved for opening sessions, creating sessions, pinning, editing project tags, deleting projects, and search filtering.
