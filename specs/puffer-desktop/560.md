# Desktop Task Tab

## Summary

The desktop shell now exposes a Tasks tab for workflow task triage across agent
and monitor sources.

## Details

- The sidebar includes a Tasks navigation item between Workflows and Settings.
- The Tasks screen loads the workflow snapshot and renders one task per row with
  source, status, owner/scope metadata, timestamps, search, source filters, and
  status filtering.
- Monitor task actions can be started directly from the row. Opening a task
  starts the existing `/tasks show <task-id>` command in an agent session.
- The daemon workflow snapshot now includes a normalized `tasks` array and
  `task_error`, sourced from workspace tasks, per-session agent task stores,
  team task stores, and monitor task stores.
- Existing `monitor_tasks` remains present for compatibility with older desktop
  views and running daemons.

## Compatibility

The tab falls back to `monitor_tasks` when a running daemon has not yet been
updated to send the normalized `tasks` field. Existing workflow snapshot
consumers can ignore the new optional fields.
