# Corbina Connector Workflow Command Runs

The Pipelines screen now lets users run selected connector workflow commands
directly from the connector detail panel.

## Contract

- Selected trigger-capable connector details keep showing deterministic
  `/workflows new` and `/workflows append` command previews.
- Each preview has a play action when the desktop runtime exposes an agent
  command runner.
- Running the draft preview sends the exact `/workflows new ...` command to the
  agent session runner.
- Running the append preview sends the exact `/workflows append ...` command to
  the agent session runner.
- The run actions reuse existing connection-name and append-path validation and
  share the connector command busy gate so repeated clicks do not overlap.
- Existing copy actions, local draft creation, append binding creation, and
  connector setup actions keep their current behavior.

This only changes Corbina's Pipelines screen command affordances. It does not
add connector behavior, workflow execution semantics, connector auth, TUI
overlays, or webhook behavior.
