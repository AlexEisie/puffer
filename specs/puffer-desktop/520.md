# Connector Draft Pattern Picker

## Scope

Extends the Pipelines selected-connector setup panel so users can stage the
initial trigger pattern before copying or creating a workflow draft.

## Design

- Trigger-capable selected connectors show a `Draft pattern` input next to the
  connection name input.
- The selected workflow draft command appends the normalized pattern as the
  optional third `/workflows new` argument when the field is non-empty.
- Pattern command arguments are shell-quoted when needed so phrases such as
  `hello world` stay one argument for `/workflows new`.
- Creating a selected-connector workflow draft passes the staged pattern into
  the local workflow draft trigger.
- Blank patterns still omit the third command argument and keep the draft
  default of `.*`.

## Compatibility

Existing connector rows and connection rows keep emitting the same draft command
when no selected-connector pattern is staged. No daemon API changes are needed.
