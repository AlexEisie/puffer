# Workspace Project Card Shrink Guard

## Summary

The Projects view now keeps project cards at their natural height when a loaded
session page contains many distinct projects.

## Design

- `.pf-pw-list` remains a vertical flex container with scroll overflow.
- `.pf-pw-project` rows opt out of vertical flex shrinking so their header and
  agent strip are not compressed to thin clipped bars when the loaded page has
  more natural height than the viewport.
- The workspace UI regression fixture renders 30 one-session projects and
  asserts every project card remains tall enough to show its contents.

## Compatibility

This changes only the Projects list layout. Sidebar grouping, pagination,
search, project actions, and session cards keep their existing data and event
contracts.
