# Inline Canvas in chat

## Summary

Canvas tool calls now render an inline interactive Canvas inside the desktop
chat transcript. The generated browser HTML remains available as a fallback,
but users can toggle, select, slide, and type directly from the Canvas tool
card.

## Design

- `ToolCard` detects completed `Canvas` tool calls whose input is a Canvas
  component tree.
- `InlineCanvas` initializes control values from the same component tree used
  by the browser renderer.
- `InlineCanvasNode` renders the supported Canvas design-system nodes,
  including `toggle`, `singleSelect`, `multiSelect`, `barSelect`, `slider`, and
  `textInput`.
- Control changes call the existing daemon `canvas_state_update` RPC through a
  typed desktop API helper, so `CanvasState` reads the same state whether the
  user interacted inline or in the generated browser page.

## Compatibility

The existing Canvas workflow tool, generated HTML files, state-file format, and
`CanvasState` contract are unchanged. Inline rendering is additive and scoped to
the desktop transcript display.
