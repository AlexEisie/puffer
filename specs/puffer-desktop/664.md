# puffer-desktop Update 664: Chat Composer Scroll Anchoring

## Summary

The primary agent Chat transcript now follows composer height changes when the
user is already reading at the bottom of the conversation.

## Architecture

`ConversationView.svelte` keeps the behavior local to the main Chat composer.
Before a scheduled textarea resize, it checks whether the chat thread is near
the bottom using the same threshold-based pinned semantics as agentenv's
message list. After the textarea resize, it scrolls the thread to the bottom
only when the textarea height or overflow state changed and the thread was
previously pinned.

## Behavior

- Long drafts can grow the composer without hiding the newest message below the
  visible thread.
- Shrinking a long draft keeps the newest message anchored when the user was at
  the bottom.
- If the user has intentionally scrolled up, composer height changes do not
  force the thread back to the bottom.
- Session switching keeps its existing scroll-to-top behavior.

## Contracts

Submit payloads, attachment payloads, keyboard behavior, provider/model
routing, permission controls, and Ask Puffer/legacy composers are unchanged.

## Verification

Playwright coverage in `apps/puffer-desktop/tests/chat-session-ui.spec.ts`
asserts that composer autosize preserves bottom anchoring for pinned users and
does not steal scroll position from users who have scrolled upward.
