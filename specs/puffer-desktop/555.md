# Workflow Overview Rows

## Change

The Workflows overview is reduced to a single list surface with a Project-style
header. Each workflow now renders as one row with its name, slug, trigger,
compact pipeline preview, run count, success count, failure count, enabled
state, and detail affordance.

## Architecture

- `Workflows.svelte` keeps the existing detail editor and daemon-backed
  snapshot flow.
- Overview rows derive stats from `snapshot.runs` by workflow slug.
- Pipeline previews reuse normalized workflow graph metadata, but render as
  inline labels instead of graph cards.
- Search lives in the screen top bar beside the create action.
- The overview list is flush to the content area with no outer or table inset
  padding; rows keep their internal text spacing.
- Manual refresh is removed from the Workflows screen chrome; normal daemon
  refresh still happens on mount and after workflow mutations.
- Clicking a row still opens the workflow detail editor.

## Compatibility

The daemon contract is unchanged. No workflow definition, run, trigger, or
binding schema changes are introduced.
