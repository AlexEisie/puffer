# Vite Rust Target Watch Ignore

## Design

The desktop Vite dev server ignores Rust build output directories. On Windows,
Cargo can lock procedural macro DLLs under `src-tauri/target/debug/deps` while
Vite is walking the tree, which causes `EBUSY` watcher failures before Tauri can
open the app.

## Logic

`vite.config.ts` now sets `server.watch.ignored` for the Tauri-local
`src-tauri/target/**` directory and the workspace `target/**` directory. This
keeps HMR focused on frontend source files while Cargo continues compiling in
parallel.

## Compatibility

The change affects dev mode only. Production builds and frontend source HMR are
unchanged; Rust rebuilds remain owned by Tauri/Cargo rather than Vite.
