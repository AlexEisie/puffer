# Workflow Detail Canvas Chrome

## Change

The workflow detail canvas no longer repeats the workflow title in an inner
header. The top screen chrome is the only title source.

The selected-node editor now floats on the right side of the canvas only while
a trigger or node is selected. It can be closed without changing workflow
state. On narrow screens it falls back to a static bottom panel to preserve form
usability.

## Architecture

- `Workflows.svelte` removes the duplicate `.pf-run-header` detail heading.
- `workflow.css` positions `.pf-canvas-selected` absolutely inside the existing
  relative canvas main area.
- Detail pages open with no selected node. Selecting a graph node opens the
  panel; the panel close action clears the selection.
- Node removal is a bottom panel action rather than a header action.
- The graph, toolbar, selected node form, and save/toggle controls keep the
  existing state and mutation paths.

## Compatibility

This is a presentation-only change. No daemon RPC, workflow definition, run, or
binding contract changes are introduced.
