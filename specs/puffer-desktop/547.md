# Desktop Workflow And Connector Settings Redesign

## Scope

The desktop shell removes the Deployments navigation item from the primary GUI.
Any stale persisted `deployments` screen tweak is normalized back to the
workspace screen during settings load.

## Workflow UI

The Pipelines screen now starts on a workflow overview page. The overview shows
summary counts, running or queued workflow runs, searchable workflow definitions,
and recent workflow runs. Selecting a workflow or run moves into a detail page
with the existing graph editor, run trajectory, trigger controls, connector
catalog, and wiring inspector. Creating a workflow opens the same editor in a
dedicated create page with an unsaved draft selected.

This preserves the existing workflow snapshot contract:

- `workflow_list` remains the source of workflows, runs, connectors,
  connections, bindings, and monitor tasks.
- `workflow_save`, `workflow_toggle`, `workflow_binding_create`, and
  `workflow_binding_delete` keep their existing request and response shapes.
- Local draft tracking remains client-side until the user saves.

## Connector Settings

Settings adds a Connectors page backed by `workflow_list`. It does not encode
per-connector setup forms. The page selects a connector and connection slug,
starts `/connect <connector_slug> <connection_slug>` in a temporary agent
session, subscribes to that session's stream, and renders any
`user-question-request` payload as deterministic input controls.

Supported AskUserQuestion fields:

- `type: "input"` renders a text or password input, with password selected by
  credential-like prompt terms.
- `type: "choice"` renders radio choices.
- `multiSelect: true` renders checkbox choices.
- question text is used as the answer key sent back to `resolve_user_question`.

The page refreshes the connector snapshot when the setup turn completes, so new
connections appear without a full app reload. This keeps connector setup
generic: new connectors only need to expose a `/connect` flow that asks
questions through AskUserQuestion.

## Compatibility

The desktop fake daemon now emits a generic AskUserQuestion request for any
`/connect <connector> <connection>` test turn and records the resolved
connection in its workflow snapshot. Deployment-specific Playwright tests are
skipped because the Deployments tab is no longer reachable from the GUI.
