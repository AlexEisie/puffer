# Workflow Append Command Discovery

## Design

The Corbina pipeline connector picker now exposes the same deterministic append
workflow command that the TUI supports. Trigger-capable connectors and
connections are searchable by `append`, `file`, `save`, and their generated
`/workflows append ...` command.

## Contract

- The existing direct `workflow_binding_create` RPC remains the primary create
  path for the plus button.
- The selected connector panel shows a copyable `/workflows append` command for
  the current connection name, append path, and optional pattern.
- Planned connections include `--connector <connector-slug>` in the command.
  Existing connections omit it.
- The Append filter preset reuses the shared connector search input and does
  not add new connector presets or webhook catalog entries.

## Compatibility

This is a UI discovery change. It does not change daemon request payloads,
workflow binding persistence, connector schemas, or existing workflow action
rendering.
