# Corbina searchable question search status

Searchable `AskUserQuestion` prompts in Corbina now show match counts and a
query-specific empty state while the user filters options.

## UX

- Empty searchable prompts show the total option count before the user types.
- Filtered prompts show `N/T matches`, or `1/T match`, so connector pickers make
  it clear whether a query narrowed to the intended connector.
- Empty filtered prompts show the exact query that produced no matches while
  keeping the submit button disabled until the user selects a valid option.

This only changes the shared Corbina pending-question renderer. It does not add
or change connector implementations, connector auth, workflow storage, or
webhook behavior.
