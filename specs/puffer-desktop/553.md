# Browser Preview Backend Auto-Connect

## Design

The Vite browser preview now defaults to the local Corbina backend WebSocket
when it is running on localhost without an explicit backend URL. This removes
the need to pass `?pufferBackend=ws://127.0.0.1:1421/ws` during normal desktop
development.

## Architecture

`configuredBrowserDaemonHandshake` still prefers explicit URL params,
localStorage, and Vite environment variables. After those sources, non-Tauri
Vite development pages on localhost fall back to `ws://127.0.0.1:1421/ws`.

## Logic

- Tauri windows keep using the native daemon bootstrap path.
- Explicit browser backend configuration continues to override the default.
- Non-localhost pages and non-dev builds do not receive the fallback.
- The default handshake uses the existing development token fallback.

## Contracts

Browser preview RPCs continue to use the existing `DaemonClient` WebSocket
request/response protocol. No daemon or Tauri command contract changed.

## Compatibility

Packaged desktop builds are unchanged. Local browser previews now leave preview
mode automatically when the Tauri dev backend WebSocket is available on the
standard development port.
