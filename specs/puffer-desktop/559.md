# Workflow Detail Inspector Refresh

## Change

The workflow detail selected-node panel has been restyled as a compact
inspector. It no longer stretches to fill the right side of the canvas, uses a
clear header, tighter two-column field layout, softer form controls, a cleaner
provider switcher, compact wiring controls, and a quieter remove action area.

## Architecture

- `workflow.css` owns the inspector shell, header, field grid, wiring controls,
  and responsive single-column fallback.
- `Workflows.svelte` keeps the existing selected-node, trigger, provider,
  wiring, close, and remove behavior unchanged.
- Provider and wiring controls continue to use the same mutation handlers and
  workflow dirty-state flow.

## Compatibility

This is a presentation-only change. Workflow pipeline nodes, trigger settings,
dependency wiring, save behavior, and daemon contracts are unchanged.
