# Searchable Connector Questions

## Summary

Desktop `AskUserQuestion` rendering now preserves the typed question contract
used by deterministic `/connect` flows.

## Behavior

- Streamed and transcript-loaded question prompts preserve `type: input` and
  `searchable: true` metadata.
- Input questions render as direct answer fields instead of option lists with
  an `Other` row.
- Searchable single-choice questions render a filter field and only submit a
  selected catalog option, matching the TUI connector-selection behavior.

## Compatibility

Existing choice prompts keep their option rows and custom answer support.
Legacy custom-only prompts without `type: input` continue to use the existing
`Other` input path.
