# Desktop Workflow And Connector Responsive Polish

The desktop workflow and connector surfaces now handle narrow widths without
collapsing into unreadable columns.

- Settings owns its responsive rules in `settings.css`, so the connector page
  stacks navigation above content at narrow widths even when component CSS loads
  after global app CSS.
- Settings rows, connector setup forms, saved connection cards, and question
  panels collapse to a single content column on narrow screens.
- Workflow overview no longer shows a stale selected workflow slug in the page
  header.
- Workflow detail and create toolbars use clearer labels for creating a new
  workflow and adding provider nodes, and the create page no longer offers a
  redundant second create-draft button.
