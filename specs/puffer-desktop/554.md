# Connector Settings Modal Create Flow

## Design

The Connectors settings page now keeps creation out of the main page layout.
Users open a `New connection` modal, choose the connector, enter the
connection slug, and start setup from that modal. Dynamic setup questions remain
in their own modal.

## Architecture

`Settings.svelte` owns two local UI states for the connector page:

- `connectorCreateOpen` controls the create-connection modal.
- `connectorTab` switches the same page between `Connections` and `Catalog`.

The existing connector setup command, session event subscription, and
`AskUserQuestion` answer path remain unchanged.

## Logic

- `New connection` opens the create modal after ensuring a connector is
  selected.
- `Start setup` closes the create modal immediately and lets the background
  connector turn continue.
- A `user-question-request` opens the setup-questions modal.
- A completed setup switches the page back to the `Connections` tab.
- Clicking a catalog row selects that connector and opens the create modal.

## Contracts

No daemon RPC contract changed. The frontend still starts connector setup with
`/connect <connector> <connection>` and resolves dynamic `AskUserQuestion`
requests with the same answer payload shape.

## Compatibility

Existing connector definitions continue to work without per-connector UI code.
The settings tests cover the create modal, dynamic setup question modal, and the
Connections/Catalog tabs.
