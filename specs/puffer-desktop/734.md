# puffer-desktop Update 670: Media Generation Settings Labels

## Summary

Composer media configuration actions now use explicit generation-oriented
labels: `Image generation settings` and `Video generation settings`.

The image settings action no longer fails silently when the settings snapshot is
temporarily unavailable. The modal opens with a loading state, uses typed
defaults for rendering, and keeps saving disabled until the real settings
snapshot is present.

## Frontend Contract

- The Add content menu exposes media configuration as generation settings, not
  generic media settings.
- `MediaSettingsModal` accepts a `settingsReady` flag. When false, it renders a
  loading message and cannot save.
- The modal hydrates its local form state from the latest media settings once
  settings become ready, then reselects the default available capability when no
  saved provider/model exists.
- Saving still sends the existing `update_config` media payload contract through
  the daemon and preserves the opposite media kind.

## Backend Contract

- The Tauri preview backend and fake daemon continue to expose
  `list_media_capabilities` and `generate_media`.
- OpenAI image capability discovery treats stored OpenAI/Codex credentials,
  `OPENAI_API_KEY`, `PUFFER_OPENAI_API_KEY`, and native `HOME/.codex/auth.json`
  as available credential sources. Empty environment values are ignored.

## Tests

- Playwright covers image generation settings save behavior.
- Playwright covers video generation settings with no available capabilities.
- Playwright covers stale saved image model invalidation with the new labels.
- Rust backend tests cover `PUFFER_OPENAI_API_KEY` media capability discovery.

## Compatibility Notes

No legacy `Image settings` or `Video settings` UI labels are retained for the
composer actions. Existing daemon media config payloads remain unchanged.
