# Pipeline Connector Search Index

## Summary

The Pipelines connector picker now uses indexed connector and connection search
rows instead of scanning connector metadata on every visible connection row.

## Behavior

- Connector search accepts multiple whitespace-separated terms.
- Every term must match somewhere in the row's indexed metadata.
- Connector rows index slug, description, skill, and action slugs.
- Connection rows index slug, description, connector slug, and the matching
  connector's description, skill, and action slugs.
- Trigger-ready connection rows and connector template rows keep their existing
  selection behavior.

## Compatibility

Single-term searches keep the prior substring behavior. Multi-term searches add
the ability to find connectors by combining terms from separate metadata fields,
such as description and transport details.
