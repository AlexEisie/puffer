# puffer-desktop Stale Turn Retry

The desktop session loader now detects a stale loaded session whose transcript
still ends in a user message, whose activity is `running`, and whose last update
is older than the recovery timeout. When no live turn state exists in the UI, it
asks the daemon to recover the session.

A successful recovery reuses the existing session event stream and marks the
returned `turnId` as the active turn so stop/cancel UI behaves like a normal
agent run. If the daemon reports that the prompt was already retried once, the
desktop refreshes the session after the daemon appends the terminal marker,
unlocking the composer for manual follow-up.

The desktop does not decide retry eligibility alone. The daemon is authoritative
for in-flight turn checks, durable retry markers, and the final retry/no-retry
decision.
