# Turn Prose Activity Split

Assistant prose with a `turnId` belongs to the same visual assistant response
as the turn's tool work, but it is not itself agent activity. The conversation
row builder keeps those assistant messages in chronological order inside the
agent row and renders them with normal message-body styling.

Operational children such as tool calls, diffs, verified skill gates, approvals,
questions, and canvas offers remain activity surfaces. Completed turns with a
final assistant message may still collapse those operational children behind an
Agent activity header, but intervening assistant prose stays outside that header.

This preserves the live-stream shape after reload or interruption: text, Sleep
or other activity, more text, and later activity all remain one assistant turn
without converting the text chunks into intermediate-message activity boxes.
