# Pipeline Connection Repair Commands

## Summary

Corbina Pipelines can run deterministic connector repair/setup commands directly
from existing connection rows.

## Behavior

- Workflow connection rows accept optional `connect_command` from the daemon
  snapshot.
- Each connection row shows a wrench action that runs
  `/connect <connector_slug> <connection_slug>`.
- Connection search indexes the command and repair keywords, so searches such
  as `repair slack` find the matching connection.
- Monitor buttons remain separate and continue to use `monitor_command`.
- Older daemon snapshots without `connect_command` fall back to the same
  deterministic `/connect <connector_slug> <connection_slug>` shape.

## Compatibility

This is additive UI metadata and does not change workflow draft state or the
agent-session command path used to run connector setup.
