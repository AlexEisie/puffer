# Video Settings Media Directory

## Scope

The desktop video generation settings modal should expose the workspace video
media directory with the same path-row interaction used by image settings.

## Change

- The video settings modal displays `<session cwd>/.puffer/media/videos`.
- The field is read-only and uses the same `Open folder` outline action as the
  image settings media directory.
- `open_video_dir` creates and opens `.puffer/media/videos` for the session cwd.
- The Tauri command keeps rejecting non-absolute cwd values.
- Newly generated video artifacts are persisted under
  `.puffer/media/videos/<artifact-id>/...`, so the settings folder contains the
  MP4 outputs exposed by the modal.
- Generated-video access resolves the video directory first and keeps accepting
  legacy `.puffer/media/artifacts/<artifact-id>/...` paths for old sessions.

## Compatibility

This is additive UI and command surface. Existing video media settings and
capability loading are unchanged. Existing generated video artifacts under the
legacy artifact directory remain readable.
