# Monitor history digest clarity

Task monitor history now distinguishes shared digest outcomes from individual
message outcomes.

- The daemon history response may include `digest_batch_id`,
  `digest_batch_count`, `digest_batch_position`, and `digest_outcome_shared`
  for monitor messages whose triage result came from the same digest run.
- The Tasks history sidebar includes a compact digest message-count label.
- The selected-message detail shows a digest note explaining that the triage
  result used the whole batch rather than only the selected message.
- Shared outcomes are labeled as digest triage in the agent result section.

This makes casual context messages visible without implying that each one
individually created the displayed tasks.
