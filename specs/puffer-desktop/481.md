# Pipeline Monitor Command Snapshot

## Summary

Corbina Pipelines now uses daemon-provided monitor commands for connector task
monitoring actions.

## Behavior

- Workflow connection rows accept optional `monitor_command` from the daemon
  snapshot.
- Monitor-capable rows show a `monitor` capability tag and expose the exact
  command in the monitor button label/title.
- Connector search indexes the monitor command, so searches such as
  `monitor telegram` find trigger-ready Telegram connections.
- Older daemon snapshots without `monitor_command` still fall back to
  `/monitor <connection_slug>` for trigger-ready connections.

## Compatibility

This is additive UI metadata. It does not change workflow draft state, workflow
trigger selection, or the agent-session command path used to run monitor setup.
