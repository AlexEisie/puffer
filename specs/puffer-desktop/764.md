# Canvas Auto Expansion

## Summary

Canvas tool cards now auto-expand once when desktop detects that a transcript
tool call is a Canvas. This keeps the rendered inline widget visible in the chat
instead of leaving the tool row collapsed. Desktop-launched local daemons also
set `PUFFER_DESKTOP_INLINE_CANVAS=1` so the Canvas workflow suppresses the
external browser fallback for this inline-capable host.

## Behavior

- Canvas still renders through `InlineCanvas`.
- The card opens by default even when the tool input/output hydrates after the
  Svelte component first mounts.
- Collapsed `Agent activity` groups that contain a Canvas auto-expand once and
  select the Canvas child so the widget panel is visible immediately.
- The generated HTML artifact is still written, but Puffer Desktop no longer
  auto-opens it externally.
- Users can still collapse or expand the card manually after the initial
  auto-open.

## Compatibility

CanvasState is unchanged. CLI/TUI launches that do not set the desktop inline
marker keep the existing browser fallback.
