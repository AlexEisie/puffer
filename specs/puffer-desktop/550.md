# Workflow Terminology Cleanup

The desktop shell and workflow screen now use Workflows as the only visible
navigation label for the workflow surface.

- Sidebar navigation exposes `Workflows`; existing persisted `pipelines`
  screen preferences migrate to the `workflows` screen id.
- The workflow screen component and stylesheet are named for workflows, and all
  visible fallback text says workflow instead of pipeline.
- Desktop Playwright coverage moved to `workflows-ui.spec.ts` and opens the
  Workflows navigation item before testing overview, detail, create, connector,
  monitor, and run interactions.
- The serialized daemon contract still uses `workflow.pipeline.*` fields. This
  is intentionally preserved to avoid a protocol and storage migration.
