# Project Delete Confirmation Paper Style

The Projects tab delete action now opens a dedicated confirmation dialog before
calling the project deletion handler. The project row button only requests the
dialog; the destructive app callback runs from the dialog's confirm button.

The confirmation dialog uses its own Paper-matched classes instead of the shared
workspace modal shell so visual changes are not diluted by generic modal styles.
It keeps the centered white surface, subtle white scrim, compact header, red
triangle-alert icon treatment, concise body copy, neutral cancel button, and red
destructive confirm button. It intentionally omits the project path and extra
irreversible action hint to match the Paper modal.

No backend contract changes are required. The app still deletes by folder path
through the existing `onDeleteProject` callback after confirmation.
