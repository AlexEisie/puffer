# Windows OAuth Browser Open

## Summary

Desktop OAuth login no longer opens authorization URLs by hand with
`cmd /C start`. The local Tauri OAuth fallback delegates external URL opening to
`tauri_plugin_opener`, so Windows query parameters such as `client_id`,
`redirect_uri`, and `scope` are not reparsed by `cmd.exe`.

## Behavior

- The OAuth authorization URL remains generated by the provider-specific login
  bundle.
- Browser launch success/failure is still represented as a boolean to the
  surrounding login flow.
- Windows URL handling is delegated to the existing Tauri opener stack instead
  of constructing a shell command in Desktop code.
- macOS and Linux behavior also uses the same opener abstraction instead of
  Desktop-maintained `open` / `xdg-open` command branches.

## Compatibility

OAuth request parameters, redirect listener behavior, callback parsing, and
credential storage are unchanged. The change only affects how Desktop asks the
OS to open the already-built authorization URL.
