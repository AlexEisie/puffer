# Task configure filter rules

- The Tasks Configure dialog now includes a read-only Filter rules section.
- Users can select any monitor binding and inspect its trigger rule plus daemon-owned ignore filters.
- Payload ignore filters are rendered as compact scalar key/value rules, while tagged regex and expression filters keep their native pattern/expression text.
- Monitor memory editing and new monitor creation remain in the same dialog; filter rules are intentionally view-only so the daemon remains the source of truth.
