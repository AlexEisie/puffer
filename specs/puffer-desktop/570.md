# Modal Header And Footer Standardization

Workspace modals now use the project delete confirmation dialog as the shared
header and footer style reference. Modal headers keep only the primary title and
close/status controls, without eyebrow labels or subtitle copy. Modal footers
keep only action buttons, right aligned, without explanatory hint text.

The shared `.pf-modal-head`, `.pf-modal-title`, `.pf-modal-close`,
`.pf-modal-foot`, and `.pf-modal-foot-btns` classes were aligned to the delete
confirmation spacing, typography, and white footer treatment. Existing modal
bodies retain their operational field hints and validation/status text because
those are part of the form workflow rather than header/footer chrome.
