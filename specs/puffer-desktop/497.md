# Connector-First Pipeline Drafts

## Scope

Corbina's Pipelines connector picker now supports creating a workflow draft
directly from a searched connection or trigger-capable connector.

## Behavior

- Trigger-ready connection rows expose a plus action that creates a disabled
  draft using that connection as the workflow trigger.
- Trigger-capable connector rows expose the same draft action. Existing
  connections are preferred; otherwise the connector's suggested connection
  slug is used as a planned trigger.
- The selected connector setup panel also exposes draft creation and respects
  the typed deterministic connection slug before the user runs `/connect`.
- Setup-only connectors keep their existing setup action and do not create
  trigger drafts.
- Draft slugs derive from the connection slug, for example
  `telegram-user-workflow`, and still use the existing dirty/save path.

## Compatibility

The existing connector-row selection behavior, setup command buttons, monitor
buttons, and workflow save RPC contract are unchanged. New drafts remain
disabled until the user explicitly saves and resumes them.
