# Pipeline Connection Dropdown Readiness

## Summary

The Pipelines editor connection dropdown now follows the same workflow-trigger
readiness rules as the searchable connector picker.

## Behavior

- Switching a workflow to the `connection` trigger type chooses the first
  trigger-ready connection instead of the first connection in raw daemon order.
- Existing connections that cannot start workflow triggers remain visible in
  the dropdown, but are disabled and labeled `no trigger`.
- Changing the dropdown routes through the same trigger selection helper used
  by the searchable connection rows, so direct selection cannot bypass
  readiness checks.

## Compatibility

Planned connection trigger values are still shown when a workflow references a
connection that does not exist yet. Older daemon snapshots without readiness
metadata keep the prior `can_subscribe` fallback.
