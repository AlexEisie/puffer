# Browser Preview Workspace Handshake Fallback

## Summary

Browser-preview reopen now falls back to the current workspace daemon handshake
when the fixed development WebSocket default is unavailable.

## Design

- The Vite dev server exposes `/__puffer/daemon-handshake`, backed only by the
  workspace `.puffer/daemon.handshake` file.
- `ensureLocalDaemonClient` keeps explicit URL, stored, and environment
  handshakes authoritative.
- Only the fixed `ws://127.0.0.1:1421/ws` dev default can retry through the
  workspace handshake endpoint after its first connection attempt fails.
- Browser storage reads and writes are guarded so embedded previews that deny
  local storage can still use URL or workspace handshakes.

## Compatibility

Existing Playwright fake-daemon tests and explicit preview URLs keep their
current behavior. Reopening the local browser preview without query parameters
can now reconnect to the daemon that `puffer daemon` wrote for the workspace,
so workspace-scoped state such as saved inferred contacts remains visible.
