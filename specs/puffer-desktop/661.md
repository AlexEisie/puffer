# Chat Surface Attachment Drop

## Scope

Desktop chat now accepts file drops across the active chat surface, matching the
agent-sdk pattern where the transcript area and composer share one drop target.

## UX Contract

- Dragging files over the chat surface shows a centered overlay.
- The overlay says `Drop files to attach` and shows the existing limits:
  10 files, 20 MiB each.
- Dropping files adds them to the existing composer attachment draft strip.
- The composer keeps using the same image thumbnail and file-card UI for
  dropped files and file-picker files.
- Drops are ignored whenever the composer cannot accept input.
- Non-file drag payloads are ignored and do not activate the overlay.

## Architecture

The chat surface owns drag state with a small drag-depth counter. This avoids
flicker from nested `dragenter` and `dragleave` events while keeping the logic
local to `ConversationView.svelte`.

No new upload queue, staging API, or transport format is introduced. Dropped
files flow through the existing `addAttachmentFiles` path, so validation,
object URL creation, draft persistence by session, and preview cleanup stay in
one place.

## Testing

The desktop chat Playwright suite covers:

- `dragenter` on the chat surface shows the overlay;
- `drop` adds image and PDF drafts to the composer preview strip;
- sending the message forwards attachment metadata with the turn request;
- submitted chat messages render attachment cards instead of fallback
  `[Image: ...]` or `[File: ...]` text.
