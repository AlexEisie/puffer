# Connector Setup Question Modal

## Design

Connector setup questions now appear in a modal dialog instead of as an
embedded Settings panel. The dialog uses the existing desktop modal shell,
focus trap, and fixed footer pattern.

## Architecture

`Settings.svelte` still consumes the same `user-question-request` event and
normalizes arbitrary `AskUserQuestion` inputs into text, radio, and checkbox
controls. Only the rendering container changed from an inline section to a
modal form.

## Logic

- Starting setup still runs the selected `/connect <connector> <connection>`
  command.
- When the daemon asks for user input, Settings opens a modal with generated
  controls.
- Submitting the modal resolves the original user-question request.
- The modal stays open until answers are submitted or the turn completes/errors.

## Contracts

The daemon RPC and session event contracts are unchanged. Tests now assert that
connector setup questions render in an `aria-modal` dialog while preserving
dynamic input behavior and submitted answer payloads.

## Compatibility

Existing connector definitions do not need per-connector UI code. Any
connector that emits `AskUserQuestion` input, choice, or multiselect questions
continues to render through the same dynamic form.
