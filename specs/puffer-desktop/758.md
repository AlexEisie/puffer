# 758 - Custom Provider Display Name

## Summary

The custom provider setup form now asks for a display name alongside the base
URL and API key.

## Design

- The custom provider modal requires a non-empty display name before the
  connection button is enabled.
- The desktop UI sends the name as `openaiDisplayName` with the existing
  `openaiBaseUrl` config patch used by OpenAI-compatible custom endpoints.
- The daemon persists the field as `openai_display_name` and includes it in the
  settings snapshot.
- Runtime provider registry setup applies the configured name to the built-in
  OpenAI provider when custom OpenAI-compatible routing is configured, so
  provider pickers can show the user's chosen label.

## Compatibility

Existing configs without `openai_display_name` continue to load with no display
override. Existing custom provider connections keep their base URL and API key
behavior; the new field only affects UI labeling and provider summaries.
