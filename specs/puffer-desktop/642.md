# Browser Preview Renderer Gate

The Browser pane now resolves the requested renderer against the runtime before
probing or opening any backend. If the UI is running in the Tauri desktop shell,
the configured renderer remains active and CEF can be used. If the UI is running
in a plain browser preview without Tauri APIs, the pane forces the screencast
renderer.

This keeps CEF as the desktop default while preventing Vite/browser development
from calling native CEF setup paths that cannot complete outside the Tauri shell.
