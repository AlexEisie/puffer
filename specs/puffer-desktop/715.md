# Browser Backend Handshake Persistence

## Summary

Browser-preview daemon handshakes supplied through URL parameters are now
remembered in local storage after a successful configuration read.

## Design

- `configuredBrowserDaemonHandshake` separates explicit URL-provided backend,
  token, and workspace-root values from stored or environment fallbacks.
- When any explicit URL value is present, the normalized handshake is written to
  the existing `puffer.backendUrl`, `puffer.backendToken`, and
  `puffer.workspaceRoot` keys.
- Remote-preview handshakes mirror the same behavior with the existing
  `puffer.remote*` keys.
- The write path is browser-preview only. The Tauri shell keeps using its native
  daemon startup and restart commands.

## Compatibility

Existing URL parameter names, legacy `corbina*` parameter names, environment
fallbacks, and dev-backend fallback behavior are unchanged. Reopening a browser
preview without the original query string now reconnects to the remembered
daemon/workspace instead of falling back to an unrelated dev backend, preserving
workspace-scoped state such as inferred contacts.
