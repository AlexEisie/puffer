# Poster Backed Generated Video Cards

Date: 2026-06-09

## Scope

Desktop chat generated video cards display poster images as thumbnails while
preserving generated video access tickets for overlay playback.

## Behavior

- Message attachment thumbnail loading uses `readMessageAttachmentPreview` for
  generated video attachments.
- Generated video thumbnails call `read_generated_media_preview` and do not call
  `create_generated_video_access` before the user opens the overlay.
- Video attachments with a preview URL render an `<img>` poster inside the card
  with the existing play indicator.
- Missing poster previews fall back to the video file card without exposing
  local generated media paths.
- Opening a generated video requests `create_generated_video_access` and renders
  the returned ticket URL in the overlay video element.
- Live `/video` success attachments rely on the same poster preview loading path
  as persisted session history.

## Compatibility

The frontend API shape is unchanged. Generated image thumbnails continue to use
the generated media preview RPC, and generated video playback continues to use
daemon-issued access tickets.
