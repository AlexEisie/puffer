# Workflow Graph Editing

## Change

The workflow detail graph now behaves like an editable canvas. Nodes can be
dragged to reposition them, the canvas uses a dotted background, and dragging
from a node's right-side handle to another node creates a dependency edge.

Animated node glow has been removed. Selection and active state are shown with
static border and background styling.

## Architecture

- `Workflows.svelte` stores per-workflow node positions in local component
  state and derives graph bounds from those positions.
- Pointer movement updates node positions in graph coordinates, accounting for
  the current graph scale.
- Graph zoom is measured from the default pipeline width rather than dragged
  node bounds, so moving nodes does not resize the rendered nodes.
- Edge creation uses a temporary draft edge while dragging from the source
  handle, then writes the target node's `depends_on` list through the existing
  workflow mutation path.
- Trigger-to-node wiring clears the target dependency list, preserving the
  existing trigger edge semantics.
- `workflow.css` renders the dotted canvas background, drag cursors, larger edge
  handles, and a non-animated active node state.

## Compatibility

Node layout positions are local presentation state only. Saved workflow
definitions continue to use the existing pipeline node and `depends_on`
contracts; no daemon RPC or schema changes are introduced.
