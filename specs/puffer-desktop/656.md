# Secrets Settings Lazy List

## Summary

The desktop Secrets settings page now renders large vaults incrementally instead
of mounting every secret row at once.

## Behavior

- The list initially shows 30 secrets from the current settings snapshot.
- Reaching the list footer, or clicking the footer button, appends the next 30
  secrets until the snapshot is exhausted.
- Any snapshot item-set change resets the visible window to the first 30
  entries so saves, deletes, imports, and refreshes render from a predictable
  top-of-list state.
- Secret labels, metadata, and ids are visually truncated with native tooltips
  carrying the full value, preventing long names from widening the settings
  pane.
- The list has local metadata search over id, label, source, username,
  description, and origin. Search results still render in 30-item batches.

## Compatibility

The daemon settings snapshot contract is unchanged. Lazy loading is purely a
frontend rendering behavior over the already-loaded `secrets.items` array, so
save, delete, Chrome import, and metadata display semantics remain unchanged.
Search is local to the loaded metadata snapshot and never asks the daemon to
reveal secret values.
