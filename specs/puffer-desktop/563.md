# Task Monitor Memory Panel

The Tasks screen now exposes monitor memory returned by the daemon.

- `WorkflowSnapshot` accepts `monitor_memories` and `monitor_memory_error`.
- The Tasks screen renders a compact `Monitor memory` section above the task
  rows whenever memory files are present.
- Each memory item shows its connection slug, ignored-example count, path, and
  markdown content inside a collapsible details panel.
- Snapshot loading preserves memory errors in the same error path as task,
  connector, and binding failures.

This makes ignored monitor examples visible from the same place where monitor
tasks are reviewed, while keeping ignored tasks separated in their own tab.
