# Task Queue Redesign

## Scope

The desktop Tasks screen should be easier to scan when monitor tasks produce a
large queue and each task has multiple possible actions.

## Change

- The task stream now uses compact rows by default and opens a selected-task
  inspector only after the user clicks a task message area.
- The selected-task inspector can be closed, returning the queue to full width.
- Row-level action buttons remain in the queue. Suggested actions, monitor
  ignore, and open-in-agent controls do not require opening the inspector.
- The inspector shows the selected task summary, status, owner, kind, scope,
  timestamp, monitor memory path, and ignore analysis when present.
- Filtering, search, configure, history, refresh, and monitor warning behavior
  are unchanged.

## Compatibility

This is a renderer-only change. Task snapshot, monitor history, ignore, and
agent command RPC contracts remain unchanged.
