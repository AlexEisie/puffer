# Pipeline Draft Creation

## Scope

Corbina's Pipelines screen now exposes an explicit `New` workflow action in the
editor toolbar.

## Behavior

- `New` creates a local unsaved workflow draft named `Workflow draft` with a
  unique `workflow-draft` slug.
- The draft is selected immediately, marked dirty, and uses the existing Save
  flow to persist through the daemon `workflow_save` RPC.
- New drafts are disabled by default to avoid activating a trigger before the
  user reviews the graph.
- When a trigger-ready connection exists, the draft starts with that connection
  trigger; otherwise it falls back to the workspace task subscription topic.

## Compatibility

Existing workflow edits, save behavior, connector search, monitor panels, and
pause/resume controls are unchanged. Dirty local drafts remain protected from
snapshot refreshes through the existing dirty workflow merge path.
