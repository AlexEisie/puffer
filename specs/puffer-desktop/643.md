# Browser Tab Selection Stability

Browser tab selection now keeps a short local selection grace period after a user
clicks a tab. During that period, tab-list updates from the backend cannot
immediately overwrite the selected tab with an older active-tab value.

The selection is still reconciled with backend state once the backend reports the
same active tab, when the selected tab disappears, or when the grace period
expires. This preserves responsive tab switching without permanently diverging
from daemon state.
