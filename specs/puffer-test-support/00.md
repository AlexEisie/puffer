# puffer-test-support

## Purpose

`puffer-test-support` provides reusable helpers for Puffer integration tests,
terminal rendering tests, snapshot tests, temp workspaces, tmux probing, and
repository contract tests.

## Design

The crate is test-only support for other crates. It keeps process capture,
virtual-terminal rendering, snapshot normalization, and temporary filesystem
setup out of production crates.

It also contains repository-level contract checks for coding standards that
should remain visible during workspace test runs.

## Logic

Command helpers must run Puffer commands in controlled temporary workspaces and
capture stdout, stderr, exit status, and terminal output.

Virtual-terminal helpers must render captured escape sequences into plain text
for assertions and snapshots.

Snapshot helpers must normalize unstable data and update snapshots only when
`PUFFER_UPDATE_SNAPSHOTS=1` is set.

Tmux helpers must create isolated tmux servers/sessions for parity probing and
clean them up after tests.

Repo contract tests must scan Rust source for public function docstrings and
file-length limits.

## Contract

This crate may depend on test utilities and dev-only crates. Production crates
must not depend on it outside tests.

Snapshot updates are opt-in through the environment. Normal test runs must fail
on snapshot drift instead of silently rewriting expected output.

Contract tests enforce current repo constraints:

- Every public Rust function should have a docstring.
- Rust source files should stay under 1000 lines.

Helpers should prefer deterministic temp paths, normalized output, and explicit
assertion errors.

## Architecture Details

`puffer-test-support` is intentionally a dev/test dependency boundary. It gives
tests shared tools for process capture, terminal capture, snapshot assertion,
temporary workspaces, and repo-wide contract checks without leaking those
helpers into production crates.

The crate should remain deterministic and side-effect-contained because many
tests run in parallel and manipulate environment variables, tmux sessions, and
temporary Puffer homes.

## Contract Details

- Process contract: command helpers return captured stdout, stderr, status, and
  terminal output without assuming success.
- Terminal contract: VT/tmux helpers normalize terminal state into comparable
  text.
- Snapshot contract: snapshots are read-only unless
  `PUFFER_UPDATE_SNAPSHOTS=1` is set.
- Workspace contract: temp workspace helpers create isolated cwd, user config,
  resource, and session paths for tests.
- Tmux contract: helpers must create isolated tmux server/session names and
  clean them up.
- Repo contract: contract tests should point to concrete files/lines for missing
  docstrings or overlong Rust files.
- Failure contract: helper errors should preserve command/path context to avoid
  hiding the actual failing test behavior.
