# puffer-subscriber-runtime update 02

Adds a typed Telegram Desktop import command.

Design and behavior:

- `SubscriberCommand::TelegramImportTdata` carries optional `path`,
  `passcode`, `account_index`, and `key_file` fields.
- The command is intentionally first-class instead of `Custom` so subscriber
  implementations can handle or reject it exhaustively.
- Non-Telegram subscribers should ignore it with the same `command_ignored`
  behavior used for Telegram login commands.

Compatibility:

- Existing subscriber command JSON remains unchanged.
- New fields are optional and skipped when absent.
