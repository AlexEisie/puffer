# Subscriber control event flag

## Summary

Subscriber events now carry an optional `control` boolean. It distinguishes
command replies and lifecycle notices from incoming external messages.

## Design

- `control` defaults to `false` for backwards compatibility.
- Control events serialize the field only when true.
- Workflow routers can ignore control events while command waiters still
  receive them on the normal event bus.

## Compatibility

Existing subscriber JSON that omits `control` keeps the old message-event
behavior.
