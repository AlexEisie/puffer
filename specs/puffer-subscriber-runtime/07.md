# Telegram media command contract

- `SubscriberCommand::SendMessage` now carries optional `media` attachments in
  addition to `peer`, `text`, and `reply_to`.
- Each attachment is a typed `SendMediaAttachment` with `path`, optional
  `caption`, optional `kind` (`auto`, `photo`, `document`, `file`), optional
  `mime_type`, and optional `thumbnail`.
- `text` remains the message body for plain sends and becomes the default
  caption for the first attachment when media is present.
- Poll votes and other Telegram-specific operations are connector actions, not
  first-class runtime variants. Built-in Telegram fallback uses
  `SubscriberCommand::Custom { op: "telegram_act" }` for those operations.
