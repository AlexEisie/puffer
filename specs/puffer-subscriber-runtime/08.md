# Telegram action bridge

- Telegram-specific actions remain connector actions and are not modeled as
  first-class `SubscriberCommand` enum variants.
- Built-in fallback uses `SubscriberCommand::Custom` with `op:
  "telegram_act"` and an `args` object containing the connector `action` slug
  and original `input` JSON.
- This keeps the runtime command protocol generic while allowing the local
  `telegram-user` subscriber to execute personal-account actions when no
  external connector `act` binary is configured.
