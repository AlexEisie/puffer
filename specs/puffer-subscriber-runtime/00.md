# puffer-subscriber-runtime

## Purpose

`puffer-subscriber-runtime` runs out-of-process subscriber skills and publishes
their events to Puffer subscription routing.

## Design

Subscriber skills are child processes described by `manifest.toml`. The runtime
starts each process, feeds control commands over stdin, parses stdout as NDJSON
events, forwards stderr to tracing, and publishes parsed events onto an in-memory
broadcast bus.

Main parts:

- `Manifest` describes subscriber ID, kind, topic, run command, environment,
  and state directory.
- `EventBus` broadcasts `EventEnvelope` values by topic.
- `SubscriberSupervisor` owns child lifecycle, restart policy, and stdin sender.
- `CommandSender` serializes control commands to child stdin.

## Logic

Manifest loading must validate version 1, subscriber kind, topic, run command,
and state dir behavior. `ensure_state_dir` must create the state directory and
set `PUFFER_SKILL_STATE_DIR`.

Process startup must set:

- `PUFFER_SKILL_ID`
- `PUFFER_SKILL_TOPIC`
- `PUFFER_SKILL_STATE_DIR`

Runtime output handling must:

- Parse one JSON event per stdout line.
- Wrap each event with envelope ID, subscriber ID, and received timestamp.
- Publish to the event bus without blocking the child process.
- Log malformed stdout lines as subscriber errors.

Supervisor shutdown must terminate the child and stop restart attempts.
Unexpected exits may restart with exponential backoff unless restart is
disabled or shutdown is in progress.

## Contract

Subscriber stdout is a strict NDJSON event stream. Subscriber stderr is
diagnostic-only.

`SubscriberCommand` is the stdin control contract. It includes Telegram login
commands, email configuration, send-message commands, and custom commands.

The event bus contract is at-least-in-memory broadcast delivery. Lagging
consumers may skip events with a warning; publishers must not block forever on
slow consumers.

Child processes must be isolated from Puffer internals except for environment,
stdio, and their declared state directory.

## Architecture Details

The subscriber runtime is a supervisor and event pipe. It does not classify
events, run subscription actions, or interpret platform-specific payloads.
Those responsibilities belong to `puffer-subscriptions` and subscriber
processes.

Supervisor state is per subscriber. Event fanout is process-wide through
`EventBus`, with topic filtering applied by subscribers to the bus receiver.

## Contract Details

- Manifest contract: version, id, kind, topic, command, environment, and state
  dir are validated before process spawn.
- Process environment: child processes receive subscriber ID, topic, and state
  dir in environment variables.
- Event envelope contract: every accepted event is wrapped with envelope ID,
  subscriber ID, and receive timestamp.
- Restart contract: unexpected process exit restarts with bounded backoff unless
  shutdown or restart-disabled state applies.
- Command contract: `CommandSender` serializes JSON commands to stdin and
  reports disconnected children explicitly.
- Backpressure contract: event publishing must not block permanently on slow
  consumers.
- Shutdown contract: dropping or shutting down a supervisor must stop the child
  and prevent further restarts.
