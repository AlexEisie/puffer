# puffer-subscriber-runtime update 01

## Purpose

Subscriber manifests now normalize their containing directory to an absolute
path at load time.

## Design

`Manifest::load` still accepts relative and absolute manifest directories, but
stores the canonical directory in `Manifest::dir`. Existing callers continue to
load manifests the same way.

## Logic

State directories declared in `manifest.toml` remain relative to the manifest
directory. Because `Manifest::dir` is absolute, `ensure_state_dir` now returns
an absolute path even when the caller loaded the manifest through a relative
path.

## Contracts

Subscriber children receive an absolute `PUFFER_SKILL_STATE_DIR`. This prevents
the child process from reinterpreting a relative state path under its own
`current_dir` and creating nested state directories.
