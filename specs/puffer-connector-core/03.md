# puffer-connector-core update 03

`puffer-connector-core` now exposes the connector/workflow redesign types from
`puffer-subscriptions` so connector-facing crates can use one shared model.

The re-exported modules are:

- `catalog`: connector templates, action metadata, and action permission
  definitions.
- `connection`: authorized connection records and lifecycle states.
- `protocol`: stdin/stdout connector protocol frames for `subscribe`,
  `auth-ok`, and `act` integration.
- `proxy`: the `AgentProxy` interface, persistent `AgentProxyStore`, event
  decision helper, and Telegram bot / Slack bot proxy implementations. Proxy
  bindings preserve both the external principal and the connector reply target
  so channel/group replies are not confused with user identity.

This crate remains the compatibility boundary for existing connector runtime
code. Legacy connector runtime APIs are unchanged; the redesign types are
additive and avoid duplicating schema definitions across crates.
