# puffer-connector-telegram

## Purpose

`puffer-connector-telegram` connects a Telegram bot account to Puffer Code
sessions. It supports private chats and mentioned group chats.

## Design

The crate uses Teloxide for Telegram polling and message delivery. Telegram SDK
types are converted into `puffer-connector-core` inputs before any agent work.

Config includes bot token, allowed numeric user IDs, mention requirements,
group-key policy, and optional welcome text.

## Logic

Message handling must:

- Ignore messages sent by bots when appropriate.
- Enforce `allowed_users` by Telegram user ID when configured.
- Treat private chats as addressable.
- Require mention or reply-to-bot in group chats by default.
- Strip `@botname` mentions from the user text before dispatch.
- Build conversation keys from chat and user IDs.
- Run connector-core built-in commands before agent dispatch.
- Split outbound text with the Telegram splitter limit.
- Retry transient Telegram send errors.

## Contract

The connector requires a Telegram bot token. It does not implement personal
Telegram account login; that belongs to the telegram-user subscriber.

Group session behavior must honor `GroupKeyPolicy`, with `PerUser` as the
default for group chats.

Telegram message IDs and chat IDs are adapter metadata only; durable Puffer
conversation state lives in the session store.

## Architecture Details

The Telegram bot connector keeps Teloxide polling and send behavior local while
using connector-core for command/session/agent behavior. Private chats,
mentioned groups, and reply-to-bot interactions are normalized into shared
conversation keys.

The connector is bot-account only. Personal-account MTProto behavior lives in
the telegram-user subscriber process.

## Contract Details

- Required config: Telegram bot token.
- Optional config: allowed numeric users, mention policy, group-key policy, and
  welcome text.
- Private-chat contract: private messages are accepted without mention.
- Group-chat contract: group messages require bot mention or reply-to-bot by
  default.
- Mention contract: accepted text should have `@botname` removed before agent
  dispatch.
- Identity contract: allowed users compare Telegram numeric user IDs.
- Output contract: responses must respect Telegram message length limits.
- Safety contract: bot-originated loop messages are ignored.
