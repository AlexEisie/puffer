# puffer-observability

## Purpose

`puffer-observability` provides optional OpenTelemetry trace export for Puffer
runtime activity. It is primarily shaped for Langfuse OTLP ingestion while
remaining compatible with generic OTLP/HTTP collectors.

The component must stay opt-in and conservative: no endpoint means no
observability handle, no spans, and no exported data.

## Design

The crate exposes a small observability API:

- `ObservabilityConfig` describes endpoint, service metadata, release, tags,
  headers, sampling, shutdown timeout, and content-capture flags.
- `ObservabilityHandle` owns the OTel tracer provider and redaction policy.
- Attribute helpers provide typed names for `gen_ai.*`, `langfuse.*`, and
  `puffer.*` span attributes.
- Hook helpers create span guards for agent loops, turns, provider calls, tools,
  compaction, reflection, and subagent generation.
- Redaction helpers decide whether prompts, outputs, and tool I/O are recorded
  or summarized.

The rest of Puffer passes `Option<&ObservabilityHandle>` through runtime code.
When absent, span helpers return no-op guards.

## Architecture Details

This crate is instrumentation infrastructure, not telemetry policy. It does not
start itself, discover credentials from project files, or force tracing on. The
caller decides whether to initialize from explicit config or environment.

The exporter uses OTLP/HTTP with the OpenTelemetry SDK. Span construction is
synchronous from the caller perspective, and shutdown is bounded so trace flush
cannot hang CLI exit indefinitely.

Langfuse-specific behavior is expressed through attributes and helper
configuration, not by coupling the rest of Puffer to Langfuse APIs.

## Logic

Initialization logic must:

1. Return `Ok(None)` when no OTLP endpoint is configured.
2. Parse OTLP headers from `OTEL_EXPORTER_OTLP_HEADERS`.
3. Add Langfuse Basic auth and ingestion-version headers only when both
   `LANGFUSE_PUBLIC_KEY` and `LANGFUSE_SECRET_KEY` are present.
4. Parse sample rate, release, tags, and content-capture flags from environment.
5. Build an OTLP `/v1/traces` exporter and tracer provider.
6. Register the provider globally for background span creation.
7. Build the redaction policy from explicit include flags and always-redact tool
   IDs.

Span helpers must no-op when tracing is disabled, propagate parent span context
for child spans, record token usage on provider spans, mark cancellation/error
states, and apply redaction before content attributes are attached.

Redaction logic must default to summary-only output. Raw prompts, assistant
outputs, and tool I/O are captured only through explicit include flags. Tool IDs
on the always-redact list are redacted even when tool I/O capture is enabled.

## Contract

The public runtime contract is:

- `ObservabilityHandle::try_init_from_env()` returns `Ok(None)` for the normal
  off state.
- `ObservabilityConfig::for_langfuse()` constructs a Langfuse OTLP endpoint and
  required headers from host plus public/secret keys.
- `SpanGuard::Disabled` is safe to create and drop without side effects.
- `SpanGuard::Active` ends the span on drop and supports explicit error,
  cancellation, token usage, string, numeric, and redacted content attributes.

The environment contract includes:

- `OTEL_EXPORTER_OTLP_ENDPOINT`
- `OTEL_EXPORTER_OTLP_HEADERS`
- `OTEL_SERVICE_NAME`
- `PUFFER_OBSERVABILITY_SAMPLE_RATE`
- `PUFFER_OBSERVABILITY_INCLUDE_PROMPTS`
- `PUFFER_OBSERVABILITY_INCLUDE_OUTPUTS`
- `PUFFER_OBSERVABILITY_INCLUDE_TOOL_IO`
- `PUFFER_OBSERVABILITY_INCLUDE_CONTENT`
- `PUFFER_OBSERVABILITY_RELEASE`
- `PUFFER_OBSERVABILITY_TAGS`
- `LANGFUSE_PUBLIC_KEY`
- `LANGFUSE_SECRET_KEY`

## Contract Details

- Default privacy contract: content capture is off by default.
- Secret contract: Langfuse secret keys may come from environment, not workspace
  TOML.
- Sampling contract: sample rate `>= 1.0` is always-on, `<= 0.0` is always-off,
  and intermediate values use parent-based trace-id ratio sampling.
- Attribute contract: callers should use exported constants and `AttributeBag`
  rather than stringly typed attribute names.
- Redaction contract: redacted content includes byte length and short SHA-256
  prefix, not the original text.
- Size contract: raw captured content is truncated at the per-attribute cap on a
  UTF-8 boundary.
- Shutdown contract: callers that need deterministic flush should call
  `ObservabilityHandle::shutdown`; flush is best-effort within the configured
  timeout.
- Exclusion contract: this crate must not reintroduce default telemetry,
  analytics, feedback uploads, or reporting flows.
