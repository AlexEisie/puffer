# Telegram action bridge compatibility

- Removing the poll-specific Telegram runtime command leaves the email
  subscriber with only generic `SendMessage`, `EmailConfigure`, Telegram login
  helper commands, and `Custom` to match.
- Telegram connector actions routed as `Custom { op: "telegram_act" }` are
  rejected by the existing unknown-custom-op path, which is the correct
  behavior for email.
- Email text sending remains unchanged; media attachments are still ignored by
  the SMTP fallback.
