# Alertmanager Webhook Normalization

## Change

The generic webhook connector now recognizes Prometheus Alertmanager webhook
notifications and converts them into `InboundMessage` values after the more
specific Grafana Alerting normalizer.

## Recognition

A payload is considered Alertmanager-shaped when it has a non-empty `alerts`
array, top-level `status` of `firing` or `resolved`, a `receiver`, and either
webhook payload `version` 4 or an Alertmanager `groupKey`. Payloads with
Grafana-specific `orgId` or `state` fields are intentionally rejected so
Grafana deliveries keep their dedicated routing path.

## Conversation Routing

Conversation ids are stable by receiver and group or alert subject:

- `alertmanager:<receiver>:group:<group-key>`
- `alertmanager:<receiver>:alert:<fingerprint>`
- `alertmanager:<receiver>:alert:<alertname>`

If no subject id exists, the connector falls back to a receiver-scoped
Alertmanager group conversation.

## Message Summary

Summaries include status, receiver, alert count, group key, group labels, common
labels, common annotations, up to five alert summaries, source URLs, truncated
alert count, and the Alertmanager external URL when those fields are present.

## Compatibility

The Alertmanager normalizer runs after Grafana in the provider chain because
Grafana Alerting and Alertmanager share a related alert payload shape.
