# Trello Webhook Normalization

## Summary

Adds Trello-specific handling to the generic webhook connector. The router now
accepts webhook validation `HEAD` requests and normalizes Trello action payloads
into Puffer inbound messages.

## Routing

The webhook path accepts `HEAD` so Trello can validate `callbackURL` during
webhook creation. The same origin and bearer-token checks used by `POST` apply
to validation probes, so unauthenticated Trello validation works with the
`trello-webhook` preset while token-protected generic webhooks remain protected.

Trello conversion recognizes:

- `X-Trello-Webhook`
- JSON bodies with an `action` object containing `type` plus `data`
- JSON bodies with an `action` object and updated `model`

## Message Shape

Puffer groups the conversation by the first stable Trello subject in the
delivery, for example `trello:card:<card-id>`. Messages include action type,
actor, board, subject, list, URL, timestamp, and comment text when present.

## Compatibility

Native Puffer JSON, GitHub, Linear, Asana, Jira, Stripe, and GitLab payload
routing is unchanged. Trello signature headers are used only as provider hints;
the preset does not verify signatures until it collects an application secret.
