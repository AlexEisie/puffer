# Asana Webhook Normalization

## Summary

Adds Asana-specific handling to the generic webhook connector. The router now
accepts Asana's `X-Hook-Secret` handshake, acknowledges signed heartbeat
deliveries with empty `events`, and normalizes non-empty event arrays into one
Puffer inbound message.

## Routing

The webhook handler now reads the request body as bytes so handshake requests
without JSON can be accepted. Normal JSON webhook payloads continue to parse
through the same inbound conversion path.

Asana conversion recognizes:

- `X-Hook-Signature`
- `X-Hook-Secret`
- JSON bodies with `events` containing Asana-style `action` plus a resource or
  parent object with a `gid`

## Message Shape

Puffer groups the conversation by the first stable resource in the delivery,
for example `asana:task:<gid>`. Multiple Asana events in one delivery are
summarized in a single message and include resource, action, change field,
parent, actor, and timestamp when present.

## Compatibility

Native Puffer JSON, GitHub, Linear, Jira, Stripe, and GitLab payload routing is
unchanged. Empty Asana heartbeats are acknowledged without starting an agent
turn so Asana does not delete otherwise idle webhook subscriptions.
