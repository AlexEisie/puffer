# Jira Webhook Payload Normalization

## Design

The webhook router recognizes Jira webhook requests through payload
`webhookEvent` and Jira-shaped `issue`, `comment`, `project`, and
`issue_event_type_name` fields. Jira payload parsing lives in
`router/payloads.rs` alongside GitLab parsing so the main router stays below the
Rust file line limit.

## Logic

- Issue events thread as `jira:<project>:issue:<issue-key>`.
- Comment events thread to their parent issue when issue metadata is present.
- Project events thread by project key or id.
- Messages include event, issue event type, actor, subject, URL, compact body
  excerpt, and updated changelog fields when available.

## Compatibility

Native Puffer, GitHub, and Linear payload parsing remains in the same order as
before. Jira parsing is attempted before GitLab and only accepts Jira-shaped
payloads.
