# GitHub Payload Normalization

## Summary

The webhook connector accepts GitHub webhook JSON payloads in addition to the
native Puffer webhook payload.

## Logic

- Native payloads with `conversation_id`, `message`, and optional `user_id`
  keep their existing behavior.
- GitHub payloads are recognized from repository fields and
  `x-github-event` headers.
- Issues, pull requests, discussions, comments, and pushes are normalized into
  `InboundMessage` values with stable `github:<repo>:...` conversation ids.
- The generated message includes repository, event, action, sender, subject,
  URL, body snippet, and push commit summaries when available.
- Unknown valid JSON that matches neither shape is rejected with HTTP 400.

## Compatibility

Bearer-token auth, origin checks, health checks, and response shaping remain
unchanged. GitHub support is transport-level normalization only; secret
signature validation is not part of this change.
