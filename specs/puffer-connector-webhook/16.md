# Bitbucket Webhook Normalizer

The webhook connector now recognizes Bitbucket Cloud repository callbacks and
converts them into provider-specific inbound messages before the generic webhook
fallback.

## Design

- Payload detection uses Bitbucket's event header when present and otherwise
  accepts repository payloads that contain Bitbucket-specific `push`,
  `pullrequest`, `issue`, or `comment` objects.
- Supported event families include repository push, pull request lifecycle,
  pull request comments, and issue/comment payloads.
- Conversation ids prefer stable identifiers in this order:
  `bitbucket:<workspace/repo>:pull-request:<id>`,
  `bitbucket:<workspace/repo>:issue:<id>`,
  `bitbucket:<workspace/repo>:push:<branch-or-reference>`, then
  `bitbucket:<workspace/repo>:<event>:<request-or-event>`.
- Message summaries include event key, repository, actor, pull request or issue
  title, branch names, state, URL, comment body, and short commit summaries when
  present.

## Compatibility

The shape guard rejects generic repository-shaped payloads without a Bitbucket
event marker or Bitbucket-specific body fields. Existing provider normalizers
keep their precedence because Bitbucket runs after Asana and before Jira.
