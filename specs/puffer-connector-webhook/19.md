# Vercel Webhook Preset

## Scope

Adds first-class normalization for Vercel account and integration webhook
payloads received through the generic `puffer serve` webhook connector.

## Design

- A new provider normalizer recognizes Vercel webhook envelopes with `type`,
  `id` or `createdAt`, and an object `payload`.
- Supported event families include `deployment.*`, `project.*`, `domain.*`,
  `flag.*`, `firewall.*`, `integration.*`, and `billing.*`.
- Conversation IDs prefer stable Vercel identifiers:
  - deployment events use deployment id
  - project environment variable events use project id plus env var id
  - project domain events use project id plus domain
  - project events use project id
  - domain and flag events use their own stable names
- Rendered messages include actor, team, deployment, project, domain, flag,
  target, plan, regions, dashboard links, event timestamp, and event region
  when present.

## Compatibility

The normalizer is additive. Unknown Vercel-like events and unrelated `type`
payloads fall through to later provider normalizers or the generic webhook
formatter.
