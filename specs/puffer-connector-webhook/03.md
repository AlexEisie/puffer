# GitLab Webhook Payload Normalization

## Design

The webhook router recognizes GitLab webhook requests through `X-Gitlab-Event`,
`X-Gitlab-Event-UUID`, and payload `object_kind` markers. GitLab payload parsing
is isolated in `router/payloads.rs` so the main router stays below the Rust file
line limit.

## Logic

- Issues thread as `gitlab:<project>:issue:<iid>`.
- Merge requests thread as `gitlab:<project>:merge-request:<iid>`.
- Notes/comments thread to their parent issue, merge request, or commit when
  parent metadata is available.
- Push events thread by ref and include a short commit summary.
- Messages include event, action, sender, subject, URL, and a compact body
  excerpt when available.

## Compatibility

Native Puffer, GitHub, and Linear payload parsing remains in the same order as
before, with GitLab attempted only after those formats fail.
