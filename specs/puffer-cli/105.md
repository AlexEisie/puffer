# Monitor ignore analysis

- Ignoring a monitor-created task now appends the memory entry, marks the task ignored, and derives a pre-agent ignore filter from stable metadata when available.
- Filter derivation is connector-agnostic: the daemon accepts only exact JSON event-payload filters that contain both a stable scope identity and a stable actor/source identity.
- Creating or resuming a monitor refreshes its generated triage prompt so existing monitors copy only exact structured source identity metadata without asking the triage agent to propose ignore filters.
- The workflow list RPC backfills those filters from already ignored monitor tasks, allowing older ignores to take effect after the next refresh.
- During that refresh, pending monitor tasks whose exact metadata-derived filter matches a prior ignore filter are marked completed/ignored so already-created repeats leave the active queue.
- Each ignore also starts a read-only, no-tools analysis agent with the ignored task, metadata, memory path, and installed filter. The agent can only report concerns or propose memory text; the daemon owns all filter writes and does not automatically apply the analysis output.
