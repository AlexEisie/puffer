# Monitor Memory Snapshot Context

The daemon workflow snapshot now includes monitor memory files so clients can
show the ignore context used by monitor triage.

- `workflow_list` and `workflow_save` add `monitor_memories` plus
  `monitor_memory_error` alongside task and monitor-task context.
- Memory files are read from `.puffer/runtime/monitors/*.md`, sorted by path,
  and returned with `connection_slug`, absolute/display path, content, and a
  `truncated` flag.
- Snapshot memory content is capped to protect the daemon and desktop UI from
  very large monitor notes while preserving the on-disk file unchanged.
- Missing monitor memory directories are treated as an empty memory list.

This is read-only API surface. Ignore behavior still writes to the monitor
memory file, and monitor triage prompts continue to read that file before task
creation.
