# Contact Infer Proposal Review

## Summary

Contact inference returns and persists reviewable proposals. It no longer
promotes inferred `CreateContact` outputs into saved contacts automatically.

## Design

- `contacts_infer` still collects ranked connector candidates and asks the
  inference model for `ContactProposal` values.
- Successful inference stores the sanitized proposal list in
  `.puffer/runtime/contacts.json` through the existing proposal cache.
- Stored proposals are filtered against saved contact identities, so inference
  does not suggest contact IDs that already belong to a saved contact.
- Saved contacts are only changed by `contacts_save`; inference never inserts,
  replaces, or deletes saved contact rows.

## Compatibility

The RPC response remains a contacts snapshot with `contacts`, `candidates`, and
`proposals`. The previous `savedCount` response field is not emitted because
inference is a review step, not a save operation.
