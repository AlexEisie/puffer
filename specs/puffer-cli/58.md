# Subscriber Autostart Without Generic Connector Branches

- CLI subscriber autostart now resolves manifest-backed connections through
  `connection_subscriber_manifest` instead of hard-coded connector branches.
- Auth monitoring starts any configured manifest-backed connection subscriber
  before probing auth, then delegates the connector-specific probe to the host
  auth checker.
- Telegram subscriber action and auth-probe helpers live in
  `telegram_connector_actions`, keeping Telegram-specific command details out
  of the generic subscription runtime wiring.
- Existing Slack and Lark auth checkers keep their credential-file behavior
  under the updated `ConnectionAuthChecker` signature.
