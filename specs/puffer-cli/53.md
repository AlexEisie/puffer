# Subscriber Autostart Routing Fixes

## Scope

Hardens subscriber autostart and direct connector actions.

## Behavior

- Autostart considers only enabled workflow bindings and enabled agent proxy
  bindings.
- When a binding stores a connector slug whose bundled manifest exists, the
  static connector topic is started instead of the arbitrary connection slug.
- Telegram connection slugs still start per-connection Telegram subscribers.
- `slack-bot` no longer routes through Slack local credential action helpers.
- Lark video attachments preserve `LarkMediaKind::Video` through upload and
  send with Lark's `media` message type.

## Compatibility

Paused workflows do not start subscriber processes on launch. Email workflows
with custom connection slugs can still use the bundled `email` subscriber.
