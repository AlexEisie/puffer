# Gmail Browser Connector

- Adds the built-in `gmail-browser` connector template with a reusable subscriber manifest instantiated per connection under `gmail-browser-accounts/<connection>`.
- Adds a `puffer __subscriber gmail-browser` driver that opens Gmail through the daemon browser, reuses a connection-scoped browser session, polls selected Gmail inboxes every 30 seconds, and emits newly observed inbox rows as subscription message events.
- Stores Gmail browser connector auth/config in the subscriber state `config.toml`: workspace root, selected Chrome profile, and selected Gmail account emails. The connection record is created or refreshed when settings save a configured account list.
- Extends browser profile discovery to expose Google accounts from each Chrome profile's `Preferences` file so UI auth setup can be profile/account selection instead of a token flow.
- Extends `browser_agent` with an optional `chromeProfile` root-session override. The daemon browser registry now keeps root-session profile overrides alongside the existing root-session browser map, so connector sessions can share browser infrastructure without changing the user's global Browser tab profile.
