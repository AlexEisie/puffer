# puffer-cli Update 01

## Summary

Document the tmux home-screen snapshot baseline update for the native CLI/TUI
entrypoint tests.

## Design And Architecture

The `puffer-cli` crate owns tmux-facing integration tests for the interactive
entrypoint. The snapshots under `crates/puffer-cli/tests/snapshots/` are the
golden contract for no-alt-screen home rendering across wide, medium, compact,
and narrow terminal sizes.

The current home statusline includes the active sandbox state. The baseline
snapshots must therefore include that suffix so golden tests describe the actual
UI contract.

## Architecture Details

This is a test-contract update only. It touches the CLI crate's tmux snapshot
fixtures and does not change the runtime boundary between `puffer-cli`,
`puffer-tui`, or `puffer-core`.

The affected snapshots are width-specific render fixtures. Each fixture captures
the complete visible home screen for one terminal geometry, so the statusline
suffix must match the truncation behavior for that geometry.

## Logic

The updated snapshot expectation is:

- Wide, medium, and compact-wide home screens show
  `sandbox workspace-write`.
- Narrow home screen shows the truncated sandbox suffix that fits the viewport.
- Help and post-send snapshots are unchanged.

No CLI command behavior, daemon behavior, auth behavior, session behavior, or
TUI rendering code changes are part of this update.

## Contracts

- Snapshot tests remain the regression gate for tmux home rendering.
- The statusline contract includes model, remaining context, cwd, and sandbox
  state when terminal width allows it.
- Golden-mode rendering must stay deterministic enough for
  `cargo test -p puffer-cli --test tmux_snapshots`.
- Future changes that alter home-screen status content must update these
  snapshots and add a new numbered spec in this folder.

## Contract Details

- Wide, medium, and compact-wide snapshots include the full
  `sandbox workspace-write` suffix.
- Narrow snapshots may include the shortened suffix dictated by viewport width.
- Snapshot refreshes must be intentional and paired with focused tmux snapshot
  test verification.
- The update must not change public CLI command behavior, daemon behavior, auth
  behavior, or session storage.

## Verification
