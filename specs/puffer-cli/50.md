# Slack internal CLI and connector actions

## Summary

The CLI now exposes a hidden `puffer internal-tool slack` surface through the
`slack` Bash helper and executes Slack connector actions directly through
stored connection credentials.

## Design

- `slack configure-app` stores Slack app credentials under a `slack-app`
  connection by default.
- `slack login-token`, `slack login-browser`, and `slack import-local` store
  Slack account credentials under a `slack-login` connection by default.
- `slack list/search/read` commands call back into the parent runtime so agents
  can resolve Slack users, conversations, and messages before taking actions.
- `ConnectorAct` for `slack-app`, `slack-login`, and legacy `slack-bot` sends
  text messages, uploads local files with captions, and adds/removes reactions
  via the Slack Web API.
- Active Slack connections use a process-provided auth checker so the
  subscription monitor validates stored credentials instead of degrading them
  for lacking a subprocess `auth-ok` command.

## Compatibility

Existing `puffer serve` Slack bot behavior is unchanged. Slack action routing
now works for workflow actions without requiring a subscriber process.
