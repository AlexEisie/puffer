# Canvas State Session Cwd

## Summary

The daemon `canvas_state_update` RPC now resolves the target session cwd before
patching Canvas state. Inline Desktop Canvas updates therefore read and write
the same `.puffer/canvas` directory that the agent used when it rendered the
Canvas.

## Behavior

- `canvas_state_update` still requires `sessionId`, `canvasId`, and `patch`.
- The handler loads the session record through `SessionStore`.
- State patches are applied under the session cwd instead of the daemon process
  cwd.

## Compatibility

Existing Canvas state file format and `CanvasState` behavior are unchanged.
This only fixes Desktop daemon routing when the daemon process cwd differs from
the active workspace.
