# Legacy Inferred Contact Repair

## Summary

The daemon repairs contact stores written by the brief auto-save inference path.
Those stores promoted inferred `ContactProposal` rows into saved contacts, so
they showed up as user-curated contacts after restart.

## Design

- `contacts.json` remains the canonical saved-contact store.
- `inferred_contacts.json` remains the inferred proposal store.
- On load, a legacy `version: 0` contact store with saved rows, no embedded
  proposals, and an empty inferred store is checked for the old auto-save shape.
- Rows are repaired only when every saved row has a generated UUID id, a
  non-empty name, a non-empty description, and at least one routable contact id.
- Matching rows are converted back to `ContactProposal` entries, written to
  `inferred_contacts.json`, and removed from `contacts.json`.
- Before clearing matching rows, the original file is preserved once as
  `contacts.auto-saved-inferred-v0.bak.json`.

## Compatibility

The repair is intentionally narrow. Existing workspaces that already have
inferred proposals keep their saved-contact rows unchanged, and current
`version: 1` saved-contact stores are not rewritten by this migration.
