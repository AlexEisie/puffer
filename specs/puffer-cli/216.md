# Browser Test Module Split

## Summary

The daemon browser unit tests are split so each Rust test source file stays
below the repository line-limit contract.

## Design

- Expression, parser, and ref-resolution focused browser tests move into
  `daemon_browser/tests/expression_tests.rs`.
- `daemon_browser/tests.rs` keeps the session, tab, CEF, and shutdown tests and
  registers the expression test module.
- Test logic and assertions are unchanged; only module placement and imports
  changed.

## Compatibility

This is test-only structure. Browser daemon runtime behavior, RPC contracts, and
test names remain stable aside from the Rust module path prefix.
