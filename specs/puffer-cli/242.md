# Canvas State Daemon Bridge

## Design

The daemon exposes a `canvas_state_update` RPC for generated Canvas pages. The
Canvas HTML uses the existing authenticated WebSocket bridge and sends
`{ sessionId, canvasId, patch }` whenever an interactive control changes.

## Logic

The daemon handler validates required fields, delegates path/session validation
and patch application to `puffer-core`, and returns `{ ok, canvasId,
updatedAtMs }`. The handler is intentionally narrow: it does not start turns,
read arbitrary files, or accept caller-provided paths.

## Compatibility

The bridge is additive. Existing `run_agent_turn` Canvas action callbacks keep
working; interactive controls now persist state before a later `CanvasState`
read or action-triggered continuation.
