# 55 - Media Kind Alias Parsing

Subscriber-backed connector actions accept common media kind aliases without
rejecting otherwise valid attachments.

- `audio`, `voice`, `video`, and `media` are parsed as document-style Telegram
  uploads.
- Existing `auto`, `photo`, `image`, `document`, `doc`, and `file` behavior is
  unchanged.
- The parser still rejects unknown kind strings so malformed workflow payloads
  fail clearly.

This keeps connector actions compatible with the broader action schema while
preserving the current Telegram outbound transport model.
