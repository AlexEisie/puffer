# Generated Media Poster Preview RPC

Date: 2026-06-09

## Scope

The daemon keeps `read_generated_media_preview` as the single generated media
preview RPC for both image artifacts and generated video poster thumbnails.

## Behavior

- The RPC name and request shape stay `read_generated_media_preview` with
  `sessionId` and `artifactId`.
- Image artifacts return their existing generated image preview bytes.
- Video artifacts return poster bytes when artifact sidecar metadata records an
  available poster under `.puffer/media/artifacts/<artifact-id>/poster.jpg`.
- Missing sidecars, absent poster metadata, failed poster metadata, and missing
  poster files return `state = "missing"`.
- Wrong preview kind, unsafe paths, symlink escapes, bad declared MIME types,
  or bytes that do not sniff as the declared image MIME type return
  `state = "unsupported"`.

## Compatibility

Generated video playback is unchanged. Clients still use
`create_generated_video_access` for `/media/generated-video/<ticket>` playback
URLs instead of using the preview RPC as a video source.
