# Monitor filter rules in workflow snapshots

- Workflow list snapshots now include each binding's `ignore_filters` array alongside the existing trigger `filter_pattern`.
- The field is a direct JSON serialization of daemon-owned `FilterSpec` values, preserving tagged regex/jq filters and structured payload subset filters.
- This is read-only snapshot context for desktop inspection. Filter installation and synchronization remain daemon-owned through monitor task ignore handling.
- Existing monitor ignore synchronization still runs before snapshot assembly, so the returned rules reflect filters derived from ignored monitor tasks when sync succeeds.
