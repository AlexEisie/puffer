# Corbina planned binding connector inference

The desktop workflow binding RPC now mirrors the terminal workflow append
behavior for planned connections.

## Contract

- `workflow_binding_create` may omit `connector_slug` when `connection_slug`
  matches exactly one trigger-capable connector's suggested connection slug.
- The RPC still accepts explicit `connector_slug` for custom planned connection
  names such as `team-mail`.
- Existing connection slugs continue to ignore inference and validate against
  the stored connection connector.
- The response remains the refreshed workflow snapshot shape used by Corbina:
  workflows, runs, connectors, connections, workflow bindings, and monitor task
  metadata.

This keeps Corbina's append-workflow creation path deterministic while avoiding
an extra prompt for built-in planned connection names.
