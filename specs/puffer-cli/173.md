# Native CEF DevTools Readiness

The daemon browser bridge now treats the desktop native CEF DevTools endpoint as
usable only after the browser websocket endpoint is responsive.

- Remote DevTools probing checks IPv4 loopback before IPv6 loopback because the
  desktop CEF listener is bound on `127.0.0.1`.
- Each DevTools HTTP probe uses short connect and total timeouts so a stuck
  accepted TCP connection does not consume the whole readiness window.
- Native CEF browser websocket discovery waits longer before failing, covering
  desktop startup and target prewarm latency.
- Existing Chrome fallback behavior remains unchanged when the desktop CEF
  remote debugging port is not configured.
