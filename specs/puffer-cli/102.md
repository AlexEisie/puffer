# Monitor Task Ignore RPC

The daemon now exposes a direct ignore path for monitor-created tasks.

- `task_monitor_ignore` and `monitor_task_ignore` accept `task_id`/`taskId`
  plus an optional `reason`.
- The handler updates `.puffer/runtime/claude_workflow/monitor_tasks.json`,
  marks the task metadata as ignored, records the ignore reason, and changes
  status to `completed`.
- The ignore reason is appended to the task's monitor memory file so future
  monitor triage can skip similar connector events.
- The RPC returns the same refreshed workflow/task snapshot as `workflow_list`.
