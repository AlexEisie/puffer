# Mock Provider Request Reads

## Scope

Hardens the tmux agent-loop, tmux overlay, and daemon model-discovery test
harnesses used by `puffer-cli`.

## Design

The in-process mock Anthropic server now reads the complete HTTP request before
writing an SSE response. It parses the request headers, honors `Content-Length`,
and keeps reading until the declared body has arrived.

The in-process mock OpenAI discovery server uses the same complete-request read
pattern before returning its `/v1/models` fixture.

The resume overlay tmux test now drives `/resume` directly and leaves popup
completion acceptance coverage to the login overlay test.

## Logic

The mocks still accept loopback TCP connections directly. For each connection:

- read chunks until `\r\n\r\n` reveals the header/body split
- parse `Content-Length` case-insensitively
- continue reading until the full body is present
- then record the request and write the configured SSE fixture

## Contracts

- Large tool-list requests must not be truncated by the test mock.
- The mocks must not close the socket while the client is still uploading a
  valid request body.
- Tmux tests remain local-only and use workspace-local provider overrides.
- Daemon discovery tests must return the local discovered model instead of
  falling back to static defaults because the fixture socket closed early.
- Resume overlay coverage must not fail because a separate completion
  acceptance step left the prompt at a partial `/resu` prefix.

## Compatibility

This changes only test infrastructure. It does not change CLI runtime behavior
or provider request construction.
