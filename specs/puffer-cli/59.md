# Telegram Account State Resolution

- Subscriber-backed Telegram connector actions now always instantiate the
  shared `telegram-user` manifest with the selected connection slug.
- The default `telegram-user` account uses
  `<puffer-home>/telegram-accounts/telegram-user` for subscriber state instead
  of the bundled resource manifest state directory.
- Alternate accounts continue to use their connection slug as the subscriber
  id, topic, and account state directory.
- This keeps peer lookup, auth checks, sends, and Telegram connector actions on
  the same per-account session imported by the Telegram skill.
- Telegram connection health checks now use the subscriber's side-effect-free
  `telegram_auth_ok` command instead of peer search, so startup probes do not
  consume message update backlog before subscription workflows can process it.
