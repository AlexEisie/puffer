# Monitor Create Daemon RPC

The daemon now exposes a direct monitor creation RPC for desktop task workflows.

- `task_monitor_create` and `monitor_create` accept `connection_slug` (or
  `connectionSlug`) for an existing event-capable connector connection.
- The handler validates the connection, creates `.puffer/runtime/monitors/<slug>.md`
  when needed, creates or resumes the `monitor-<slug>` triage-agent workflow
  binding, starts the subscriber if the connector uses a manifest-backed event
  source, refreshes connection consumer state, and returns the same snapshot as
  `workflow_list`.
- Monitor creation now fails before mutating workflow bindings when auth refresh
  reports the selected connection as degraded or otherwise not usable.
- Existing monitor bindings are resumed instead of overwritten so user-edited
  monitor prompts or filters are preserved.
- Setup failure rolls back the workflow binding mutation while leaving monitor
  memory intact, matching the command-driven monitor behavior.
