# Gmail browser inbox polling fix

- Gmail browser subscribers now open selected accounts with Gmail's `authuser` selector instead of embedding the email address in `/mail/u/<account>/`, avoiding Gmail temporary-error pages for multi-account profiles.
- Gmail temporary-error pages are surfaced as connector `poll_error` control events and do not initialize or advance the seen cache.
- Gmail polls now wait for visible inbox rows or an explicit empty-inbox state before treating a page read as successful, so repeated polling does not keep reloading Gmail and baselining an in-progress load as an empty inbox.
- When a Gmail seen cache is empty, the first healthy inbox snapshot only emits the top visible row and baselines the rest, preventing a large backlog flood while still allowing a just-arrived message to create a task even if Gmail already marked it read.
