# Windows daemon stack guard

## Summary

`puffer-cli` now runs its existing CLI entrypoint on a named worker thread with
an explicit 32 MiB stack.

## Design

- The process `main` remains a small synchronous wrapper.
- The previous CLI startup body moved into `run_main`.
- Panics from the worker thread are resumed on the outer thread so panic
  behavior remains unchanged.
- Spawn failures are reported with context through the existing `anyhow`
  result path.

## Compatibility

This preserves all command parsing, daemon startup, provider auth, and exit
behavior. It addresses Windows builds where constructing the large Clap command
surface can overflow the default executable main stack before daemon handshake
output is printed.
