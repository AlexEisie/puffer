# Lark Thread Lookup CLI

## Scope

Extends `puffer internal-tool lark read-messages`.

## Behavior

- The command now accepts `--thread-id` as an alternative to `--chat-id`.
- Clap enforces that one, and only one, container id is supplied.
- The parent-runtime JSON forwards `thread_id` to the internal `Lark` tool.

## Compatibility

Existing `--chat-id` calls keep working. Agents can now inspect Lark thread
reply context before choosing message ids for replies, reactions, or lookups.
