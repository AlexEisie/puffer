# Lark Internal CLI

## Scope

Adds `puffer internal-tool lark` plus direct `ConnectorAct` execution for
credential-backed `lark-app` and `lark-login` connections.

## CLI Surface

- `configure-app --app-id --app-secret[-stdin] --brand <lark|feishu>`
- `login-token --user-access-token[-stdin] [--app-id] --brand <lark|feishu>`
- `import-env`, reading `LARK_APP_ID` + `LARK_APP_SECRET` or
  `LARK_USER_ACCESS_TOKEN`, and honoring `LARK_BRAND` unless overridden.
- Lookup commands: `list-chats`, `search-chats`, `search-users`,
  `read-messages`, `search-messages`, and `mget-messages`.

## Action Routing

`lark-app` and `lark-login` connector actions are handled in process:

- `send_message` accepts `oc_...` chat ids, `ou_...` open ids, or explicit
  `receive_id_type`.
- `send_message` supports raw Lark `content`, plain text, replies, local
  images/files, existing `img_...` keys, and existing `file_...` keys.
- Captions are sent as text messages before media because Lark IM media content
  carries media keys, not captions.
- `react` and `send_reaction` add a Lark message reaction by `emoji_type`.
- `remove_reaction` deletes a reaction by `reaction_id`, matching Lark's API
  contract.

## Auth Checks

The subscription runtime now installs a combined built-in auth checker for
Slack and Lark. The Lark checker only handles `lark-app` and `lark-login` and
returns `None` for unrelated connector templates.

## Compatibility

The CLI follows the existing Slack internal-tool pattern: Bash aliases call the
hidden CLI, the parent runtime executes the internal workflow tool, and
successful auth creates or updates a connection record.
