# Telegram connector action fallback

- `ManagerConnectorActionExecutor` routes Telegram personal-account connector
  actions other than `send_message` through the `telegram-user` subscriber with
  `Custom { op: "telegram_act" }`.
- The CLI fallback preserves the original connector action input JSON so the
  subscriber owns Telegram-specific validation and peer/message semantics.
- `send_message` remains a typed `SendMessage` command because it is the common
  cross-subscriber outbound path.
- The fallback is limited to `telegram`, `telegram-user`, and `telegram-login`
  connector slugs so Slack, email, and bot connectors do not inherit
  personal-account MTProto actions accidentally.
