# Workflow Runner Provider Fallback

Monitor and workflow agent runs now avoid unauthenticated default providers.

- Background workflow state preserves an explicit model when the workflow pins
  one, including scoped `provider/model` selectors.
- When no model is pinned, the runner checks whether the selected provider has
  stored auth. If not, it switches to the first authenticated registered
  provider, preferring OpenAI before Anthropic.
- The fallback clears unpinned default-model state so runtime model resolution
  uses the fallback provider's configured default model instead of reselecting
  an unauthenticated provider.
- This only affects detached workflow execution; interactive model/provider
  preferences remain unchanged.
