# Connector media send and poll vote fallback

- The host-enforced connector action fallback for `send_message` now accepts
  optional media alongside text.
- Media input can be a string path/URL, object, or array under `media`,
  `attachments`, `files`, `file`, or `path`.
- Attachment objects accept `path`/`file`/`url`/`source`, `caption`,
  `kind`/`type`/`media_type`, `mime_type`/`mime`, and `thumbnail`/`thumb`.
- `message`, `text`, or `caption` may provide outbound text. Text is optional
  when at least one media attachment is present.
- The fallback also handles Telegram-specific connector actions by enqueueing
  `SubscriberCommand::Custom { op: "telegram_act" }` with the action slug and
  original action input.
- Both fallbacks enqueue subscriber commands; they do not perform Telegram
  network I/O directly in the CLI process.
