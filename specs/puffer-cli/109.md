# Safe managed browser profiles

- Chrome profile selection remains metadata-driven: Puffer discovers local Chrome profiles and Gmail accounts from Chrome metadata so users can choose the intended profile/account in settings and connector auth flows.
- Managed browser sessions no longer clone real Chrome user-data directories, `Local State`, cookies, token databases, preferences, local storage, or other auth/session material from desktop Chrome.
- On first launch after this change, existing managed browser profile roots without the v2 marker are reset to purge legacy cloned auth state. Subsequent launches preserve the Puffer-owned profile directory so users can sign into browser connectors inside that isolated profile.
- The selected Chrome profile is used only to choose the Puffer-owned `--profile-directory` slot. Gmail and other browser connectors must authenticate within the managed Puffer browser profile instead of replaying desktop Chrome cookies.
- This prevents Google from seeing copied desktop Chrome session cookies in a separate headless user-data-dir and invalidating the user's normal Chrome Gmail sessions.
