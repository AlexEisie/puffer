# Telegram multi-account CLI routing

- The internal `telegram` CLI accepts `--connection`/`--connection-slug` and
  `--account`/`--account-slug` as global options. The default remains
  `telegram-user`.
- `run_telegram` forwards the chosen account connection slug into the internal
  tool payload for import, peer lookup, message search, QR login, phone login,
  code submission, and 2FA password submission.
- `ManagerConnectorActionExecutor` uses the resolved connection slug when
  routing Telegram `send_message` and Telegram-specific connector actions to a
  subscriber. A direct `telegram-login` slug without a connection still maps to
  the compatibility `telegram-user` subscriber.
- Telegram connection slugs are validated as non-empty kebab-case ASCII before
  spawning a synthetic subscriber.
- `--account-index` remains an import-time selector for a local Telegram
  Desktop/native storage slot and is not treated as the durable account
  identity.
- Subscription autostart detects enabled workflows bound to `telegram-login`
  connections and starts a derived Telegram subscriber when no static manifest
  exists for that connection slug.
- Derived Telegram subscribers reuse the bundled `telegram-user` run command
  while overriding manifest id, topic, display name, and state directory.
