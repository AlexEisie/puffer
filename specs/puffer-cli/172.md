# Browser Network Idle Path Hardening

Browser network-idle filtering no longer treats broad first-party analytics or
metrics paths as automatically ignorable.

- Known telemetry hosts remain ignored for explicit `waitNetworkIdle`.
- CDP non-blocking resource types such as ping, preflight, beacon, reporting,
  eventsource, and websocket remain ignored.
- First-party paths such as `/api/analytics/report` and `/internal/metrics`
  stay meaningful unless another signal marks the request as telemetry.
- Narrow first-party telemetry-style paths such as `/rum`, `/beacon`,
  `/session-replay`, `/telemetry`, `/traces`, and `/vitals` remain ignored.
- Ref-based click, double-click, and hover actions now resolve a scrolled,
  viewport-clamped target point and dispatch CDP mouse input instead of relying
  on synthetic DOM click events.
