# Repo Contract Cleanup

## Summary

The CLI crate now satisfies the repository contract for Rust source file length
while preserving the existing browser, Telegram contacts, and subscription
behavior.

## Design

- Telegram peer-cache hydration and candidate seeding moved from
  `daemon_contacts_telegram.rs` into a sibling helper module.
- The browser session test constructor moved into the existing browser
  `test_support` module.
- The one-use subscriptions store path helper was inlined at its call site.
- No runtime contracts or persisted data shapes changed as part of this cleanup.

## Compatibility

The extracted helpers keep the same module visibility and call sites. Existing
daemon browser, contacts, and subscription behavior remains unchanged.
