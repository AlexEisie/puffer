# 57 - Lark Import Env Default Connection

`lark import-env` now chooses its default connection slug from non-empty app
credential environment variables.

- `lark-app` is used only when both `LARK_APP_ID` and `LARK_APP_SECRET` are
  present and non-empty.
- Otherwise the command defaults to `lark-login`, matching the
  `LARK_USER_ACCESS_TOKEN` import path.

This prevents an empty app-secret environment variable from creating a user
token login under the misleading default slug `lark-app`.
