# Subscriber Action Completion Semantics

## Summary

Subscriber-backed outbound actions now wait for terminal subscriber events
instead of reporting success after enqueueing a stdin command.

## Design

- `ManagerOutbound` waits for `send_complete`, `send_error`, or
  `send_unsupported` before returning to workflow dispatch.
- Built-in Telegram `send_message` and Telegram-specific connector actions wait
  for `send_*` or `telegram_act_*` events before returning.
- Error terminal events become action errors, allowing workflow runs to avoid
  acking connector cursors after failed side effects.
- The auth monitor refreshes consumers before auth checks and starts Telegram
  personal-account subscribers for active Telegram connections so the manager's
  built-in auth probe has a live control channel.
- The `telegram search-messages --succint/--succinct` help text now describes
  the actual plain-text output.

## Compatibility

Successful sends still return one-line summaries for workflow history. Failed
subscriber sends now fail synchronously from the workflow/runtime point of view
instead of being reported later as detached subscriber events.
