# Monitor Memory Save RPC

The daemon now exposes a focused save path for monitor memory files.

- `monitor_memory_save` and `task_monitor_memory_save` accept a
  `connection_slug`/`connectionSlug` plus full memory `content`.
- The handler writes only to `.puffer/runtime/monitors/<connection_slug>.md`
  and rejects empty, traversal, separator, and NUL-containing slugs.
- Parent directories are created on demand.
- The RPC returns the same refreshed workflow/task snapshot as
  `workflow_list`, including the updated `monitor_memories` entry.

This keeps monitor-memory editing available to desktop clients without adding
an arbitrary filesystem write API.
