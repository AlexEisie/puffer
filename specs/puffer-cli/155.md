# Stale Turn Recovery RPC

`puffer daemon` now exposes `recover_stale_turn` for desktop clients that reload a
session whose persisted transcript ends in an unanswered user message.

The RPC validates that no in-memory turn is active for the session, checks the
session age against `retryAfterMs`, and inspects the append-only transcript. When
the latest meaningful event is an old user message with no prior recovery marker
for that prompt hash, the daemon appends a system recovery marker and starts a
normal `run_agent_turn` with the original message text. If the same prompt was
already retried automatically, the daemon appends a terminal manual-follow-up
marker instead of retrying again.

This keeps retry policy durable across daemon restarts without adding opaque
turn-state storage. The marker prevents retry loops after repeated crashes while
preserving the original transcript and the retried prompt as normal session
events.
