# Monitor Batch Prompting

Monitor triage batch execution now renders one prompt for the full trigger
batch instead of splitting a batch into one agent turn per trigger. Each trigger
is enriched with available source context before rendering, and source text is
stamped back onto any tasks for every trigger after the shared turn completes.

The monitor creation prompt now documents burst handling: when a short
same-conversation burst forms one request, the triage agent should produce one
concise task that captures the combined intent rather than focusing only on the
final message. If a batch contains unrelated actionable requests, the agent may
still create separate tasks. Source fidelity remains tied to the relevant
trigger text, and combined tasks use the latest relevant trigger's `receivedAt`
when several messages form one request.
