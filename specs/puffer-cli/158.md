# puffer-cli Update 158: Staged Attachment Turn Persistence

## Summary

Desktop daemon turns persist staged chat attachments by ID and return attachment
state in session timeline DTOs.

## Contract

- `run_agent_turn` accepts `attachmentIds`/`attachment_ids` as the durable
  attachment input.
- The daemon does not persist client-supplied attachment metadata from the old
  `attachments` field.
- Staged IDs are loaded from `puffer-session-store` before provider execution
  and before appending the user message.
- If any staged ID is invalid or missing metadata, the turn fails without
  appending a user message.
- The turn cancellation fallback that writes an unpersisted user prompt must
  include the same loaded attachments.
- Slash-command and sessionless turn paths persist empty attachment lists.
- `load_session_detail` maps stored attachments onto user timeline items with
  computed `available` or `missing` state.

## Compatibility

Old transcripts continue loading because `UserMessage.attachments` defaults to
an empty list. Placeholder text remains ordinary message text and is not parsed
into cards.

## Verification

CLI tests should cover timeline DTO attachment state and daemon
`run_agent_turn` persistence with a staged attachment ID.
