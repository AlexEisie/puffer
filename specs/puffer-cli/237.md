# Monitor digest next-action task wording

Monitor digest triage prompts now require task wording to point at concrete
next actions.

- Hourly digest guidance tells the triage agent to treat casual chatter and
  acknowledgements as context unless they change the next action.
- Task subjects must be imperative next actions rather than neutral topic
  summaries.
- Task descriptions must explain why action is needed, cite the source
  request/context, and state the concrete next step.
- `TaskCreate` tool metadata reinforces the same subject and description
  expectations for monitor digest tasks.

This keeps digest-created tasks useful in `/tasks` even when the digest includes
low-signal context messages.
