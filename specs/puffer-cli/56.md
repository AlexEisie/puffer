# 56 - Lark Auto Media Message Types

Lark direct connector actions now infer audio and video message types for local
file uploads when the media kind is `auto`.

- Opus audio paths such as `.ogg` and `.opus` upload and send as Lark `audio`
  messages.
- Video-looking paths such as `.mp4`, `.mov`, `.avi`, `.mkv`, and `.webm`
  upload and send as Lark `media` messages.
- Non-image, non-audio, non-video paths continue to send as generic Lark
  `file` messages.

Explicit media kinds still take precedence over extension inference, but
explicit `audio` on a non-Opus local file fails clearly instead of making a
Lark API call that is known to be rejected.
