# Desktop Media RPCs

## Summary

The local CLI daemon now implements the desktop media generation RPC surface
used by the composer generation settings UI.

## Contract

- `list_media_capabilities` returns deterministic media capability descriptors
  filtered by optional `kind`.
- OpenAI image generation is advertised as `openai/gpt-image-1` when OpenAI or
  Codex auth is present, when `OPENAI_API_KEY` or `PUFFER_OPENAI_API_KEY` is
  set to a non-empty value, or when native Codex auth exists at
  `HOME/.codex/auth.json`.
- `generate_media` validates `kind` and `prompt`, then returns the same queued
  image job contract as the desktop preview backend. Video generation remains
  unavailable until a video capability adapter exists.
- Settings snapshots include `config.media` with image and video defaults.
- `update_config` accepts a `media` patch and `media: null` resets media
  defaults while unknown keys continue to fail fast.

## Compatibility

This removes the real desktop app's `unknown method` failure for media settings
against the CLI local daemon. The fake daemon and Tauri preview backend already
implemented this surface; the CLI daemon now matches that contract.
