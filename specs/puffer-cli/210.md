# Desktop Timeline Generated Video Attachments

## Scope

The CLI desktop API timeline now synthesizes generated-media attachments from
successful `VideoGeneration` tool results.

## Change

- Timeline generation uses a generated-media helper that recognizes only
  `ImageGeneration` and `VideoGeneration`.
- Existing image attachment behavior is unchanged, including
  `generated-image:<artifact_id>` ids, image names, MIME handling, and sidecar
  fallback behavior.
- `VideoGeneration` artifacts with `video/*` MIME types become metadata-first
  attachments with `generated-video:<artifact_id>` ids, `Generated video`
  names, `kind = "video"`, and `GeneratedMedia` sources.
- Video extensions are derived from MIME type: `video/mp4` maps to `MP4`,
  `video/webm` maps to `WEBM`, and other video MIME types map to `VIDEO`.
- Video attachment state is based on the output path's existence; transcript
  synthesis does not read video bytes into memory.

## Compatibility

No legacy video tool names, aliases, or compatibility shims are recognized.
The current image attachment contract remains the image contract.
