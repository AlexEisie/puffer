# Desktop Activity Turn Completion

## Design

Desktop session activity derives from the append-only transcript after applying
rewrites. A `TurnBoundary` with `state: "finished"` is now treated as a terminal
marker for activity scans.

## Logic

When scanning backward for pending permission requests or unanswered user
actions, a finished turn stops the scan and returns idle for that activity
category. A started turn remains transparent so a started turn followed by a
user message still reports `running` until a response, permission request,
rewrite, or finished boundary changes the state.

Dirty git diff snapshots continue to report `running` independently, preserving
the existing desktop signal that a session has outstanding file changes.

## Compatibility

Older transcripts without turn boundary events keep the previous behavior:
an unanswered user message is `running`, a pending permission request is
`awaiting`, and an assistant response makes the session idle.

Transcripts that contain a finished turn without an assistant message, such as a
cancelled or otherwise closed daemon turn, no longer appear stuck in the desktop
session list.
