# Workflow Snapshot Monitor Tasks

## Scope

The desktop `workflow_list` RPC now includes workspace-level connector monitor
tasks alongside workflows, runs, connectors, and connections.

## Behavior

- `workflow_list` reads `.puffer/runtime/claude_workflow/monitor_tasks.json`.
- Missing monitor task storage is reported as an empty `monitor_tasks` array.
- Invalid or unreadable monitor task storage returns an empty array plus
  `monitor_task_error`.
- Each monitor task row includes task id, subject, description, status, monitor
  connection, monitor connector, monitor memory path, ignored state, action
  labels/prompts, possible ignore reasons, and timestamps.

## Compatibility

The RPC remains backward compatible. Existing clients can ignore the optional
`monitor_tasks` and `monitor_task_error` fields, and no workflow storage schema
or connector runtime contract changes.
