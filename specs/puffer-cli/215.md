# Exact Image Media Daemon Runtime

The CLI daemon media RPCs now use the shared exact media runtime.

- `list_media_capabilities` resolves descriptor-backed capabilities from the
  provider registry and auth store instead of hard-coded OpenAI checks.
- Environment-only OpenAI keys and native Codex auth files are not treated as
  media capabilities unless credentials are imported into the auth store.
- `generate_media` requires a saved concrete image provider/model and executes
  the descriptor-selected adapter synchronously.
- Successful image generation persists media job and artifact sidecars and
  returns a succeeded job with artifact id and local path.
- Stale provider/model and unsupported parameter selections fail before any
  provider HTTP request.

Video generation remains unavailable until an implemented video capability
adapter is declared and resolved.
