# Monitor digest batch prompt

Workflow monitor triage batches now run as one task-agent turn instead of one
turn per trigger.

- `ProcessWorkflowRunner::triage_agent_batch` enriches all triggers, renders a
  single digest prompt, and calls one task-agent session for the batch.
- Multi-trigger prompts are labelled as hourly monitor digest triggers and
  explicitly tell the agent to infer conversation-level intent and avoid one
  task per short message.
- Single-trigger rendering remains labelled as the existing workflow trigger.
- After the batch turn, source-text grounding is attempted for every trigger so
  tasks that copy the correct `monitor_envelope_id` still receive verbatim
  source metadata.

This pairs with the subscription manager's hourly monitor digest queue so
monitor tasks are created from conversation windows instead of individual chat
messages.
