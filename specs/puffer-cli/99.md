# Daemon Slash Command Title Gating

## Design

Daemon turn title generation remains available for regular user prompts and
prompt-backed slash commands. Local and UI slash commands now bypass the
model-backed title path before command dispatch.

## Architecture

`daemon_title::should_generate_title_for_turn` centralizes the title eligibility
check. It combines the existing empty-title and no-prior-user-message rules with
slash command classification from the core command registry.

## Logic

- Regular user text can still create a generated title.
- Prompt commands such as `/commit` can still create a generated title.
- Local/UI commands such as `/connect` and `/model` do not call title models.
- Unknown slash commands do not call title models because command dispatch will
  handle them locally as command errors.

## Contracts

Connector setup remains deterministic: `/connect` reaches its local
`AskUserQuestion` flow without waiting on title model I/O. The daemon event
contract is unchanged, including `turn-start`, `user-question-request`,
`turn-complete`, and `turn-error`.

## Compatibility

This preserves auto-title behavior for normal chat sessions while removing an
unintended model dependency from deterministic slash command flows.
