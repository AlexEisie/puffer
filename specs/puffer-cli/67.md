# Workflow Snapshot Monitor Commands

## Summary

The desktop daemon workflow snapshot now carries deterministic monitor commands
for event-capable connector connections.

## Behavior

- Connection rows with `can_trigger_workflow=true` include `monitor_command`,
  formatted as `/monitor <connection_slug>`.
- Connection rows that cannot produce workflow trigger events expose
  `monitor_command=null`.
- The monitor command is computed by the daemon from the same trigger-readiness
  decision used for workflow connection triggers.

## Compatibility

The field is additive. Existing desktop clients can ignore it, while newer
clients can run monitor setup without re-deriving command strings from
connection slugs.
