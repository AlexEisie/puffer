# Chrome Profile Selection

- Puffer config now supports `[browser].chrome_profile`, exposed through daemon `update_config` as `browserChromeProfile`.
- The daemon discovers local Chrome profiles from Chrome `Local State`, defaults to the most recently used profile when no explicit profile is configured, and exposes profile choices in the settings snapshot.
- Managed browser roots are seeded from a Puffer-owned copy of the selected Chrome profile before launch, avoiding writes or locks against the user's live Chrome profile.
- Changing the selected profile shuts down existing managed browser roots so the next browser tab is launched from the new profile seed.
