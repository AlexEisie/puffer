# Manifest-Backed Workflow Triggers

- `WorkflowCreate` trigger support is now based on generic connector stream
  support or manifest discovery through connector subscriber metadata.
- The workflow path no longer special-cases any concrete connector slug when
  deciding whether a trigger is valid or when starting a subscriber.
- `SubscriptionCreate` uses the same generic connection manifest resolution,
  while command-backed connectors continue to be started by
  `refresh_connection_consumers`.
- Legacy direct subscriber manifests remain supported by connection slug or
  connector slug, so existing email-style subscribers keep their shared topic
  behavior.
