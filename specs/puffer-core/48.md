# Connector Workflow Lifecycle Fixes

## Summary

Workflow and connector tools now refresh runtime consumers after lifecycle
mutations and surface connector action failures as tool failures.

## Design

- `ConnectionDelete`, `ConnectorDelete`, `ConnectorRegister`,
  `ConnectorUpdate`, `SubscriptionDelete`, and `SubscriptionPause` refresh
  connection consumers immediately after their storage mutation.
- `ConnectorAct` returns a tool error when a connector protocol response or
  fallback dispatcher result reports failure.
- `WorkflowCreate` persists the workflow binding before starting the Telegram
  subscriber, avoiding orphaned subscriber side effects when the binding create
  fails.
- Connector template starter output examples now use `completed` rather than
  `queued` to match synchronous action completion semantics.

## Compatibility

Existing workflow and connector tool inputs are unchanged. Failed connector
actions now fail through the normal internal-tool error channel instead of
requiring callers to inspect a successful JSON envelope with `success: false`.
