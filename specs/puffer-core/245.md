# /night - autonomous overnight work

## Summary

`/night` is a `Prompt`-kind slash command that launches a guardrailed,
autonomous "while you're away" work routine: it surfaces candidate work via
AutoDream centered on the user's own tasks/interests, then drives **multiple
subagents** to implement bounded tasks in an **isolated git worktree**, with
end-to-end tests + screenshots, and an **experimental, default-off** option to
open PRs to the user's fork.

## How it plugs in

- Registered in `command/registry.rs` as `cmd("night", ..., CommandKind::Prompt)`.
- `command_helpers/prompt.rs::prepare_prompt_command_specialization` routes
  `"night"` to `prepare_night_prompt_command(state, session_store)`, which:
  1. pulls AutoDream leads via `autodream_suggestions_with_store(session_store)`,
  2. reads the experimental gate `state.config.night.submit_pr`,
  3. returns `PromptCommandPreparation::DirectPrompt(build_night_directive(...))`.
- The directive runs through the normal prompt-command path
  (`execute_user_prompt_with_tool_filter`), so it uses puffer's existing tools
  (Agent/Task subagents, the worktree tool, e2e/screenshot tools, PR creation).
  The orchestration logic lives in the directive + the config gate, not in a new
  bespoke engine.

## The directive (`build_night_directive`) encodes the guardrails

1. **Find work** - aggressively surface candidates centered on the user's
   accumulated tasks + their `soul.md`/`user.md` interests; pick 1-3 bounded
   tasks; prefer depth over random breadth.
2. **Isolation** - ALL work in a fresh `.worktree/` git worktree; never edit the
   main checkout, the user's current branch, or master.
3. **Non-destructive** - no `rm -rf`, `reset --hard`, force-push, or deleting the
   user's files/branches/shared state; abandon a task rather than do a
   destructive step. Do not break the system.
4. **No drift** - write each task's goal first, hold to it, stop when done/stuck.
5. **Per task** - spawn one or more subagents (parallel for independent tasks);
   add + run end-to-end tests; capture screenshots under
   `.worktree/<task>/artifacts/`; verify by running, not by claiming.
6. **Finish** - fork-PR is gated:
   - `night.submit_pr = false` (default): no PR; leave each task on its worktree
     branch and report it.
   - `night.submit_pr = true`: open a PR to the user's fork (never upstream/main)
     per finished, tested task.

## Config (`puffer-config`)

`NightConfig { submit_pr: bool }` (default false), exposed as `PufferConfig.night`
(`#[serde(default)]`, alias `submitPr`). Experimental - must be deliberately
enabled.

## Tests

- `command_helpers::prompt::tests::night_directive_carries_guardrails_and_gates_pr`:
  asserts the worktree/non-destructive/no-drift/subagent/e2e+screenshot guardrails
  are present, AutoDream leads are embedded (with an empty-leads fallback), and the
  fork-PR clause flips with `submit_pr`.

## Notes / follow-ups

- v1 enforces non-destructiveness via the directive + worktree isolation + the
  global system-prompt safety rules. A hard tool-deny filter for `/night` (deny
  force-push/reset --hard at the permission layer) is a possible follow-up.
- v1 seeds candidates from accumulated AutoDream suggestions; triggering a fresh
  AutoDream provider-review inside `/night` (needs providers/auth threaded into
  the prep) is a follow-up.
