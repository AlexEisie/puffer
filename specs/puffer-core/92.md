# Jira Webhook Connect Flow

## Design

`/connect jira-webhook <connection>` routes to the serve-mode webhook
configuration writer. The flow asks deterministic `AskUserQuestion` input
questions for bind address and URL path, defaults an empty path to `/jira`,
and writes the shared `[connectors.webhook]` table used by `puffer serve`.

## Contracts

- `/workflows connectors jira issue comment` discovers the preset through
  catalog search terms.
- The connector row reports `runtime=serve`, `connect=/connect jira-webhook
  jira-webhook`, and `[no-trigger]`.
- The setup summary instructs the user to run `puffer serve`.

## Compatibility

Jira uses the same webhook runtime as generic, GitHub, GitLab, and Linear
webhook presets. The shared writer preserves existing wrapped or bare connector
config shape.
