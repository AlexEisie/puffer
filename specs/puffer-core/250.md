# Secret Question Hardening

`AskUserQuestion` and `RequestSecret` now avoid model-visible raw secrets in
more cases.

- `AskUserQuestion` accepts `secret: true` only on input questions.
- Runtime answers for secret questions are registered in the session masked
  secret store and replaced with `PUFFER_SECRET_...` placeholders before the
  tool result is returned.
- Secret choice questions are rejected because options are inherently
  model-visible.
- `RequestSecret create` requires `value` to be exactly one Puffer secret
  placeholder; raw and compound values are rejected, and agents must use
  `RequestSecret collect` for new user-entered secrets.
- `RequestSecret collect` always includes a non-overridable save disclosure in
  the user prompt, even when the agent supplies custom prompt text.
- `Agent` tool calls remain on the serial execution path so
  `PUFFER_SECRET_...` placeholders are not expanded into raw secrets before a
  nested model turn.
