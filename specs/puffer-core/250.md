# puffer-core Update 250: Derive Image Artifact Format From Response Bytes

## Summary

The images_json adapter now derives a generated artifact's file extension and
MIME type from the actual response bytes (magic-number sniffing) instead of
assuming PNG whenever the selected model declares no `output_format` parameter.

## Runtime Contract

After the provider response is decoded (either `b64_json` or a downloaded
`url`), the runtime resolves the output format in this order:

1. The signature of the returned bytes (`png`, `jpeg`, or `webp`).
2. The declared `output_format` parameter value, when sniffing is inconclusive.
3. `png`, as the final fallback.

The resolved format drives the saved filename extension, the artifact
`mime_type`, and the `mimeType` recorded in artifact metadata so all three stay
consistent with the bytes on disk.

## Compatibility

BytePlus Seedream 4.5/4.0 omit `output_format` (see Update 248) and return JPEG
bytes. Previously such artifacts were written as `image.png` with
`image/png`, mislabeling JPEG content. They are now written as `image.jpeg`
with `image/jpeg`. Models that do declare and honor `output_format` are
unaffected, since the bytes match the requested format.
