# Workflow create rollback

## Summary

`WorkflowCreate` now rolls back a newly persisted workflow binding when
post-create setup fails.

## Design

- The binding is still written before setup so the selected subscriber can
  emit events against an existing workflow definition.
- If Telegram subscriber startup or consumer refresh fails, the tool deletes
  the just-created binding, refreshes consumers again, and returns the original
  setup error with rollback context.

## Compatibility

Successful `WorkflowCreate` output is unchanged. Failed setup no longer leaves
an enabled workflow that conflicts with retry.
