# Monitor Task Next-Step Wording

- The `/monitor` command prompt mirrors daemon monitor wording for generated
  task fields: subjects should answer what the user should do next, while
  descriptions explain context, importance, and the next step.
- Backfill monitor agents receive the same next-step field guidance so
  historical tasks do not regress to message-summary titles.
- The declarative `TaskCreate` metadata now prefers actionable task subjects and
  descriptions over neutral summaries for all durable workflow tasks.
