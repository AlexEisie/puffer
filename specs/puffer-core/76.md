# Workflow Connector Existing Connections

`/workflows connectors` now exposes existing connection names on connector
catalog rows.

Behavior:

- Connector rows include `connections=<slug,...>` when stored connections use
  that connector template.
- Connector search matches those stored connection slugs and descriptions, so a
  query such as `/workflows connectors team-email` returns the owning connector.
- The deterministic `connect=/connect <connector> <suggested-connection>`
  hint remains present even when existing custom connection names are listed.
- The default `/workflows` view benefits from the same connector row metadata
  for trigger-capable connectors.

Compatibility:

- The command output format remains plain text and additive.
- Existing connector slug, skill, description, action, and connect-command
  search terms keep their previous behavior.
