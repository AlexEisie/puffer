# Gmail Browser Connector Auth Flow

- `/connect gmail-browser <connection>` is now a deterministic connector setup flow.
- The flow discovers local Chrome profiles from Chrome `Local State` and profile `Preferences`, asks for the profile through `AskUserQuestion`, then asks for Gmail accounts with a multi-select question.
- Setup writes `${PUFFER_HOME}/gmail-browser-accounts/<connection>/config.toml` with `workspace_root`, the selected `chrome_profile`, and selected `accounts`.
- Setup upserts the connection record as `gmail-browser`, marks it authenticated, clears prior auth failure notification state, refreshes connector consumers, and re-runs connection auth refresh.
- `AskUserQuestion` now accepts larger multi-select choice lists so account pickers are not capped at four options. Searchable multi-select remains unsupported.
