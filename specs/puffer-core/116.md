# puffer-core update 116

`/telegram` is no longer part of the user-facing slash-command surface.

- The local command registry no longer advertises a `telegram` command, so TUI
  slash search and command completion cannot route users into connector-specific
  Telegram behavior.
- Local command dispatch no longer has a `/telegram` branch. Submitting
  `/telegram ...` now follows the normal unknown-command path instead of
  blocking the TUI while a Telegram subscriber request times out.
- Telegram connector setup remains available through deterministic connector
  commands such as `/connect telegram-login <connection>`, and workflows should
  continue to use generic workflow/connector commands.
- Internal Telegram runtime and subscriber APIs remain available to tools and
  skills through the typed internal-tool path; only the slash-command alias was
  removed.
