# Workflow Monitor Task Test Coverage

## Summary

Adds command-surface regression coverage for monitor task rows rendered by the
`/workflows tasks` dashboard view.

## Behavior Covered

- Monitor task output includes the filter hints users need to find active,
  ignored, actionable, and ignorable tasks.
- Active monitor tasks render their connection, connector, monitor memory path,
  subject, summarized actions, `/tasks show`, and `/tasks ignore` commands.
- Ignored tasks are searchable by `ignored` and use a stable fallback ignore
  command when no ignore reason is present.
- Empty and no-match states remain explicit.
- Existing monitor task command tests now use the required `receivedAt` and
  `expiresAt` fields so they continue to reflect the current monitor task
  contract.

## Compatibility

- This is test coverage only.
- No workflow storage, task metadata, connector, or slash-command contract
  changes are introduced.
