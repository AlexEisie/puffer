# GitLab Webhook Connect Flow

## Design

`/connect gitlab-webhook <connection>` routes to the serve-mode webhook
configuration writer. The flow asks deterministic `AskUserQuestion` input
questions for bind address and URL path, defaults an empty path to `/gitlab`,
and writes the shared `[connectors.webhook]` table used by `puffer serve`.

## Contracts

- `/workflows connectors gitlab merge request` can discover the preset through
  catalog search terms.
- The connector row reports `runtime=serve`, `connect=/connect gitlab-webhook
  gitlab-webhook`, and `[no-trigger]`.
- The setup summary instructs the user to run `puffer serve`.

## Compatibility

GitLab uses the same webhook runtime as generic, GitHub, and Linear webhook
presets. The shared writer preserves existing wrapped or bare connector config
shape.
