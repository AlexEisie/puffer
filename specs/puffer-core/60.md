# Deterministic Connect Command

`/connect <connector-slug> <connection-name>` now runs as a local deterministic
command instead of a provider-backed prompt command.

- The command is registered as `CommandKind::Local`, so connector auth does not
  require provider availability and cannot vary with model behavior.
- `execute_connect_flow` owns the connector/auth state machine. It asks missing
  connector and connection values with first-class `AskUserQuestion` input
  questions, asks connector-specific method choices with choice questions, and
  then calls the typed workflow helpers directly.
- Supported deterministic flows are `telegram-login`, `slack-app`,
  `slack-login`, `lark-app`, `lark-login`, and `email`. Unknown no-auth
  connectors fall back to `ConnectionCreate`; unknown auth-required connectors
  fail with an explicit unsupported-flow message.
- Telegram auth supports local Telegram Desktop import, QR login approval, and
  phone login with follow-up code and 2FA password questions.
- Secret-bearing helper calls are intentionally not returned as persisted
  `ToolInvocation` records. The local command returns only a concise connection
  summary.
- This supersedes the prompt-backed `/connect` draft in `59.md`; prompt-level
  resource scoping is no longer part of the `/connect` contract.
