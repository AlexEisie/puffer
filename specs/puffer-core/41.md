# puffer-core update 41

Adds runtime handling for Telegram local auth import.

Design and behavior:

- The internal `Telegram` workflow tool accepts `action=import_local`.
- `TelegramImportLocal` is also routed as a direct workflow tool handler for
  compatibility with explicit tool ids.
- The handler ensures the `telegram-user` subscriber is running, sends
  `SubscriberCommand::TelegramImportTdata`, waits for `login_complete` or
  `login_error`, and returns a normal complete/error payload.
- Permission prompts summarize the import path without exposing passcodes.

Compatibility:

- Existing login actions and events remain unchanged.
- Import success uses the existing `login_complete` event so downstream
  connection creation flows do not need a new auth-complete state.
