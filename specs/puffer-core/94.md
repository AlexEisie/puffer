# Asana Webhook Connect UX

## Summary

Adds `asana-webhook` to the deterministic `/connect` and connector search UX.
Users can search for Asana task/project terms in the TUI catalog and run:

```text
/connect asana-webhook asana-webhook
```

## Design

Serve-backed webhook presets now share one `connect_webhook_preset` dispatcher
instead of adding a new public helper for each provider. The helper maps a
connector slug to provider display text, default path, and summary method.

Asana uses:

- bind prompt: `What bind address should the Asana webhook listen on?`
- path prompt: `What URL path should Asana post webhook events to?`
- default path: `/asana`
- welcome message: `Asana webhook ready.`
- method: `Asana event webhook`

## Compatibility

Existing webhook preset behavior is preserved for GitHub, GitLab, Jira, Linear,
and Stripe. The shared helper reduces future line-limit pressure in
`connect.rs` while keeping setup prompts deterministic and compatible with the
AskUserQuestion flow.
