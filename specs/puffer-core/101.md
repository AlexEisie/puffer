# Datadog `/connect` Webhook Preset

## Change

`/connect datadog-webhook <connection-name>` now runs the deterministic
serve-mode webhook preset flow.

## Flow

- Resolve `datadog-webhook` from exact slug or searchable terms such as
  `datadog monitor`.
- Ask for the bind address.
- Ask for the URL path and default blank answers to `/datadog`.
- Write `[connectors.webhook]` in `.puffer/connectors.toml` with the selected
  display name, bind address, path, and `Datadog webhook ready.` welcome
  message.
- Return the same `puffer serve` next-step summary as the other serve-backed
  webhook presets.

## Compatibility

The preset remains setup-only. It does not claim workflow trigger support until
the serve-backed webhook runtime can publish connector trigger events directly.
The top-level `/connect` dispatch continues to use the shared webhook preset
registry instead of a repeated slug list.
