# Serve connector /connect setup

`/connect` now handles the serve-mode connector catalog entries directly instead
of falling through to the generic auth-required error.

- `discord-bot`, `matrix-bot`, and `webhook` dispatch to deterministic local
  setup flows from `execute_connect_flow`.
- The flows ask only for required user-provided values through
  `AskUserQuestion`: Discord bot token, Matrix homeserver/username/password,
  webhook bind address, and an optional webhook bearer-token choice.
- Setup writes `.puffer/connectors.toml` with `[connectors.<platform>]` for new
  or already wrapped files, and preserves bare top-level connector TOML shape
  for existing bare configs.
- Generated configs include `enabled = true`, `display_name = <connection>`,
  and safe defaults where relevant: `require_mention = true`,
  `group_key_policy = "per_user"`, and webhook `path = "/puffer"`.
- The command summary points the user at the written config path and the next
  operational step: run `puffer serve`.
