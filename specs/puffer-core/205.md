# Workflow Alias Cleanup

The slash command registry now exposes workflow terminology only for workflow
commands.

- `/workflows` remains the canonical command and `/workflow` remains the
  singular alias.
- `/pipeline` and `/pipelines` are no longer registered aliases.
- Workflow creation output points users to the Workflows screen instead of the
  removed Pipelines wording.
- The underlying native workflow schema still contains `pipeline` fields for
  compatibility with existing persisted workflow definitions.
