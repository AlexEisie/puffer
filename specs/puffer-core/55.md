# 55 - WorkflowCreate Subscriber Startup

`WorkflowCreate` now starts an installed subscriber manifest when an enabled
workflow is created for a legacy/static subscriber connection.

- Telegram login connections still use the per-account Telegram subscriber
  manifest path.
- Other connections try an installed subscriber manifest for the connection
  slug first, then the connector slug. This supports `work-email` connections
  backed by the static `email` subscriber topic.
- Missing manifests remain a soft no-op so custom connector definitions can be
  registered before their process is installed.
- Startup failure after persistence rolls back the workflow binding, matching
  the existing Telegram setup behavior.

This makes the "first enabled workflow turns on delivery" contract hold for
legacy subscriber-backed connectors, not only Telegram.
