# Searchable Connect Selection

`/connect` now treats an omitted connector slug as a catalog selection step
instead of a free-form slug input.

- `execute_connect_flow` calls `ConnectorList` when the connector argument is
  missing and asks `AskUserQuestion` with `searchable: true`.
- Connector options use the connector slug as the answer label and include the
  catalog description plus auth/subscribe/proxy traits in the option
  description.
- Direct `/connect <connector-slug> <connection-name>` remains deterministic
  and bypasses the picker.
- `AskUserQuestion` validation now allows searchable single-select choice
  questions with up to 50 options. Non-searchable choice questions keep the
  existing 2-4 option contract, and searchable multi-select or input questions
  are rejected.
