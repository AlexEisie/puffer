# Workflow list search counts

Terminal workflow listing now treats `list` queries like the other workflow
dashboard sections.

## Contract

- `/workflows list <query>` filters workflow definitions by workflow slug, name,
  trigger label, enabled/paused state, node ids, agents, prompts, models, and
  tools.
- Workflow definition lists show `showing N/T workflows`, including the query
  text when a filter is active.
- `/workflows runs <query>` shows `showing N/T runs` and can match run status,
  trigger payloads, trigger keys, node ids, node status, output, and errors.
- Empty filtered lists keep explicit `no matching` rows, while an empty store
  still reports `none registered` or `none recorded`.

This only changes terminal workflow dashboard rendering and search metadata. It
does not change workflow registration, connector setup, connector auth, run
storage, Corbina, TUI overlays, or webhook behavior.
