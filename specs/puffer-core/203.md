# Workflow Dashboard Text Layout

The workflow dashboard output is now organized around scan-first status rows
and explicit command follow-ups.

- `/workflows` prints stable view shortcuts for runs, tasks, connections, and
  connectors immediately after the dashboard counts.
- `/workflows` prints a deterministic next-step chain for connecting a source,
  creating a workflow, and enabling a monitor.
- Connection and connector rows keep identity, status, and descriptions on the
  primary row, with repair/create/append/monitor commands on a separate
  `commands:` row.
- Monitor task rows expose memory context in the primary row when present and
  move show/ignore commands to a separate `commands:` row.
- `/monitor` setup output points users back to workflow tasks, task detail, and
  connection views after creating or re-enabling monitors.
