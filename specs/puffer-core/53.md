# Workflow Connector Topics

## Scope

Updates `WorkflowCreate` to retain the connector template slug on each binding.

## Runtime Contract

New workflow bindings store both:

- `connection_slug`: the authorized account/connection selected by the user
- `connector_slug`: the template slug of that connection

The router already matches either event topic, so this lets legacy static
subscribers such as `email` emit their connector topic while custom connection
slugs such as `work-email` still own the workflow.

## Compatibility

Connection-scoped subscribers such as Telegram keep matching by
`connection_slug`. Static connector-topic subscribers gain compatibility with
custom connection names without changing their event envelope format.
