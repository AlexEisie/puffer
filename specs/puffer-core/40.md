# puffer-core update 40

Connector, connection, and workflow create tools now use the connection-driven
workflow model.

Tool surface:

- `ConnectorCreation` returns a starter connector template, skill text, and
  Python protocol skeleton.
- `ConnectorRegister`, `ConnectorUpdate`, and `ConnectorDelete` manage custom
  connector templates.
- `ConnectorList` returns built-in plus custom connector templates.
- `ConnectorAct` enforces connector action metadata and runs a typed connector
  `act` subprocess when available, falling back to the legacy outbound bridge
  for built-in message send actions.
- `ConnectionCreate`, `ConnectionList`, and `ConnectionDelete` manage
  authorized connection records. Create runs `auth-ok` when the connector
  exposes it, rejects explicit failed auth, and creates no-auth connectors as
  already authenticated so they can become active when a consumer appears.
  List performs routine auth checks and returns one-time `auth_notices` for
  connections that just degraded.
- `WorkflowCreate` is the workflow write path. It persists a direct connection
  trigger plus action or YAML action graph. The referenced connection must
  already exist; use `ConnectionCreate` before creating the workflow.
- `WorkflowValidate`, `WorkflowList`, `WorkflowToggle`, and `WorkflowInspect`
  operate over both direct workflow bindings and native workflow definitions.

Workflow inspection returns historical runs as `[RunID, TriggerInfo,
ActionSummary]` and run details as `[TriggerInfo, ActionLog]` for direct
workflow bindings. Native workflow runs keep their existing node-level run
details.

Compatibility:

Legacy subscription Rust handlers remain for old callers, but the bundled
model-visible resources use connector, connection, and workflow create terms.
