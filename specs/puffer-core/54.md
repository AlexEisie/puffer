# Lark Thread Message Reads

## Scope

Extends the internal `Lark` runtime tool's `read_messages` action.

## Behavior

- `read_messages` accepts exactly one of `chat_id` or `thread_id`.
- `chat_id` reads regular chat history through the Lark `chat` container.
- `thread_id` reads thread replies through the Lark `thread` container.
- Output records the selected `container_type` and `container_id`.

## Compatibility

Existing `chat_id` callers are unchanged. Thread reads add coverage for Lark
reply contexts without moving outbound replies out of `ConnectorAct`.
