# RequestSecret Loose Query Input

## Summary

`RequestSecret` now tolerates provider-emitted loose query input instead of
failing before the secret workflow can run.

## Behavior

- A bare string input is treated as `action: "search"` with the string as the
  metadata query.
- An object with `query` but no `action`, `id`, or `label` is also treated as
  a metadata search.
- These fallback searches return only secret metadata and do not reveal raw
  secret values or register `PUFFER_SECRET_...` placeholders.

## Compatibility

Existing object inputs keep their behavior: omitted `action` with `id` or
`label` still requests a masked placeholder, explicit `search` still searches,
and explicit `create` still writes to the encrypted vault after approval.
