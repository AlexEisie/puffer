# Workflow Draft Creation

## Scope

The `/workflows` local slash command now supports `new` and `create` for
creating a disabled native workflow draft directly from the TUI command path.

## Behavior

- `/workflows new [slug] [connection-slug]` persists a starter workflow in the
  workspace workflow store.
- The slug is normalized to kebab-case ASCII and suffixed when a workflow with
  that slug already exists.
- Drafts are created disabled so a newly saved trigger cannot fire before the
  user edits and resumes it.
- When a connection slug is supplied, the draft uses a connection trigger with a
  permissive pattern.
- Without a connection argument, Puffer chooses the first trigger-ready
  connection from the subscription runtime, preferring active consumers, and
  falls back to the `workspace.task.created` subscription topic when connector
  runtime state is unavailable.

## Compatibility

The existing `/workflows list`, `connections`, `connectors`, `tasks`, and `runs`
subcommands are unchanged. The registry argument hint includes `new` so command
search and help make the creation path discoverable.
