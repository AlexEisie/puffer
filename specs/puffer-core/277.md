# Generated Media Missing Sidecar Metadata

## Scope

Generated media attachment metadata can be synthesized for timeline rendering
when an `ImageGeneration` tool result references an image artifact whose durable
artifact sidecar is missing.

## Change

- `generated_media_attachment_metadata_with_fallback` returns authoritative
  sidecar metadata when the sidecar can be loaded.
- If the sidecar path is absent, the helper returns image metadata with
  `state = "missing"` from the tool result MIME type and byte count.
- Artifact ids must remain simple media artifact identifiers before fallback is
  considered.
- Existing but invalid sidecars are not masked by fallback.
- Existing sidecars with unsafe image paths still return no metadata.

## Compatibility

Only missing sidecars use tool-result fallback metadata. Corrupt sidecars,
non-image artifacts, invalid ids, and unsafe paths remain rejected.
