# Telegram QR Workflow Tool

- The consolidated internal `Telegram` workflow tool now accepts
  `action=import_desktop`, `action=login_qr`, and `action=login_qr_wait`.
- `action=import_local` is retained as a compatibility alias and routes to the
  Telegram Desktop importer.
- `login_qr` waits for `login_qr`, `login_complete`, or `login_error` and
  returns `status=qr_pending` when the agent must show the approval URL.
- `login_qr_wait` waits for `login_complete`, refreshed `login_qr`, or
  `login_error`; its tool timeout tracks the requested subscriber wait window.
- Direct legacy tool ids `TelegramImportDesktop`, `TelegramLoginQr`, and
  `TelegramLoginQrWait` route to the same implementation.
