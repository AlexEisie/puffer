# Lark Workflow Tool

## Scope

Registers the internal `Lark` runtime workflow tool and permission summaries.

## Runtime

`runtime:workflow:lark` dispatches to `workflow::lark::execute_lark`, which
supports:

- app credential setup
- user-token setup
- environment import
- chat listing/search
- user search
- chat message reads
- message search
- batch message detail reads

Credential setup validates against Lark before writing secret material and then
creates or updates the corresponding connection record.

## Permissions

Permission summaries are action-specific so approval prompts distinguish
credential writes, environment import, chat/user/message reads, and message
searches.

## Compatibility

The workflow tool intentionally does not implement message sends. External side
effects remain connector actions so `ConnectorAct` permission categories and
audit summaries stay the authority for outbound behavior.
