# Agent VideoGeneration Workflow Tool

## Scope

Core agent workflow execution now exposes `VideoGeneration` as the
model-callable text-to-video tool.

## Change

- Added `runtime::claude_tools::workflow::video_generation` with a
  `VideoGenerationInput` boundary of `prompt`, scalar string `parameters`, and
  optional `purpose`.
- Added `VideoGenerationMediaContext` as a sibling to the existing
  `ImageGenerationMediaContext`; the image context name and behavior stay
  unchanged.
- `VideoGeneration` resolves literal prompt text or a safe workspace-relative
  prompt file, reads `config.media.video`, merges saved video parameters with
  tool overrides, and builds an `ExactMediaGenerationRequest` with
  `kind = "video"` and `count = 1`.
- Execution reuses `generate_exact_media_with_cache`, so provider/model/adapter
  validation, OpenAI-video polling, downloads, persistence, and auth behavior
  remain in the existing exact media runtime.
- The workflow dispatcher passes separate image and video media contexts and
  routes only the exact `VideoGeneration` tool id to the new workflow.
- The runtime system prompt now tells the model to use `VideoGeneration` for
  video creation, one call per logical video request, and to report unsupported
  image/reference/frame video requests as text-to-video only.

## Compatibility

No generic `MediaGeneration` tool, compatibility alias, image-to-video input,
frame input, count input, seed input, or provider-specific field was added.
Unavailable video settings fail with
`video media provider/model/adapter is not configured`.
