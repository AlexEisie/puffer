# Image Generation Batch Limits

## Scope

Descriptor-driven `images_json` execution must support providers whose image
endpoint can generate fewer images per request than Puffer's requested count.

## Change

- `MediaExecutionDescriptor` now accepts optional `max_images_per_call`.
- `images_json` splits a generation into ordered HTTP calls when the descriptor
  declares a per-call limit.
- Per-call response validation still requires each batch to return the number
  requested for that batch.
- Count-aware provider parameters are derived from the largest per-call batch,
  not the total requested count.
- Unlimited descriptors keep the previous single-request batch behavior.

## Compatibility

Existing provider descriptors omit `max_images_per_call` and retain their
current request shape. Limited providers avoid sending `n` for one-image calls
because the existing request builder only emits `n` when the per-call count is
greater than one.
