# Workflow Connection Filter Terms

## Summary

`/workflows connections` now advertises stable connection filter terms and
matches the same monitor and lifecycle concepts shown in connection rows.

## Behavior

- The connection section prints a `filters:` line before result counts.
- Supported terms are `trigger-ready`, `no-trigger`, `monitor`, `repair`,
  `active`, and `idle`.
- Trigger-ready connections index `monitor` and their deterministic
  `/monitor <connection>` command.
- Non-trigger connections index `no-trigger`, `no trigger`, and `setup-only`.
- Connection search includes lifecycle state, consumer state, repair command,
  connector description, connector skill, runtime hints, and action slugs.

## Compatibility

The filter line and extra search terms are additive. Existing row formatting,
repair commands, monitor commands, and result counts remain text-based.
