# Generated Video Poster Preview

Date: 2026-06-09

## Scope

Core media runtime persists a single poster JPEG beside each generated video
artifact and exposes that poster through the existing artifact preview reader.

## Behavior

- `MediaArtifact` has an optional typed `preview` field for poster metadata.
- Poster metadata uses `kind = "poster"` with `available` or `missing` state.
- Completed video jobs write video bytes first, then attempt poster extraction
  before saving the artifact sidecar.
- Poster extraction runs one deterministic `ffmpeg` command, scales the long
  edge to 480 px, uses JPEG quality `3`, and is bounded by a 10 second timeout.
- Poster extraction failure never fails the completed video artifact. The
  sidecar records a missing poster preview with a diagnostic reason.
- Available posters are stored as `.puffer/media/artifacts/<artifact-id>/poster.jpg`.
- Video poster preview reads canonicalize under the artifact directory, reject
  symlink escapes, and only return sniffed supported image bytes.

## Compatibility

Image artifact preview behavior is unchanged. Generated video playback still
uses the generated video access ticket path and is not routed through poster
preview metadata.
