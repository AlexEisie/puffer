# RequestSecret Name Label Compatibility

`RequestSecret` accepts provider-emitted payloads that include both `name` and
`label`.

- `name` and `label` deserialize as separate optional fields instead of a serde
  alias pair, avoiding duplicate-field failures.
- Runtime normalization uses a non-empty `label` first and falls back to a
  non-empty `name`.
- Search payloads with empty optional fields remain valid when they include a
  non-empty `query`.
- The compatibility preserves existing `name`-only create and collect calls.
