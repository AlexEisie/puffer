# Alertmanager `/connect` Webhook Preset

## Change

`/connect alertmanager-webhook <connection-name>` now runs the deterministic
serve-mode webhook preset flow.

## Flow

- Resolve `alertmanager-webhook` from exact slug or searchable terms such as
  `prometheus alertmanager`.
- Ask for the bind address.
- Ask for the URL path and default blank answers to `/alertmanager`.
- Write `[connectors.webhook]` in `.puffer/connectors.toml` with the selected
  display name, bind address, path, and `Alertmanager webhook ready.` welcome
  message.
- Return the same `puffer serve` next-step summary as the other serve-backed
  webhook presets.

## Compatibility

The preset remains setup-only. It does not claim workflow trigger support until
the serve-backed webhook runtime can publish connector trigger events directly.
The top-level `/connect` dispatch now asks the serve configuration module
whether a slug is a webhook preset, avoiding repeated slug lists in the
line-sensitive command helper.
