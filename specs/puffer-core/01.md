# puffer-core update 01

## Purpose

Telegram login workflow tools now report the subscriber's actual login state
instead of assuming a control command succeeded once stdin accepted it.

## Design

`TelegramLoginStart`, `TelegramLoginSubmitCode`, and
`TelegramLoginSubmitPassword` send subscriber commands through
`SubscriptionManager::send_command_and_wait`.

## Logic

`TelegramLoginStart` waits for either `login_awaiting_code` or `login_error`.
Code submission waits for `login_complete`, `login_awaiting_password`, or
`login_error`. Password submission waits for `login_complete` or `login_error`.

## Contracts

The tools only ask the agent to collect a code after the Telegram subscriber
emits `login_awaiting_code`. Subscriber errors and timeouts are returned as
tool errors with the subscriber's payload message when available.
