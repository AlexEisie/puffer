# Plan slug test isolation

## Summary

Plan slug tests now serialize access to the in-process slug cache.

## Design

- The production cache remains unchanged and keyed by session id.
- The test module uses a local mutex guard around cases that clear or override
  cached slugs.
- This prevents parallel `cargo test` runs from clearing the cache while another
  plan test is asserting a copied fork path.

## Compatibility

Runtime plan behavior is unchanged. The update only removes a flaky unit-test
race observed during full workspace verification.
