# Connect Connector Search Resolution

`/connect` now treats the first argument as a connector slug or connector search
term before asking for connection details.

- Exact built-in connector slugs keep the fast path and do not need a catalog
  lookup before their deterministic setup flow starts.
- Non-exact terms are matched against connector slug and description words. A
  single match is selected automatically; multiple matches are presented through
  the existing searchable `AskUserQuestion` choice.
- Unknown terms are preserved so the later connector registration path reports
  the same not-found diagnostics as before.
- Connector catalog formatting moved into a `connect/catalog.rs` helper module
  to keep the command helper under the repo line limit while preserving the
  same built-in fallback when the subscription runtime is not running.

Compatibility: existing `/connect <connector-slug> <connection-name>` commands
continue to work, while partial commands such as `/connect tele telegram-user`
resolve to `telegram-login`.
