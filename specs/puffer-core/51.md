# Slack internal runtime tool

## Summary

The runtime implements the consolidated internal `Slack` tool for Slack app
auth, Slack login auth, local app import, and read-only lookup operations.

## Design

- `configure_app` authenticates bot/app tokens with `auth.test`, persists the
  credential, and registers a `slack-app` connection.
- `login_token`, `login_browser`, and `import_local` authenticate account
  credentials, persist them, and register a `slack-login` connection.
- Lookup actions list conversations, search conversations/users, read channel
  history or thread replies, and run Slack message search.
- Runtime output includes connection/workspace/user metadata and never returns
  Slack tokens.
- Permission summaries describe Slack auth, import, lookup, and message-read
  actions for approval prompts.

## Compatibility

Slack outbound side effects remain connector actions. The internal runtime
tool is for auth and lookup only, matching the connector/action split used by
Telegram.
