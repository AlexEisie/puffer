# Exact Image Count Enforcement

## Scope

Exact image generation through the `images_json` adapter must either produce
the requested number of images or fail the media job.

## Change

- The adapter now validates the response `data` length before writing image
  artifacts.
- If a provider returns fewer images than requested, the job is persisted as
  `Failed` with `image generation returned <actual> image(s), expected <count>`.
- Partial image artifacts are not written for under-produced responses.
- Descriptor-backed `sequential_image_generation` parameters are promoted from
  `disabled` to `auto` when the request count is greater than one.

## Compatibility

Providers that do not declare `sequential_image_generation` keep their existing
request shape. Successful multi-image responses still persist one job with
ordered artifact records.
