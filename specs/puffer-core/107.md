# Vercel Webhook Connect UX And Watcher Filtering

## Scope

Adds `vercel-webhook` to the deterministic `/connect` and workflow connector
search surfaces, and tightens resource watcher filtering for editor noise under
resource directories.

## Design

- `/connect vercel-webhook <connection>` uses the generic webhook serve-mode
  writer and stores the connection under `[connectors.webhook]`.
- The preset defaults the webhook path to `/vercel` and describes the setup
  method as `Vercel event webhook`.
- Connector search metadata marks the preset as `serve`, setup-only, and
  searchable by terms such as `vercel`, `deployment`, `project`, `domain`, and
  `flag`.
- The resource watcher now classifies watched-subdir paths as concrete child
  resource writes, parent-only notifications, noisy dotfile/tempfile writes, or
  unrelated paths.
- Reload signals are raised only for concrete child resource paths so macOS
  parent-directory notifications caused by editor swap files do not trip the
  reload signal.

## Compatibility

No existing connector slug changes. The preset follows the same serve-mode
contract as the other webhook presets and does not claim workflow trigger or
connector action capabilities. The watcher remains additive for real resource
file writes while ignoring ambiguous parent-only events.
