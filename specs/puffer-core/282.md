# Canvas Natural Control IDs

## Summary

Canvas state patching now accepts natural control ids that contain spaces or
punctuation. Generated Canvas specs already store control ids as JSON object
keys when initial state is written, so patching must accept the same key space
when users change inline controls.

## Behavior

- Empty state keys are still rejected.
- State keys longer than 128 bytes are still rejected.
- Control characters are rejected to avoid malformed or unreadable state.
- Human-readable ids such as `Run tests?` are accepted and persisted.

## Compatibility

Canvas ids used in file paths remain strictly validated. This change only
relaxes the validation for keys inside the JSON `values` object.
