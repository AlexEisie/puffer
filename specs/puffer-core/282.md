# Native Image Attachment Runtime Input

Runtime state and provider serialization now support local uploaded images as
native multimodal input.

## State Contract

- `RenderedAttachment` stores the durable `StoredAttachment` reference used by
  session replay and current-turn dispatch.
- `RenderedMessage.attachments` preserves user-message attachment references in
  `AppState`.
- `RenderedAttachment.model_url` holds only the transient hydrated model URL and
  is skipped during serialization.

## Conversation Conversion

`transcript_to_items` converts hydrated user image attachments into
`ContentPart::Image` next to the user text. Current-input dedup treats a final
user message whose first text part matches the input as already present even
when image parts follow it.

## Provider Wire Formats

OpenAI Chat Completions serializes user image parts as `image_url` content.
OpenAI Responses serializes them as `input_image` content. Anthropic Messages
parses hydrated `data:` URLs into base64 image source blocks.

Compaction summary input renders image parts as small markers such as
`[image attachment: image/png]` and never embeds base64 data URLs.

## Runtime Gate

Native image turns are rejected before provider dispatch when the selected
model is text-only, the provider API is not image-enabled, an image is
unhydrated, or a hydrated data URL is malformed. Image turns also bypass
simple-turn local replies so they always reach the provider path.

Static provider resources declare image input support for this feature.
Unknown discovered models remain text-only until curated metadata explicitly
adds image input capability.
