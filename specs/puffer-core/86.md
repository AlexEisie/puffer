# GitHub Webhook Connect Flow

## Summary

`/connect` can configure the `github-webhook` connector deterministically, and
the terminal workflow connector dashboard can search and display the preset.

## Behavior

- `/connect github-webhook <connection>` asks for a bind address and GitHub
  webhook path through structured user questions.
- The flow writes `[connectors.webhook]` in workspace `.puffer/connectors.toml`
  with the requested display name, bind address, normalized path, and a GitHub
  welcome message.
- Connector slug resolution accepts GitHub-specific search terms such as
  `github event` and still disambiguates generic webhook searches.
- `/workflows connectors github` lists the preset with `serve`, `no-trigger`,
  and `/connect github-webhook github-webhook`.

## Compatibility

The preset reuses the existing serve-mode webhook config table, so it remains
compatible with `puffer serve`. It does not add workflow trigger support until
webhook delivery has a durable connector event stream.
