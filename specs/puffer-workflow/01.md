# puffer-workflow update 01

Workflow triggers now include `connection` in addition to the legacy
`subscription` trigger.

The connection trigger stores:

- `connection_slug`: authorized connector instance to consume.
- `filter`: optional structured filter JSON evaluated by
  `puffer-subscriptions`.
- `pattern`: optional shorthand regex retained for CLI compatibility.
- `classify_prompt`: optional classifier prompt.

Native AgentFlow-style workflow storage, validation, DAG execution, and run
history are unchanged. The new direct workflow create path is owned by
`puffer-subscriptions`; `puffer-workflow` only preserves compatibility for
native workflows that still use the old workflow store and DAG runner.
