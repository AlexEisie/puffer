# Succinct Telegram Message Search

- `TelegramSearchMessages` now accepts a `succinct` boolean.
- Full search output remains the default for compatibility.
- Succinct output sets `payload.format` to `succinct`, keeps one compact chat
  summary, and returns each result as `match` plus `context`.
- Succinct message entries use scalar `from`, `offset`, `date`, `outgoing`,
  `is_match`, `text`, and `text_truncated` fields instead of repeating full
  sender peer objects on every context message.
- Context offsets are relative to the matched message, so agents can directly
  ask for "the second message after the match" without inferring positions
  from Telegram message ids.
