# Outbound media and poll interaction

- Telegram outbound send handling moved to a dedicated outbound helper module.
- Plain `SendMessage` remains unchanged for text-only sends, including
  `reply_to`.
- Media sends support one local path, URL, or multiple attachments. A single
  attachment is sent as one media message; multiple attachments are sent as a
  Telegram album.
- Attachment captions use the attachment-specific caption when supplied, or
  the command text for the first attachment. Later attachments default to an
  empty caption.
- Attachment kind `auto` sends common image MIME/extensions as Telegram photos
  and all other sources as documents. Explicit `photo`, `document`, and `file`
  are honored for local uploads; URL `file` is sent as a document because
  Telegram external media does not expose force-file semantics through
  grammers.
- Search and live message output now include structured `poll` metadata:
  question, status, total voters, answer indexes, answer text, voter summaries,
  chosen/correct flags, and `option_hex` tokens.
- Succinct poll text includes answer indexes so agents can vote without
  requesting full JSON when the text is enough.
- The `vote_poll` connector action fetches the poll message by id, resolves
  answer selectors by index, exact answer text, or hex option token, and calls
  Telegram `messages.sendVote` through the same `telegram_act` bridge as other
  Telegram-specific actions.
