# Telegram connector action coverage

- The Telegram personal-account subscriber handles `Custom { op:
  "telegram_act" }` once authenticated.
- Supported actions cover poll voting, message edit/delete/forward, pins,
  reactions, read and mention state, typing/upload indicators, join/leave,
  invites, kick/ban/unban, account profile and username updates, avatar upload,
  group title/name/username/photo updates, and stories.
- Unauthenticated `telegram_act` commands emit `telegram_act_error` with the
  attempted action, peer when available, and a login-required error.
- Action success emits `telegram_act_complete`; validation, permission, peer
  resolution, or Telegram RPC failures emit `telegram_act_error`.
- Poll voting now returns through the connector action bridge instead of a
  dedicated runtime command variant.
