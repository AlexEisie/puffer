# Telegram Reply Metadata

- Incoming Telegram subscriber events now expose `reply_to` when Telegram
  provides a reply header, plus `reply_count` when available.
- Message search includes `reply_to` on both full and succinct payload items.
  Full JSON fetches a one-level `resolved_message` for message replies when
  Telegram allows it.
- Reply metadata includes message/story kind, peer id, replied message id,
  thread top id, scheduled/forum-topic/quote flags, quote text and offset,
  reply media kind, and forwarded reply metadata.
- Succinct search remains plain text by prefixing message text with a compact
  label such as `[reply to #42: previous text]`.
