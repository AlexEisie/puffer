# Telegram Message Search

- The Telegram user subscriber now accepts `TelegramSearchMessages` control
  commands with `peer`, `query`, optional `limit`, and optional `context`.
- Message search requires an authenticated personal-account session and emits
  `message_search` on success or `message_search_error` on validation,
  resolution, or Telegram search failures.
- `peer` intentionally accepts only a public `@username` or stable numeric id
  from peer lookup. Chat titles are not accepted as send/search targets.
- Results include the matched message plus surrounding context messages. The
  matched message is marked with `is_match: true`.
- Context is gathered from the chat history rather than by title lookup. Older
  context is fetched by offset id; newer context scans recent chat history up
  to a bounded limit to avoid unbounded reads.
- Message payloads include message id, RFC3339 date, sender peer payload,
  outgoing flag, text, and truncation status.
