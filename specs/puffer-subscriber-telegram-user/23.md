# puffer-subscriber-telegram-user update 23

Telegram user subscribers now poll recent history while the live update loop is
running.

- The subscriber still performs startup catch-up before entering live update
  handling.
- While running, it repeats history catch-up every 30 seconds using the delivery
  cursor to avoid duplicate message events.
- Periodic catch-up failures are logged and do not stop the subscriber, so a
  transient Telegram history read failure does not disable live updates.
- This keeps desktop-imported Telegram sessions useful even when Telegram
  Desktop remains logged in and live update delivery is incomplete.
