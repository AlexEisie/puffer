# puffer-subscriber-telegram-user update 01

## Purpose

Telegram user-account login now falls back to hardcoded public Telegram Android
API credentials instead of the rejected Telegram Desktop pair.

## Design

Credential resolution is a single pair-based operation. It accepts credentials
from explicit `TelegramLoginStart` input, persisted credentials,
`PUFFER_TELEGRAM_API_ID`/`PUFFER_TELEGRAM_API_HASH`, or the built-in Android
default pair.

## Logic

Only complete explicit, persisted, or environment credential pairs are used.
Incomplete pairs are ignored so the login path still has a complete built-in
default pair. The default `InitParams` identify as an Android client to match
the built-in pair.

## Contracts

Operators may provide Telegram app credentials from `my.telegram.org/apps`, but
first login can proceed with the built-in default pair. Successful logins
continue to persist whichever credential pair was used for later reconnects.
