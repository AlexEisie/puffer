# Telegram Peer Lookup

- The Telegram internal CLI now exposes `telegram list-peers` and
  `telegram search-peers <query>` so agents can resolve a stable numeric peer
  id before sending a message or creating a workflow filter.
- Peer lookup is implemented as a subscriber control command over the existing
  authenticated `telegram-user` process. It does not send messages and it does
  not overload `SendMessage` with title resolution.
- The subscriber walks the Telegram dialog list and returns JSON peer records
  with `id`, `numeric_id`, `kind`, `title`, `username`, `handle`, `usernames`,
  and `handles`.
- `id` is serialized as a string and is the value agents should pass to
  Telegram sends or workflow filters. `numeric_id` is included only for human
  diagnostics.
- Search is case-insensitive and matches dialog title, username, alternate
  usernames, handles, or numeric id. Optional `peer_kind` filters support
  `user`, `group`, and `channel`.
- The telegram skill now instructs agents to run peer lookup first when the
  user names a chat such as `C & Jason`.
