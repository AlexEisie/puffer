# Telegram Subscriber Command Fairness

- The Telegram user subscriber now prioritizes stdin control commands while in
  the authenticated update loop.
- This prevents high-volume Telegram update backlogs from delaying commands
  such as auth probes, peer searches, message searches, and outbound sends.
- Message updates still flow continuously when no command is ready; the change
  only affects scheduling when both a command and a Telegram update are ready.
- This keeps connection auth monitoring from falsely degrading a healthy
  connection during message bursts.
