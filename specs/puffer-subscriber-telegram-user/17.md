# Side-Effect-Free Auth Checks

- The Telegram user subscriber now handles `telegram_auth_ok` by calling
  `is_authorized` on the active client and emitting an `auth_ok` control
  event.
- Login-required subscribers return `auth_ok` with `ok = false` instead of
  requiring a peer-list command to infer auth state.
- Health probes no longer call peer or dialog APIs, so pending Telegram message
  updates remain available for the workflow router.
