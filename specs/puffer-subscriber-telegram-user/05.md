# QR Login And Desktop Import Scope

- Telegram Desktop local import remains `TelegramImportTdata` at the runtime
  boundary, but user-facing naming is scoped to Telegram Desktop `tdata`.
- The prior `import-local` path is compatibility-only and does not claim to
  import native macOS Telegram storage.
- `TelegramQrLoginStart` creates a fresh unauthenticated MTProto client,
  calls `auth.exportLoginToken`, and emits `login_qr` with a short-lived
  `tg://login?token=...` URL.
- `TelegramQrLoginWait` listens for `UpdateLoginToken`, re-exports the token,
  handles datacenter migration via `auth.importLoginToken`, saves the resulting
  session, reconnects, verifies authorization, and emits `login_complete`.
- If approval is not observed before the wait timeout, the subscriber tries to
  refresh the QR token and emits a new `login_qr` instead of discarding state.
