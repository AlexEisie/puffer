# puffer-subscriber-telegram-user update 03

## Purpose

Telegram user-account login now uses the public Telegram Desktop credential
pair by default so `cargo run` can request login codes without environment
overrides.

## Design

Credential resolution still prefers explicit tool input, persisted credentials,
and complete `PUFFER_TELEGRAM_API_ID`/`PUFFER_TELEGRAM_API_HASH` pairs before
falling back to the built-in default. The fallback `InitParams` now identify as
Telegram Desktop on Linux to match that public pair.

## Logic

When a running subscriber is already authorized, a new `TelegramLoginStart`
command emits `login_complete` with the current user summary instead of
requesting another code. The Puffer TUI tool wrapper treats `login_complete` as
a successful terminal event.

## Contracts

Fresh login still returns `awaiting_code` and requires the subsequent code or
password tool calls. Reconnecting an already-authorized session is idempotent
and does not trigger Telegram `AUTH_RESTART` from duplicate code requests.
