# puffer-subscriber-telegram-user update 02

## Purpose

Telegram login-code submission now tolerates a transient MTProto transport
disconnect between requesting a code and submitting it.

## Design

The subscriber classifies `sign_in` transport EOF/reset errors as retryable.
It preserves the in-flight login token, reconnects with the same API credential
pair and persisted pre-auth session, then retries the submitted code once.

## Logic

`request_login_code` saves the pre-auth session so DC routing survives a
reconnect. Non-auth transport failures such as `read 0 bytes` no longer clear
the token immediately; invalid-code and other auth errors still emit terminal
`login_error` events.

## Contracts

Agents still call `TelegramLoginStart`, then `TelegramLoginSubmitCode`, then
optional `TelegramLoginSubmitPassword`. A correct code should not be rejected
solely because Telegram closed the socket before `sign_in`.
