# puffer-subscriber-telegram-user update 24

Telegram user subscribers now suppress muted notification streams before events
reach the runtime bus.

- Message delivery refreshes per-chat notification settings on demand through
  Telegram `account.getNotifySettings`, with a short cache to avoid repeated
  RPCs for active chats.
- Messages from muted or notification-silent chats, and sender-silent messages,
  are recorded in the delivery cursor but are not emitted as connector events.
- Dialog and raw notification updates still update the local mute cache so
  list-peers and the live stream converge quickly after mute changes.
- `list_peers` resolves current notification settings for returned peers so
  monitor backfill agents skip the same muted chats as the live stream.
- Monitor router filtering remains as a defensive fallback for any synthetic or
  legacy events that still carry `notification_muted` or `notification_silent`.
