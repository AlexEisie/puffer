# Native macOS Telegram.app Desktop Import

- `TelegramImportTdata` remains the runtime command behind user-facing
  `import-desktop`, but on macOS it now falls back to native Telegram.app
  storage when the default Telegram Desktop `tdata` path is absent or the
  caller points at a native Telegram container/prefix.
- Native import reads `accounts-metadata/atomic-state` from the Telegram app
  group container, prefers the current account, honors `account_index` against
  the sorted account records, and reports `source_kind: "macos_native"` in the
  import payload.
- The importer decrypts `.tempkeyEncrypted` with the Telegram.app default
  passcode key or the supplied passcode, opens the selected Postbox SQLCipher
  database read-only, and extracts `persistent:datacenterAuthInfoById` from the
  keychain table.
- Only permanent high-word-zero datacenter auth entries are imported. The
  resulting grammers session stores all supported DC keys and initially marks
  the account as signed in so reconnect verification can run.
- Reconnect verification now tries all imported candidate datacenters if the
  first selected DC is not accepted. On success it refreshes the saved session
  with the verified Telegram user id and winning datacenter.
- Native import persists the public macOS Telegram API credential pair for
  reconnects; Telegram Desktop `tdata` imports keep the existing public Desktop
  credential pair.
