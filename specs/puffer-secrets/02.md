# Token-Scored Secret Search

Secret metadata search now tolerates keyword-heavy agent queries.

- Queries are tokenized across punctuation and whitespace, then deduplicated
  case-insensitively.
- Specific site, domain, username, or label tokens drive matching.
- Generic login words such as `login`, `account`, and `password` no longer
  exclude a stored credential when a specific token matches.
- Generic-only queries still work for broad searches such as `api key`.
- Results are sorted by match score first, then by recency and label.
- Secret values remain encrypted and are never inspected during search.
