# Resolved Secret Metadata

## Summary

`ResolvedSecret` now preserves non-secret metadata from the encrypted record
when a caller reveals a secret. The reveal path returns the decrypted value
alongside `id`, `label`, `description`, `username`, and `origin`.

## Architecture

- `SecretRecord` remains the storage source of truth for encrypted value and
  searchable metadata.
- `SecretSummary` remains metadata-only and never carries decrypted material.
- `ResolvedSecret` is the runtime reveal shape. It now carries the same
  non-secret account hints as `SecretSummary` so higher-level tools can fill
  username/account fields without misusing a secret placeholder.

## Compatibility

Existing callers that only read `value` continue to work. The new fields are
optional and populated from existing records without changing the store format.
