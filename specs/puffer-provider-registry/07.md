# 07 - Codex Experimental Bearer Import

## Summary

Codex credential import now treats `experimental_bearer_token` from
`~/.codex/config.toml` as the preferred OpenAI credential.

## Design

The Codex importer reads `experimental_bearer_token` from either the selected
`model_providers.<name>` entry or the top-level Codex config. When present, the
token is returned as an API-key-style bearer credential and Codex `auth.json`
OAuth/API-key candidates are not emitted for that import pass.

The importer still carries over the Codex OpenAI base URL, static headers,
environment-derived headers, and query parameters from the selected provider.

## Contracts

- The experimental bearer token takes precedence over Codex `auth.json`.
- The token may be imported even when `~/.codex/auth.json` does not exist.
- Stored credentials use the existing API-key shape so request builders send
  `Authorization: Bearer <token>`.
- Descriptions and import metadata must not expose token material.
- When the experimental token is absent, existing Codex `auth.json` import
  behavior is unchanged.
