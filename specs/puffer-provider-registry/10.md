# Exact Image Media Descriptors

Provider descriptors now support optional `media.image` capability metadata for
exact image generation.

- `ProviderDescriptor.media` is optional, so providers without media metadata
  continue to parse.
- Image descriptors declare static discovery metadata, one execution adapter,
  concrete model ids, supported operations, and allowed parameter values.
- Validation rejects empty ids, `auto`, wildcard or regex-like model ids,
  missing operations, empty declared parameter lists, and empty execution paths.
- Only implemented enum values are modeled: `static` discovery,
  `openai_images` execution, and `generate` operation.

The registry schema remains declarative. It does not infer image generation from
chat model descriptors or provider names.
