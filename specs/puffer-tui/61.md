# Searchable Question Matching

The TUI `AskUserQuestion` overlay now precomputes normalized search text for
each choice option when the prompt is opened.

- Searchable choices no longer lowercase every label and description every time
  the overlay renders rows or clamps selection.
- Connector-style searches support multiple terms, so a query like
  `slack account` can match an option whose slug and description contain those
  words separately.
- Selection, preview rendering, arrow navigation, and Enter confirmation still
  use the existing filtered-option index mapping.

Compatibility: the prompt schema is unchanged. Existing searchable choice
payloads continue to use `"searchable": true` with normal option labels and
descriptions.
