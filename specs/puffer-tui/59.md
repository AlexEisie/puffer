# TUI Connect Question Worker

The TUI now handles `/connect` through a dedicated local worker instead of the
provider-turn worker.

- Submitted `/connect` commands are intercepted before shell shortcuts and
  provider prompt submission.
- The worker installs the existing `AskUserQuestion` prompt handler, runs
  `puffer_core::execute_connect_flow`, and forwards each question to the normal
  user-question overlay.
- Connector auth failures are converted into a completed local turn with
  `/connect failed: ...` text, avoiding the provider request failure surface for
  local auth issues.
- The worker returns the cloned auth store and session permission state without
  clearing provider auth state.
- While the worker runs, the pending-submit status hint reads `Connecting...`.
