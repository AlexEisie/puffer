# puffer-tui update 126

Workflow, connector, and monitor popup copy now matches the redesigned workflow
mental model.

## Design

The slash popup distinguishes three connector tasks:

- create a draft workflow from a connector event stream
- append connector events to a file
- create a monitor workflow for an event-capable connection

Configured connection rows and catalog rows use the same action phrasing so
`/workflows`, `/monitor`, and `/connect` feel like one flow.

## Contracts

- Popup replacements are unchanged.
- Search text still includes connector slugs, connection slugs, repair terms,
  monitor terms, workflow terms, and connector action slugs.
- `/monitor` background execution keeps the same AskUserQuestion flow, but the
  pending status hint now describes monitor workflow creation directly.

## Compatibility

No command syntax changes. Existing users can keep invoking `/connect`,
`/workflows new`, `/workflows append`, `/workflows delete`, and `/monitor` with
the same arguments.
