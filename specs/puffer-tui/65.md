# Workflow Connection Filter Hints

## Summary

The TUI-visible `/workflows connections` output now names the stable filter
terms users can type to inspect monitorable and repairable connections.

## Behavior

- `/workflows connections` prints `trigger-ready`, `no-trigger`, `monitor`,
  `repair`, `active`, and `idle` before the result count.
- `monitor` matches connections that can run `/monitor <connection>`.
- `active` and `idle` match consumer state, independent of the serialized
  connection lifecycle state.
- Connector metadata is indexed for connections, so users can combine terms
  such as `monitor telegram` or `repair slack`.

## Compatibility

The command remains plain text and query-driven. Existing workflow dashboard
sections continue to render in the same order.
