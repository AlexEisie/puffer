# Workflow Connector Result Counts

## Summary

The TUI workflow dashboard now mirrors the desktop Pipelines connector picker by
showing how many connector and connection rows match the current query.

## Behavior

- `/workflows connections [query]` prints `showing <visible>/<total>
  connections` below the section header.
- `/workflows connectors [query]` prints `showing <visible>/<total>
  connectors` below the section header.
- The default `/workflows` connector section keeps its trigger-focused scope and
  labels the count as `trigger-ready connectors`.
- Query summaries include the exact query text with Rust debug escaping so
  spaces and quotes remain unambiguous in terminal output.

## Connector Labels

- Trigger-capable connectors continue to show the `trigger` capability.
- Setup-only connectors shown by `/workflows connectors` now show `no-trigger`.
- Connector search matches `no trigger`, `no-trigger`, and `setup-only`, so
  users can quickly find connectors that can be configured but cannot directly
  start workflow triggers.

## Compatibility

This change only affects local command rendering and search metadata. It does
not change workflow storage, connector manifests, connection records, or desktop
daemon APIs.
