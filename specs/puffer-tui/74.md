# puffer-tui Update 74: Generated Session Titles in Resume Picker

## Design

The resume picker now treats a session's generated title as the display title
when no user-set display name exists. Manual display names remain authoritative.

## Architecture

The change is local to TUI overlay row rendering. Session metadata, daemon title
generation, and session-store persistence are unchanged.

## Logic

`/resume` rows resolve the visible title in this order:

1. `display_name`
2. `generated_title`
3. `<unnamed>`

Path, slug, and tag context continue to render after the title.

Session picker filtering also matches `generated_title`, so a generated title
can be typed into the picker just like a manual display name.

## Contracts

Desktop session title semantics already use `generated_title`; the TUI resume
picker now matches that visible fallback without changing command behavior.

## Compatibility

Existing sessions without generated titles still render as `<unnamed>`. Sessions
renamed by the user continue to show the manual name.
