# Pending turn loading indicator

Pending-turn rendering now treats an in-flight submit as active when it has a
start timestamp, prompt text, status hint, or pending tool calls. The loading
line is rendered from that active lifecycle state rather than only from the
visible prompt string.

This keeps the transcript footer honest during long-running agent work,
including edge cases where the prompt label is unavailable or omitted while the
background worker is still running. Queued prompt rendering is unchanged, and
the pending-submit render snapshot lives in its own small module so
`render.rs` stays below the repository line limit.
